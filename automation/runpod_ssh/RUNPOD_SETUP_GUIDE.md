# RunPod — пошаговая настройка пода для тестов DataProcessor (по кнопкам)

Цель: поднять GPU-под, к которому Claude сможет подключаться по SSH и гонять тесты
компонентов DataProcessor (action_recognition и далее по списку), которые требуют GPU/torch —
в самой Cowork-песочнице GPU нет.

Публичный SSH-ключ уже сгенерирован и лежит в репозитории:
`automation/runpod_ssh/id_ed25519.pub`. Его содержимое (нужно будет вставить в шаге 2):

```
ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAIApx5EZ5NLUAqSqdWjiMNg/PaZ34mx02aweR3DDWK7+w claude-cowork-trendflow
```

---

## Шаг 1. Аккаунт и оплата

1. Открой https://console.runpod.io/signup — зарегистрируйся (email + пароль или Google).
2. Подтверди email по ссылке из письма.
3. Слева в меню → **Billing** → **Add funds** — привяжи карту и закинь хотя бы $10-20
   (GPU уровня RTX 4090/A5000 стоит ~$0.27-0.35/час на Community Cloud).

## Шаг 2. Добавить SSH-ключ в аккаунт

1. Клик по иконке профиля (правый верхний угол) → **Settings**.
2. Найди блок **SSH Public Keys**.
3. Вставь ключ целиком одной строкой (текст выше, начинается с `ssh-ed25519`).
4. Сохрани (кнопка **Save** / галочка).

Это заставит RunPod автоматически прописывать этот ключ в `~/.ssh/authorized_keys`
на любом поде с официальным шаблоном (RunPod PyTorch и т.п.).

## Шаг 3 (рекомендуется). Network Volume — постоянное хранилище

Нужно, чтобы веса моделей/чекпоинты/датасеты не терялись при остановке/пересоздании пода.

1. Левое меню → **Storage** → **Network Volumes** → **+ Network Volume**.
2. Выбери дата-центр (регион) — от этого будет зависеть, какие GPU потом доступны.
3. Укажи размер, например **100 GB** (веса + npz-артефакты action_recognition и других компонентов).
4. **Create**.

## Шаг 4. Деплой пода

1. Кнопка **+ New** в правом верхнем углу консоли → **Pod**
   (или просто **Pods** в левом сайдбаре → **Deploy**).
2. **Template** (шаблон): в поиске введи `RunPod PyTorch` и выбери официальный образ
   (например `runpod/pytorch:2.x-...-cuda12.x-devel-ubuntu22.04`). В нём уже настроен
   SSH-демон и полноценный (full) SSH — то, что нужно для scp/rsync.
3. Опции под шаблоном:
   - **SSH terminal access** — включить (если появится поле для ключа — можно оставить
     пустым, ключ уже подхватится из аккаунта, см. Шаг 2).
   - **Start Jupyter notebook** — можно оставить включённым, не мешает, не обязателен.
4. **Pod name** — можешь переименовать, например `trendflow-dp-tests`.
5. Раздел **Compute** → выбери GPU:
   - Если создавал Network Volume на шаге 3 — включи фильтр **Network volume** и выбери его,
     список GPU сузится до совместимых с этим дата-центром.
   - Ищи **RTX 4090** (24GB, ~$0.34/ч Community) или **RTX A5000** (24GB, ~$0.27/ч) —
     этого достаточно для SlowFast/CLIP/MiDaS/Places365.
   - Обрати внимание на индикатор доступности (High/Medium/Low/Unavailable).
6. Раздел **Storage**:
   - **Container disk** — 20-50 GB хватит под код/venv.
   - **Persistent storage** — выбери созданный **Network Volume** (если делал шаг 3),
     иначе временный **Volume disk** (но он пропадёт при Terminate, только не при Stop).
7. Проверь **Summary** справа (цена в час, разбивка по GPU/диску).
8. Кнопка **Deploy Pod** (или **Deploy On-Demand**).

Под появится в списке **Pods** слева, статус сменится на **Running** через 30-90 секунд
(идёт скачивание образа).

## Шаг 5. Забрать данные для подключения

1. Клик по поду в списке **Pods** → открывается панель пода.
2. Вкладка/блок **Connect**.
3. Найди строку **SSH over exposed TCP** (не "Basic SSH" — тот не поддерживает scp/rsync).
   Команда выглядит так:
   ```
   ssh root@213.173.108.12 -p 17445 -i ~/.ssh/id_ed25519
   ```
   Если такой строки нет (виден только Basic SSH через `ssh.runpod.io`) — значит у пода
   нет публичного IP. Тогда: Terminate этот под и пересоздай, выбрав GPU/хост с публичным IP
   (обычно это Secure Cloud, либо часть Community Cloud — если непонятно, можно отфильтровать
   по региону/хосту с индикатором Public IP при выборе GPU, либо просто спросить меня —
   я подскажу, какой хост брать).
4. Из этой команды забери **IP** (после `root@`) и **порт** (после `-p`).

## Шаг 6. Вписать данные в файл для Claude

Открой файл в этом же репозитории:
`automation/runpod_ssh/POD_CONNECTION.md`

Заполни поля Host, Port, GPU, дату, статус. Это единственное, что нужно сделать руками —
дальше Claude (в любом Cowork-сеансе, где подключена эта папка) сам прочитает файл
и подключится.

## Остановка / очистка (когда закончили тесты)

- **Пауза на день-два**: кнопка **Stop** (квадрат) на поде. Данные в `/workspace`
  (network volume) сохранятся, но за сам volume продолжит капать оплата
  ($0.07-0.20/GB/мес в зависимости от статуса).
- **Совсем не нужен**: **Stop**, затем **Terminate** (корзина). Если данные важны —
  убедись, что они на Network Volume, а не на Container Disk (тот стирается безвозвратно).
- Новый под потом можно поднять заново на том же Network Volume — все веса/датасеты
  останутся на месте, повторять шаги 1-3 не нужно.
