# Автоматизация одного агента (Claude) для валидации компонентов TrendFlow

> Цель: я (Claude, Opus 4.8) работаю над компонентами **непрерывно, без ручного «продолжай»**;
> при вопросах — сначала ищу в интернете, если не решается — пишу тебе в **VK**, ты отвечаешь, я продолжаю;
> подами (RunPod) управляю сам, но start/stop/миграцию делаю **только после твоего подтверждения**;
> слежу за расходом токенов/деньгами и не выхожу за бюджет.
>
> Этот документ — что тебе сделать руками (с кнопками на сайтах) и что соберу я в коде.

## 0. Как это устроено (одна картинка)

```
        ТВОЙ ПК (Linux, крутится 24/7)
        ┌───────────────────────────────────────┐
        │  agent_runner.py  (Claude Agent SDK)   │
        │  ── цикл ──                            │
        │  1. взять следующий ⬜ компонент из     │
        │     COMPONENT_VALIDATION_CHECKLIST.md  │
        │  2. работать (bash/SSH к поду, правки)  │
        │  3. вопрос? → интернет → если нет →VK   │
        │  4. нужен под? → спросить в VK → API    │
        │  5. заштамповать → к п.1                │
        │  + budget_guard (токены/деньги/лимиты)  │
        └───────┬───────────────┬────────────────┘
                │ SSH/REST       │ Bot API
                ▼                ▼
          RunPod GPU-под     VK-сообщество (бот)
          (A4500)            ↕ личные сообщения
                                 ТЫ (телефон/десктоп)
```

Никакого сервера в облаке не нужно — раннер живёт на твоём ПК (у него уже есть SSH-ключи к поду
и `RUNPOD_API_KEY`). Для **одного** агента **n8n не нужен** — раннер сам шлёт вопросы в VK и читает
ответы. n8n добавим позже, когда будет второй агент (Sonnet).

## 2. Аутентификация агента (пункт A) — выбери ОДИН вариант

### Вариант A1 — Claude Max через Claude Code (рекомендую, раз ты и так берёшь Max)

Это **официально поддерживается** (Anthropic: «Use the Claude Agent SDK with your Claude plan»). Раннер
на Agent SDK берёт авторизацию из Claude Code, а Claude Code — из твоей подписки Max.

Шаги:
1. Оформи **Claude Max** на `claude.ai` → кнопка профиля (внизу слева) → **Upgrade / Manage plan** → выбрать **Max**.
2. Установи Claude Code (в терминале ПК):
   ```bash
   npm install -g @anthropic-ai/claude-code
   ```
3. Авторизуйся один раз:
   ```bash
   claude            # откроется окно логина в браузере → войди своим Max-аккаунтом → Authorize
   ```
   (или `claude setup-token` — сгенерит долгоживущий токен для headless-запуска).
4. Проверка: `claude -p "say ok"` должно ответить `ok`.

⚠️ **Честная оговорка по деньгам (важно):** с 15 июня 2026 headless-использование (Agent SDK / `claude -p`)
**не берётся из общего лимита Max**, а идёт из отдельного **кредит-пула** ($100/мес для Max 5×, $200/мес для
Max 20×), а сверх него — по **API-тарифам**. То есть Max даёт тебе месячный «пакет» на автоматизацию, дальше —
поштучная оплата токенов. Поэтому в п.8 обязателен бюджет-контроль.

⚠️ **Чего делать НЕЛЬЗЯ:** вытаскивать OAuth-токен Max и подставлять его в свой самописный сервис/n8n напрямую —
это нарушение Consumer ToS (OAuth Pro/Max разрешён только внутри Claude Code и claude.ai). Наш путь легален,
потому что авторизация идёт **через Claude Code**, а не в обход него.

### Вариант A2 — API-ключ Anthropic (предсказуемая поштучная оплата)

Если хочешь честный pay-per-token без привязки к подписке:
1. Зайди на **console.anthropic.com** → войди.
2. Слева **API Keys** → кнопка **Create Key** → назови `trendflow-runner` → **Create** → скопируй `sk-ant-…`.
3. Пополни баланс: **Billing** → **Add credits** (например, $20–50 на старт).
4. Ключ положишь в `.env` раннера (шаг в п.5).

**Что выбрать:** берёшь Max ради claude.ai в целом → **A1** (пакет уже включён). Хочешь жёсткий контроль
расходов и простоту → **A2**. Можно начать с A2 (проще предсказать деньги), позже переключить на A1.

## 3. VK-бот для вопросов (пункт B) — пошагово с кнопками

Идея: заводим **сообщество** (группу) VK. Мой раннер шлёт вопрос сообщением от имени сообщества тебе (или
в беседу), ты отвечаешь текстом, раннер читает ответ через Long Poll и продолжает.

### 3.1. Создать сообщество
1. Зайди на **vk.com** под своим аккаунтом.
2. Левое меню → **Сообщества** → кнопка **Создать сообщество** (справа сверху).
3. Тип — выбери **«Бизнес» / «Группа по интересам»** (любой; для бота неважно). Название: `TrendFlow Bot`.
4. **Создать сообщество**.

### 3.2. Разрешить сообщения
1. В сообществе → **Управление** (правое меню) → **Сообщения** → переключатель **Сообщения сообщества: Включены**.
2. Там же **Настройки сообщений** → включи **«Возможности ботов»** (Bot capabilities) → **Включены**.

### 3.3. Получить токен доступа
1. **Управление** → **Работа с API** (в новом интерфейсе может быть внутри **Настройки → Работа с API**).
2. Вкладка **Ключи доступа** → кнопка **Создать ключ**.
3. Отметь галочки прав: **Сообщения сообщества**, **Управление сообществом**.
4. **Создать** → подтверди → скопируй токен (длинная строка). Это `VK_TOKEN`.

### 3.4. Включить Long Poll
1. Та же страница **Работа с API** → вкладка **Long Poll API** → **Включён**.
2. **Версия API** — поставь последнюю из списка (например `5.199`).
3. Ниже **Типы событий** → включи **«Входящие сообщения»** (message_new).

### 3.5. Узнать свой VK id и начать диалог
1. Открой свою страницу VK → в адресе `vk.com/idХХХХХ` — это твой numeric id (если у тебя красивый ник,
   узнай id через любой «vk id» сервис или в **Настройки → Общее**). Это `VK_OWNER_ID`.
2. **Важно:** VK не даёт боту писать первым, пока ТЫ не написал сообществу. Поэтому один раз зайди в
   сообщество → **Сообщение** → напиши боту `start`. После этого раннер сможет тебе отвечать/писать.

Итог этого шага: у тебя есть **`VK_TOKEN`** и **`VK_OWNER_ID`** — отдашь их раннеру в `.env`.

### 3.6. (Опция) MAX вместо VK — с оговоркой
MAX технически умеет ботов (`@MasterBot` → `/create` → ник на `_bot` → токен). **НО:** с августа 2025
создавать/публиковать ботов в MAX разрешено **только российским юрлицам** (физлицо/самозанятый/ИП — нельзя),
плюс до 19.07.2026 требуется миграция API-домена (`platform-api.max.ru` → `platform-api2.max.ru`) и сертификат
Минцифры. Для соло-запуска это лишний геморрой и юридический риск — **рекомендую VK**. Если позже появится
юрлицо, перенос канала на MAX — это замена одного модуля `messenger.py` (2–3 часа работы).

## 4. RunPod (пункт C) — уже почти готово

Ключ и скрипты у нас есть. Что проверить руками на **runpod.io**:
1. Войди на **runpod.io** → **Settings** → **API Keys** — убедись, что ключ из `POD_CONNECTION.md` жив
   (если нет — **Create API Key** → **Read/Write** → скопировать, обновим в файле).
2. **Storage** → убедись, что **Network Volume** (наш `/workspace`, 100 ГБ) на месте — на нём venv, модели, видео.
3. Больше ничего руками не нужно: раннер сам поднимает/останавливает под через API, а `stop_pod.sh`
   (уже починен — ищет RUNNING-поды по API, id меняется при миграции) гарантирует остановку.

## 5. Что соберу Я (код раннера) — на следующей рабочей сессии

Тебе это писать не нужно — это моя часть. Складываю в `automation/runner/`:

| Файл | Что делает |
|------|-----------|
| `agent_runner.py` | главный цикл на **Claude Agent SDK**: берёт компонент → даёт мне задачу с системным промптом и инструментами → я работаю до готовности → штампую → следующий |
| `tools/ask_human.py` | инструмент «спросить человека»: постит вопрос+контекст в VK, ждёт твой ответ (Long Poll), возвращает мне текст |
| `tools/pod_control.py` | инструмент управления подом: перед start/stop/migrate шлёт тебе в VK запрос на подтверждение, на «да» — вызывает RunPod API |
| `tools/web_search.py` | поиск в интернете (первый источник ответов на вопросы — до эскалации тебе) |
| `budget_guard.py` | считает потраченные токены/деньги, читает rate-limit ошибки (429), при достижении дневного капа или лимита — ставит на паузу, гасит под, пишет тебе |
| `queue.py` | очередь = наш `COMPONENT_VALIDATION_CHECKLIST.md` + файл-замок `automation/runner/claim.json` (какой компонент сейчас в работе) |
| `messenger.py` | тонкий модуль VK (send/poll). Замена на MAX = только этот файл |
| `.env` | сюда ты впишешь `ANTHROPIC_API_KEY` **или** отметку «Max/ClaudeCode», `VK_TOKEN`, `VK_OWNER_ID`, `RUNPOD_API_KEY` |
| `runner.service` | автозапуск (systemd) — чтобы цикл жил после перезагрузки ПК |

Логика вопросов (зашита в системный промпт агента):
1. Уперся в неоднозначность → **сначала `web_search`**.
2. Не решается интернетом (нужны веса/доступы/твоё продуктовое решение) → **`ask_human` в VK** + жду.
3. Нужно поднять/погасить/мигрировать под → **`pod_control`** → запрос подтверждения в VK → жду «да».

## 6. Порядок запуска (первый прогон) — по шагам

Когда п.2–4 сделаны, а я собрал код (п.5):

1. Клонируй/обнови репо на ПК (уже есть: `/home/ilya/Рабочий стол/TrendFlowML`).
2. Установи зависимости раннера:
   ```bash
   cd ~/Рабочий\ стол/TrendFlowML/automation/runner
   python3 -m venv .venv && source .venv/bin/activate
   pip install -r requirements.txt        # anthropic / claude-agent-sdk, vk requests, и т.д.
   ```
3. Заполни `.env` (токены из п.2–4).
4. **Пробный ручной прогон (без автозапуска):**
   ```bash
   python agent_runner.py --once --dry-pod
   ```
   `--once` = один компонент и стоп; `--dry-pod` = под не трогать (проверить, что VK-вопросы доходят).
5. Напиши боту в VK `start`, дождись тестового вопроса от меня, ответь — убедись, что цикл читает ответ.
6. **Боевой одиночный прогон:** `python agent_runner.py --once` (я подниму под через подтверждение в VK,
   отработаю компонент, погашу под, пришлю отчёт).
7. **Непрерывный режим:**
   ```bash
   sudo cp runner.service /etc/systemd/system/ && sudo systemctl enable --now runner
   journalctl -u runner -f      # смотреть лог живьём
   ```

## 8. Лимиты и деньги — как не улететь (обязательно)

`budget_guard.py` даёт три предохранителя:
1. **Дневной кап расхода** (например `$10/день`) — считаю токены×цену (API) или слежу за кредит-пулом (Max).
   Достиг → пауза до следующего дня, под погашен, уведомление в VK.
2. **Обработка лимитов модели** — ловлю `429` + заголовки `anthropic-ratelimit-*`; на Max это 5-часовое окно +
   недельный кап (точных токен-квот Anthropic не публикует) → жду сброса, не долблю в стену.
3. **Авто-стоп пода** — под гасится: (а) после каждого компонента, (б) при любой паузе, (в) по «сторожу
   бездействия» (если 20 мин нет активности SSH). Это добивает твою боль «деньги капали» — деньги за GPU
   не текут в простое.

Все расходы пишу в `automation/runner/spend_log.csv` (дата, компонент, токены, $GPU, $модель).

## 10. Риски и честные оговорки

- **Деньги.** Автономный агент + GPU может незаметно тратить. Поэтому дневной кап и авто-стоп пода — не опция.
  Начни с малого капа ($5–10/день) и подними, когда убедишься.
- **Auth/ToS.** Легальные пути — Agent SDK **через Claude Code** (Max, кредит-пул→API) или API-ключ. Не
  подставляй OAuth-токен Max в сторонние сервисы.
- **MAX.** Для физлица публикация ботов закрыта (нужно юрлицо) + миграция доменов/сертификат — поэтому VK.
- **Надёжность одиночного ПК.** Если ПК уснёт/выключится — цикл встанет. Для 24/7 позже можно вынести раннер
  на дешёвый VPS, но для старта хватит ПК (проверь «не засыпать» в настройках питания).
- **Качество без надзора.** Я штампую по 4 осям (корректность/стабильность-golden/различимость/model-fit) и
  пишу отчёт по каждому компоненту — ты постфактум можешь ревьюить отчёты, не сидя онлайн.
