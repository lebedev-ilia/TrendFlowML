## Что произойдёт в ночном режиме

1. **Opus выбирает компоненты сам** из очереди (random или по приоритету)
2. **Каждый компонент обрабатывается полностью** (логика → оптимизация → инфра → тесты)
3. **При подходе к лимиту (90%)**:
   - Opus видит warning ⚠️
   - Завершает текущий шаг
   - Фиксирует состояние
   - Гасит поды
4. **Opus ждёт сброса 5ч-окна** (автоматически, без вашего вмешательства)
5. **После сброса** продолжает со следующего компонента
6. **Повторяет** N раз (пока не выбьёт лимит или не закончится очередь)

---

## Расчёт бюджета

Для эффективного использования лимитов Claude:

| Сценарий | 5ч окно | Фактический лимит |
|---|---|---|
| Standard (90% работа) | Используется весь | ~90% на работу + 7-10% на сохранение |
| Ultra | Используется весь | До 97% (жёсткий стоп) |
| Ночной режим | 8 часов ожидания | Переключение между 5ч окнами |

При `/sas 5` Opus обработает примерно:
- 5 компонентов (если легкие)
- 2-3 компонента (если тяжелые: модели, ML)
- Время в зависимости от сложности (15-60 мин на компонент)
