## Технические детали

### Health check endpoint

Модуль `fetcher/health.py` предоставляет:

- **`check_database_health()`**: Проверяет доступность PostgreSQL через простой SQL запрос (`SELECT 1`)
- **`check_redis_health()`**: Проверяет доступность Redis через `ping()` (опционально, если redis не установлен — возвращает `not_configured`)
- **`check_storage_health()`**: Проверяет доступность S3/MinIO через тестовую операцию `object_exists()`
- **`get_uptime_seconds()`**: Возвращает время работы сервиса в секундах
- **`set_startup_time()`**: Устанавливает время запуска (вызывается при импорте `api.py`)

Endpoint `GET /health`:

- **Статусы**:
  - `healthy`: Все критичные сервисы (database, storage) работают
  - `degraded`: API работает, но некритичные сервисы (redis) недоступны
  - `unhealthy`: Критичные сервисы недоступны (возвращается 503)

- **Response структура**:
  ```json
  {
    "status": "healthy",
    "api": "healthy",
    "version": "0.1.0",
    "uptime_seconds": 3600.0,
    "dependencies": {
      "database": {"status": "healthy"},
      "redis": {"status": "healthy"},
      "storage": {"status": "healthy", "type": "s3"}
    },
    "metrics": {},
    "timestamp": "2026-03-05T12:00:00Z"
  }
  ```

- **Обработка ошибок**: При любой неожиданной ошибке возвращается 503 с статусом `unhealthy`

### Monitoring alerts

Документ `MONITORING_ALERTS.md` описывает:

1. **Критичные алерты (5)**:
   - High Failure Rate (`> 10 failures/min`)
   - Circuit Breaker Tripped (`> 5 trips/min`)
   - Proxy Failure Rate High (`> 80%`)
   - Database Unavailable
   - Storage Unavailable

2. **Предупреждающие алерты (5)**:
   - Low Cache Hit Ratio (`< 30%`)
   - High YouTube 429 Rate (`> 10/min`)
   - High Download Latency (`P95 > 300s`)
   - Queue Depth Exploding (`> 1000`)
   - Redis Unavailable

3. **Информационные алерты (1)**:
   - High Ingestion Throughput (`> 50/min`)

4. **Конфигурация**:
   - Примеры PromQL запросов для каждого алерта
   - Примеры конфигурации Prometheus Alertmanager
   - Примеры правил Prometheus

## Известные ограничения

- **Health check метрики**: Health check статусы пока не экспортируются как Prometheus метрики. Можно добавить gauge метрики `fetcher_health_database_status`, `fetcher_health_redis_status`, `fetcher_health_storage_status` в будущем.
- **Активные runs**: Метрика активных runs в health check response пока пустая. Будет добавлена после интеграции с очередью (Phase 4).
- **Circuit breaker и proxy метрики**: Некоторые алерты (circuit breaker, proxy failure rate) будут актуальны только после Phase 3 (Security & Privacy).
- **Queue метрики**: Алерт для queue depth будет актуальен только после Phase 4 (Queue & Orchestration).
