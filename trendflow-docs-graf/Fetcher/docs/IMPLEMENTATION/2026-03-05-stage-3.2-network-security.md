## 2. Технические детали

### 2.1. PostgreSQL TLS

**Изменённый файл**: `fetcher/db.py`

**Изменения**:
- Функция `_get_ssl_args()` извлекает SSL параметры из настроек или DSN
- Поддержка `sslmode` через `settings.postgres_ssl_mode` или параметр `?sslmode=` в DSN
- SSL параметры передаются в `create_engine()` через `connect_args`

**Режимы SSL**:
- `disable`, `allow`, `prefer`, `require`, `verify-ca`, `verify-full`

### 2.2. Redis TLS

**Изменённый файл**: `fetcher/rate_limiter.py`

**Изменения**:
- Автоматическое определение SSL из URL (`rediss://` vs `redis://`)
- Поддержка `settings.redis_ssl` для явного включения SSL
- Поддержка `settings.redis_ssl_cert_reqs` для требования сертификата

**Конфигурация**:
```python
redis_ssl: bool = Field(False, ...)
redis_ssl_cert_reqs: str = Field("required", ...)  # none, optional, required
```

### 2.3. S3/MinIO TLS

**Изменённый файл**: `fetcher/storage.py`

**Изменения**:
- Автоматическое определение SSL из endpoint URL (`https://` vs `http://`)
- Поддержка `settings.s3_use_ssl` для явного включения SSL
- Поддержка `settings.s3_verify_ssl` для проверки сертификата

**Конфигурация**:
```python
s3_use_ssl: bool = Field(True, ...)
s3_verify_ssl: bool = Field(True, ...)
```

### 2.4. Proxy Authentication

**Изменённый файл**: `fetcher/config.py`

**Добавлено**:
- `proxy_auth_username` и `proxy_auth_password` для альтернативного способа указания credentials
- Приоритет имеет URL (если credentials указаны и там, и в отдельных переменных)

### 2.5. Security Documentation

**Создан файл**: `docs/SECURITY.md`

**Содержание**:
- Инструкции по настройке TLS для всех компонентов
- Примеры интеграции с Secrets Manager (AWS, Vault, K8s)
- Пример скрипта ротации credentials
- Best practices для production
- Команды для проверки конфигурации

## 3. Технические детали

### 3.1. Fail-Safe Approach

- TLS настройки имеют разумные значения по умолчанию
- Для локальной разработки TLS может быть отключен
- Для production рекомендуется явно включать TLS

### 3.2. Credential Rotation

- Документация описывает различные способы интеграции с Secrets Manager
- Поддержка graceful rotation через переменные окружения
- Примеры для Kubernetes, Vault, AWS Secrets Manager

---

## 4. Тестирование

### 4.1. Ручные проверки

- ✅ PostgreSQL SSL корректно настраивается через DSN или отдельную переменную
- ✅ Redis TLS корректно определяется из URL или настройки
- ✅ S3/MinIO TLS корректно определяется из endpoint URL или настройки
- ✅ Proxy authentication работает через URL и отдельные переменные

### 4.2. Edge cases

- ✅ Обработка случая, когда TLS не настроен (fallback на non-TLS)
- ✅ Обработка некорректных SSL режимов (используется значение по умолчанию)

---

## 5. Известные ограничения

- Credential rotation требует ручной настройки интеграции с Secrets Manager
- Нет автоматической проверки TLS конфигурации при старте (только через документацию)
- Нет поддержки client certificates для более строгой аутентификации
