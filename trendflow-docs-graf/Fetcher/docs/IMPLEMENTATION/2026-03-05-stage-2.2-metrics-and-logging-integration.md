## Технические детали

### Structured logging

Модуль `fetcher/logging.py` предоставляет:

- **`StructuredFormatter`**: JSON formatter, который добавляет обязательные поля:
  - `timestamp` (ISO 8601)
  - `level` (INFO, ERROR, etc.)
  - `message`
  - `logger` (имя logger'а)
  - `run_id`, `stage`, `platform`, `platform_video_id` (если переданы через `extra`)

- **`setup_logging()`**: Настройка логирования с поддержкой форматов "json" и "text"

- **`log_with_context()`**: Helper-функция для логирования с контекстом Fetcher

Пример JSON-лога:

```json
{
  "timestamp": "2026-03-05T12:00:00Z",
  "level": "INFO",
  "message": "Starting metadata worker for run_id=550e8400-...",
  "logger": "fetcher.workers.metadata",
  "run_id": "550e8400-...",
  "stage": "fetch_metadata",
  "platform": "youtube"
}
```

### Интеграция метрик

Все воркеры теперь:

1. **Измеряют latency** через histogram-метрики:
   - `fetcher_metadata_latency_seconds.labels(platform="youtube").observe(elapsed)`
   - `fetcher_download_latency_seconds.labels(platform="youtube").observe(elapsed)`
   - `fetcher_comments_latency_seconds.labels(platform="youtube").observe(elapsed)`

2. **Инкрементируют счётчики** при успешном завершении:
   - `fetcher_videos_downloaded_total.labels(platform="youtube").inc()`

3. **Логируют начало и завершение** с контекстом (`run_id`, `stage`, `platform`)

4. **Логируют ошибки** с полным контекстом и exception info

## Известные ограничения

- **Platform detection**: Сейчас `platform` жёстко задан как `"youtube"`. В будущем нужно определять из `run.source_type` или `video_source.platform`.
- **Error metrics**: Метрики ошибок (`fetcher_videos_failed_total`, `fetcher_youtube_429_total`) пока не интегрированы — это будет сделано в следующих стадиях при добавлении обработки ошибок YouTube API.
- **Cache metrics**: Метрики cache hits/misses (`fetcher_cache_hits_total`, `fetcher_cache_miss_total`) будут добавлены при реализации cache-check логики в video download worker.
