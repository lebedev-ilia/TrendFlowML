# Периодические snapshots для Fetcher

**Дата**: 2026-03-05  
**Стадия**: Phase 5 — ML Pipeline Compatibility  
**Статус**: ✅ Реализовано

## Обзор

Реализована поддержка периодических snapshots для видео с конфигурируемым расписанием. Snapshots создаются автоматически через Celery Beat согласно заданному schedule (например, на 0, 7, 14, 21 день после первого snapshot).

## Требования

Из чеклиста Phase 5:
- Конфигурируемый schedule (0/7/14/21 дней или по таймстемпам)
- Celery Beat задача для периодических snapshots
- Обработка уже существующих видео (backfill snapshots)
- Конфигурация через `snapshot_schedule_days` в settings
- Логика определения видео, требующих нового snapshot

## Реализация

### 1. Функции для периодических snapshots

**Файл**: `fetcher/snapshots.py`

**Новые функции**:

1. **`create_periodic_snapshot()`**:
   - Создаёт snapshot с указанным индексом (1, 2, 3, ...)
   - Проверяет, не существует ли уже snapshot с таким индексом
   - Использует info_dict от yt-dlp для получения текущих метрик

2. **`get_videos_needing_snapshot()`**:
   - Определяет видео, которым нужен новый snapshot
   - Вычисляет дни с первого snapshot (snapshot_index=0)
   - Сравнивает с schedule_days для определения следующего snapshot_index
   - Возвращает список (platform, platform_video_id, next_snapshot_index)

3. **`create_snapshots_for_videos()`**:
   - Создаёт snapshots для списка видео
   - Использует yt-dlp для получения текущих метрик
   - Обрабатывает ошибки и возвращает статистику

**Логика определения следующего snapshot**:
```python
# schedule_days = [0, 7, 14, 21]
# days_since_first = 10
# next_snapshot_index = 2 (14 день)
```

### 2. Celery Beat задача

**Файл**: `fetcher/tasks.py`

**Задача `periodic_snapshots_task`**:
- Запускается автоматически через Celery Beat (ежедневно)
- Получает список видео, которым нужен snapshot
- Создаёт snapshots для этих видео
- Логирует результаты
- Обрабатывает ошибки с retry

**Интеграция в beat_schedule**:
```python
beat_schedule={
    "periodic-snapshots-daily": {
        "task": "fetcher.tasks.periodic_snapshots_task",
        "schedule": 86400.0,  # Каждые 24 часа
    },
}
```

### 3. Конфигурация

**Файл**: `fetcher/config.py`

**Добавленная настройка**:
- `snapshot_schedule_days`: List[int] - Расписание в днях (по умолчанию [0, 7, 14, 21])

**Пример конфигурации**:
```python
snapshot_schedule_days = [0, 7, 14, 21, 30, 60, 90]  # Snapshots на 0, 7, 14, 21, 30, 60, 90 день
```

### 4. Логика работы

**Последовательность**:

1. Celery Beat запускает `periodic_snapshots_task` ежедневно
2. Задача вызывает `get_videos_needing_snapshot()`:
   - Находит все видео с начальным snapshot (snapshot_index=0)
   - Вычисляет дни с первого snapshot
   - Определяет следующий snapshot_index по schedule
   - Проверяет, не существует ли уже snapshot с таким индексом
3. Для каждого видео вызывается `create_snapshots_for_videos()`:
   - Получает текущие метрики через yt-dlp
   - Создаёт snapshot с нужным индексом
4. Результаты логируются

**Пример**:
- Видео создано 10 дней назад (snapshot_index=0)
- schedule_days = [0, 7, 14, 21]
- days_since_first = 10
- Следующий snapshot должен быть на 14 день (snapshot_index=2)
- Если snapshot_index=2 не существует, создаётся новый snapshot

## Использование

### Настройка через переменные окружения

```bash
export FETCHER_SNAPSHOT_SCHEDULE_DAYS="[0,7,14,21,30]"
```

### Настройка в Kubernetes ConfigMap

```yaml
apiVersion: v1
kind: ConfigMap
metadata:
  name: fetcher-config
data:
  snapshot-schedule-days: "[0,7,14,21,30]"
```

### Ручной запуск

```python
from fetcher.snapshots import get_videos_needing_snapshot, create_snapshots_for_videos

# Получить список видео, которым нужен snapshot
videos = get_videos_needing_snapshot(schedule_days=[0, 7, 14, 21], batch_size=100)

# Создать snapshots
results = create_snapshots_for_videos(videos)
```

## Известные ограничения

- **Только YouTube**: Поддержка только для YouTube (можно расширить для других платформ)
- **Batch processing**: Обрабатывается максимум 100 видео за раз (можно настроить)
- **Rate limiting**: Не учитывается rate limiting при создании snapshots (можно добавить)
- **Error handling**: Ошибки при создании snapshot для одного видео не останавливают обработку остальных
