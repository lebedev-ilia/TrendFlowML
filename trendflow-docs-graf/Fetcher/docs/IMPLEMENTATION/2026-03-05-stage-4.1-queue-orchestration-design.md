## Технические детали

### Celery app

Модуль `fetcher/celery_app.py` предоставляет:

- **Celery приложение** с broker и backend на Redis
- **Конфигурация**:
  - JSON serialization
  - UTC timezone
  - Task time limits (1 час для video download)
  - Worker prefetch multiplier = 1 (не prefetch много задач)
  - Worker max tasks per child = 50 (перезапуск worker после N задач)
- **Роутинг задач** по очередям с приоритетами:
  - `fetch.metadata`: priority 9 (high)
  - `fetch.video`: priority 1 (low)
  - `fetch.comments`: priority 5 (medium)
  - `fetch.finalize`: priority 9 (high)

### Celery задачи

Модуль `fetcher/tasks.py` содержит:

- **4 Celery задачи**, каждая обёрнута вокруг существующего воркера
- **Retry политика**:
  - Metadata: 5 retries, exponential backoff (60s * 2^retries)
  - Video: 3 retries, exponential backoff (120s * 2^retries)
  - Comments: 3 retries, exponential backoff (60s * 2^retries)
  - Finalize: 3 retries, exponential backoff (30s * 2^retries)
- **Логирование** с контекстом (run_id, stage, platform) для каждой задачи

### Оркестратор

Модуль `fetcher/orchestrator.py` реализует:

- **`fetch_video(run_id)`** — главная функция оркестратора:
  1. Нормализация source (URL → platform_video_id) через yt-dlp
  2. Проверка глобального кеша по (platform, platform_video_id)
  3. Если cache hit — сразу `finalize_task.delay(run_id)`
  4. Если cache miss — постановка задач: `fetch_metadata_task.delay()`, `download_video_task.delay()`, `fetch_comments_task.delay()`
- **State machine интеграция**: обновление статуса run (NORMALIZING_SOURCE → CHECKING_CACHE → FETCHING_METADATA/FINALIZING)
- **Обработка ошибок**: при ошибке статус run обновляется на FAILED

### Утилиты

Модуль `fetcher/utils.py` содержит:

- **`all_artifacts_ready(run_id)`** — проверка готовности всех обязательных артефактов:
  - Получает video_id через run → video_source → video
  - Проверяет наличие артефактов: `video_file`, `metadata_file`, `comments_file`
  - Все артефакты должны иметь статус `COMPLETED`

## Известные ограничения

- ✅ **Retryable vs non-retryable ошибки**: ✅ **РЕАЛИЗОВАНО** (см. `2026-03-05-stage-qa-error-classification.md`)
- **Fan-in для finalize**: Используется polling для проверки готовности артефактов. Альтернатива — Celery `chord` для более элегантного fan-in (будет реализовано позже).
- ✅ **Backpressure control**: ✅ **РЕАЛИЗОВАНО** (см. `2026-03-05-stage-4.3-backpressure-control.md`)
- ✅ **Distributed locks**: ✅ **РЕАЛИЗОВАНО** (интегрированы в `download_video_task` через `acquire_video_lock`)
- **Метрики очередей**: Пока не добавлены. Нужно добавить метрики `fetcher_queue_length`, `fetcher_active_tasks` в `fetcher/metrics.py`.
