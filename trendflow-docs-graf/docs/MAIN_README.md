1️⃣ Цель системы

Ты строишь мульти-модальную систему для предсказания популярности видео (просмотры/лайки через месяц) на базе:

Видео (кадры, объекты, эмоции, сцены)

Аудио (музыка, речь, шум, RMS, спектральные признаки)

Текст (транскрипции, метаданные, заголовки, описание)

Временные ряды (лайки, просмотры, комментарии по времени)

2️⃣ Архитектура экстракторов

Каждый экстрактор:

Получает на вход последовательность сегментов/кадров с определённым шагом (step) — это pipeline подаёт уже сегменты, экстрактор сам шаг не знает.

Обрабатывает сегменты (CNN, MLP, аудио-эмбеддинги, NLP)

Выдаёт [N, M] — N = количество сегментов, M = размер фичей

Важно: экстрактор не думает о длине видео, pooling, локальных окнах или ресэмплинге — этим занимается пайплайн.

3️⃣ Ресэмплинг и базовая временная сетка (BASE_DT)

Все выходы экстракторов приводятся к единой базовой сетке

Интерполяция или агрегация (mean, max, std) для согласования разных частот экстракторов

Позволяет объединять видео, аудио и текстовые данные по времени

Коррекция:

Короткие видео → уменьшаем BASE_DT

Длинные видео → увеличиваем BASE_DT

4️⃣ Локальные токены (WINDOW_SIZE)

Несколько сегментов объединяются в один локальный токен

Служит для:

уменьшения длины последовательности

стабилизации признаков (агрегаты: mean/std/max)

Коррекция:

Короткие видео → меньшее окно → больше локальных токенов

Длинные видео → большее окно → меньше локальных токенов

5️⃣ Learnable Pooling

Превращает M локальных токенов в фиксированное число K summary-токенов

Позволяет трансформеру работать с видео любой длины

Для коротких видео важно использовать маску, чтобы пустые summary-токены не портили внимание

6️⃣ Объединение модальностей

После pooling все модальности приводятся к единой форме [K, D]

Fusion-transformer: взаимодействие video/audio/text/metadata/temporal

Итоговая регрессия → предсказание просмотров/лайков

7️⃣ Важные параметры для корректировки
Параметр	Короткие видео	Длинные видео
BASE_DT	уменьшить	увеличить
WINDOW_SIZE	уменьшить	увеличить
Кол-во summary токенов K	фиксированное, маска	фиксированное, K < M
Агрегаты (mean/std/max)	добавить доп. статистики	стандартные
Маскирование	обязательно	не критично
8️⃣ Основная идея

Экстракторы → [N, M] без забот о длине видео

Ресэмплинг → согласуем все модальности на одной временной сетке

Локальные токены + pooling → сжимаем последовательность, стабилизируем признаки

Fusion-transformer → объединяем все модальности

Регрессия → предсказание целевой метрики

Благодаря этому пайплайн работает с видео любой длины, разной частоты экстракторов и разными модальностями, а трансформер получает фиксированное и информативное представление каждого видео.
