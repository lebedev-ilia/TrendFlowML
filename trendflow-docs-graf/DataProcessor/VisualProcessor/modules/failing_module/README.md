# `failing_module`

Utility модуль для демонстрации обработки опциональных компонентов, которые могут завершаться с ошибкой без остановки всего пайплайна.

## Назначение

Модуль предназначен для тестирования и демонстрации механизма обработки опциональных компонентов в VisualProcessor. Модуль всегда завершается с ошибкой (exit code 2), что позволяет проверить, что система корректно обрабатывает опциональные компоненты, которые могут завершиться неудачно.

## Поведение

- **Всегда завершается с ошибкой**: модуль всегда возвращает exit code 2
- **Принимает стандартные аргументы**: модуль принимает стандартные CLI аргументы VisualProcessor (`--frames-dir`, `--rs-path`) для совместимости с оркестратором
- **Выводит сообщение об ошибке**: перед завершением выводит сообщение в stderr о намеренном завершении с ошибкой

## CLI

```bash
python DataProcessor/VisualProcessor/modules/failing_module/main.py \
  --frames-dir <frames_dir> \
  --rs-path <run_rs_path>
```

## Использование

Модуль используется для:
- Демонстрации обработки опциональных компонентов (PR-4 evidence)
- Тестирования механизма обработки ошибок в VisualProcessor
- Проверки, что опциональные компоненты могут завершаться с ошибкой без остановки всего пайплайна

## Выходной код

Модуль всегда возвращает exit code **2** (FileNotFoundError code в VisualProcessor).

## Примечания

- Модуль не создаёт никаких артефактов
- Модуль не выполняет никакой реальной обработки
- Модуль предназначен только для тестирования и демонстрации
