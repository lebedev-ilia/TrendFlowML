# Главный индекс модуля Storage

Этот документ служит единой точкой входа для навигации по модулю Storage — абстракции для работы с хранилищем данных (файловая система или S3/MinIO). Каждый раздел содержит краткое описание компонентов и их назначение.

**Главный индекс DataProcessor**: [../docs/MAIN_INDEX.md](../docs/MAIN_INDEX.md) · **Vault root**: [../../docs/MAIN_INDEX.md](../../docs/MAIN_INDEX.md)

## Базовые интерфейсы и типы

### base.py
**Краткое описание**: Определяет базовые интерфейсы и типы для модуля Storage. Содержит протокол `Storage` (Protocol) с минимальным набором методов: `exists()` для проверки существования ключа, `list()` для листинга объектов по префиксу, `read_bytes()` для чтения данных, `write_bytes()` и `atomic_write_bytes()` для записи данных. Определяет исключения `StorageError` и `NotFoundError` для обработки ошибок хранилища. Содержит dataclass `ObjectInfo` для метаданных объектов (key, size_bytes, etag). Ключи являются относительными логическими путями (например, "trendflowml/result_store/youtube/<video>/<run>/..."), которые маппятся на файловые пути или S3 object keys в зависимости от реализации. Используется как контракт для всех реализаций хранилища.

**Расположение**: `storage/base.py`

## Реализации хранилища

### fs.py
**Краткое описание**: Реализация хранилища на основе файловой системы. Класс `FileSystemStorage` реализует протокол `Storage` для работы с локальной файловой системой. Методы: `__init__()` принимает root_dir для базовой директории, `exists()` проверяет существование файла, `list()` рекурсивно обходит директорию и возвращает ObjectInfo для всех файлов, `read_bytes()` читает файл с обработкой NotFoundError, `write_bytes()` создает директории при необходимости и записывает данные, `atomic_write_bytes()` выполняет атомарную запись через временный файл и os.replace(). Используется для локальной разработки и тестирования, позволяет работать без S3/MinIO инфраструктуры. Поддерживает относительные ключи с автоматической нормализацией путей.

**Расположение**: `storage/fs.py`

### s3.py
**Краткое описание**: Реализация хранилища на основе S3/MinIO. Класс `S3Storage` реализует протокол `Storage` для работы с S3-совместимыми хранилищами (AWS S3, MinIO и др.). Использует boto3 для взаимодействия с S3 API. Методы: `__init__()` принимает endpoint_url, bucket, region для настройки клиента, `exists()` использует head_object для проверки существования, `list()` использует paginator для листинга объектов с префиксом, `read_bytes()` читает объект через get_object, `write_bytes()` записывает объект через put_object с поддержкой content_type, `atomic_write_bytes()` использует PUT overwrite как атомарную операцию (для большинства use cases достаточно). Обрабатывает ClientError от boto3 и конвертирует 404 в NotFoundError. Поддерживает опциональный S3_PREFIX для организации ключей. Используется в production для хранения result_store, state, frames_dir в объектном хранилище.

**Расположение**: `storage/s3.py`

## Конфигурация и настройки

### settings.py
**Краткое описание**: Модуль для загрузки настроек хранилища из переменных окружения. Содержит dataclass `StorageSettings` с полями: backend ("fs" или "s3"), fs_root (корневая директория для FS), s3_endpoint, s3_bucket, s3_prefix, aws_region (настройки для S3). Функция `load_storage_settings()` читает переменные окружения: `TREND_STORAGE_BACKEND` (по умолчанию "s3"), `TREND_FS_ROOT` (по умолчанию "_runs"), `S3_ENDPOINT` (по умолчанию "http://localhost:9000"), `S3_BUCKET` (по умолчанию "trendflow"), `S3_PREFIX` (по умолчанию "trendflowml"), `AWS_DEFAULT_REGION` (по умолчанию "us-east-1"). Поддерживает стандартные boto3 переменные (AWS_ACCESS_KEY_ID, AWS_SECRET_ACCESS_KEY), которые обрабатываются boto3 автоматически. Валидирует backend на допустимые значения ("fs" или "s3"). Используется для инициализации хранилища в зависимости от окружения.

**Расположение**: `storage/settings.py`

### paths.py
**Краткое описание**: Модуль для работы с канонической структурой ключей в хранилище. Содержит dataclass `KeyLayout` для управления префиксами и путями. Определяет каноническую структуру ключей под единым настраиваемым префиксом (S3_PREFIX): `result_store/...`, `state/...`, `frames_dir/...`. Методы: `result_store_prefix()` возвращает префикс для result_store, `state_prefix()` для state, `frames_dir_prefix()` для frames_dir, `result_store_run_prefix()` генерирует полный путь для run'а в result_store (platform_id/video_id/run_id), `state_run_prefix()` для state, `frames_dir_run_prefix()` для frames_dir. Функция `_join()` нормализует части путей, удаляя лишние слеши и пустые части. Используется для генерации канонических ключей при работе с хранилищем, обеспечивает единообразие структуры данных.

**Расположение**: `storage/paths.py`

## Публичный API

### __init__.py
**Краткое описание**: Публичный API модуля `storage`. Экспортирует основные классы и функции: `Storage` (протокол), `StorageError`, `NotFoundError`, `FileSystemStorage`, `StorageSettings`, `load_storage_settings`, `S3Storage` (опционально, если доступен boto3). Обрабатывает отсутствие boto3 gracefully (S3Storage может быть None в окружениях без boto3). Используется для импорта компонентов хранилища в других модулях системы (DataProcessor, backend, workers). Предоставляет единую точку входа для работы с хранилищем независимо от выбранного бэкенда.

**Расположение**: `storage/__init__.py`
