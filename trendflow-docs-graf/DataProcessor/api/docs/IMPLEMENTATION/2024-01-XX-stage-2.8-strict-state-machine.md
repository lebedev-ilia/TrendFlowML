## Выполненные задачи

### ✅ Создать enum `RunStatus`

Enum `RunStatus` уже существовал в `api/schemas/state.py`:
- `PENDING = "pending"`
- `QUEUED = "queued"`
- `RUNNING = "running"`
- `RECOVERING = "recovering"`
- `SUCCESS = "success"`
- `ERROR = "error"`
- `CANCELLED = "cancelled"`

### ✅ Реализовать таблицу переходов

Создан модуль `api/services/state_machine.py` с таблицей переходов:

```python
ALLOWED_TRANSITIONS: Dict[RunStatus, List[RunStatus]] = {
    RunStatus.PENDING: [RunStatus.QUEUED, RunStatus.CANCELLED],
    RunStatus.QUEUED: [RunStatus.RUNNING, RunStatus.CANCELLED],
    RunStatus.RUNNING: [
        RunStatus.SUCCESS,
        RunStatus.ERROR,
        RunStatus.RECOVERING,
        RunStatus.CANCELLED
    ],
    RunStatus.RECOVERING: [RunStatus.RUNNING, RunStatus.ERROR],
    RunStatus.SUCCESS: [],  # Финальное состояние
    RunStatus.ERROR: [],    # Финальное состояние
    RunStatus.CANCELLED: [] # Финальное состояние
}
```

### ✅ Реализовать функцию `can_transition()`

Функция `can_transition()` проверяет, разрешен ли переход между статусами:

```python
def can_transition(from_status: RunStatus, to_status: RunStatus) -> bool:
    allowed = ALLOWED_TRANSITIONS.get(from_status, [])
    return to_status in allowed
```

### ✅ Реализовать `validate_transition()` с валидацией

Функция `validate_transition()` валидирует переход и вызывает `ValueError` если переход не разрешен:

```python
def validate_transition(
    from_status: Optional[RunStatus],
    to_status: RunStatus,
    run_id: Optional[str] = None
) -> None:
    # Валидация перехода
    if not can_transition(from_status, to_status):
        raise ValueError(...)
```

### Архитектурные решения

1. **Таблица переходов**
   - Определяет разрешенные переходы между статусами
   - Финальные статусы (SUCCESS, ERROR, CANCELLED) не могут перейти в другие статусы
   - RECOVERING может перейти только в RUNNING или ERROR

2. **Валидация переходов**
   - Автоматическая валидация в `TaskManager.update_run_status()`
   - Опциональная валидация в `save_run_state()` (по умолчанию включена)
   - Вызывает `ValueError` при недопустимом переходе

3. **Парсинг статусов**
   - Функция `parse_status()` для конвертации строк в enum
   - Поддерживает как значения enum, так и имена
   - Case-insensitive парсинг

4. **Обратная совместимость**
   - Старые строковые статусы парсятся через `parse_status()`
   - Storage продолжает использовать строки (для совместимости)
   - Redis state cache использует строки (JSON serialization)

## Использование

### Автоматическое использование

State machine работает автоматически:

```python
# TaskManager автоматически валидирует переходы
task_manager.update_run_status(run_id, RunStatus.RUNNING)
# ValueError если переход не разрешен

# Redis schema валидирует переходы при сохранении
await save_run_state(run_id, {"status": RunStatus.SUCCESS.value})
# ValueError если переход не разрешен
```

### Ручное использование

Для ручной работы с state machine:

```python
from api.schemas.state import RunStatus
from api.services.state_machine import (
    can_transition,
    validate_transition,
    parse_status
)

# Проверить разрешен ли переход
if can_transition(RunStatus.RUNNING, RunStatus.SUCCESS):
    # Переход разрешен
    pass

# Валидировать переход
try:
    validate_transition(RunStatus.RUNNING, RunStatus.SUCCESS, run_id="...")
except ValueError as e:
    # Переход не разрешен
    pass

# Парсить строку в enum
status = parse_status("running")  # RunStatus.RUNNING
```

## Известные проблемы и ограничения

1. **Парсинг старых статусов**
   - Старые строковые статусы парсятся через `parse_status()`
   - Если статус не распознан, возвращается `ValueError`
   - TODO: Добавить fallback на "unknown" статус

2. **Валидация в Storage**
   - Storage продолжает использовать строки
   - Валидация выполняется только при чтении из Storage
   - TODO: Добавить валидацию при записи в Storage

3. **Миграция существующих данных**
   - Существующие run'ы могут иметь старые строковые статусы
   - Парсинг выполняется при чтении
   - TODO: Добавить миграцию для существующих данных
