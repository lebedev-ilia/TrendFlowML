## Технические детали

### Event Loop Management

**Проблема**: Celery задачи выполняются в синхронном контексте, но нужно запускать async функции.

**Решение**:
```python
try:
    loop = asyncio.get_event_loop()
except RuntimeError:
    loop = asyncio.new_event_loop()
    asyncio.set_event_loop(loop)
```

**Закрытие**:
```python
finally:
    if loop and loop != asyncio.get_event_loop():
        loop.close()
```

### Progress Callback

**Типы событий**:
- `progress` - обновление прогресса (overall, current_component, components)
- `stage` - смена стадии (segmenter, audio, text, visual)
- `component_start` - начало компонента
- `component_complete` - завершение компонента (success, error)

**Обработка**:
```python
def progress_callback(event: Dict[str, Any]) -> None:
    event_type = event.get("type")
    event_data = event.get("data", {})
    
    if event_type == "progress":
        # Отправить событие прогресса
        _publish(run_id, {
            "type": "run.progress",
            "payload": {"progress": ...}
        })
    elif event_type == "stage":
        _emit_stage(run_id, event_data.get("stage"))
    ...
```
