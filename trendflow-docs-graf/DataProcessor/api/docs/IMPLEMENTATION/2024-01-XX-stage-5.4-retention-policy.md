# Этап 5.4: Retention Policy

**Дата**: 2024-01-XX  
**Этап**: 5.4 из [API_DEVELOPMENT_CHECKLIST.md](../../../docs/API_DEVELOPMENT_CHECKLIST.md)  
**Статус**: ✅ Завершено

## Обзор

Реализована политика хранения данных (Retention Policy) для автоматической очистки старых данных:
- Удаление Redis state старше 1 дня
- Удаление storage старше 7 дней

## Выполненные задачи

### ✅ Реализовать cron job для очистки

Создан сервис `api/services/retention.py` с функциями:

1. **`cleanup_redis_state()`** - удаление Redis state старше 1 дня
   - Проверка всех ключей `run:state:*`
   - Удаление ключей без TTL с `updated_at` < cutoff
   - Обработка ошибок с логированием

2. **`cleanup_storage()`** - удаление storage старше 7 дней
   - Обход всех run'ов в `result_store`
   - Проверка `finished_at` в `manifest.json`
   - Удаление старых run'ов через `delete_run_storage()`

3. **`run_retention_cleanup()`** - основная функция для запуска очистки
   - Выполняет очистку Redis и Storage
   - Возвращает детальные результаты

4. **`delete_run_storage()`** - удаление run'ов из storage
   - Поддержка FileSystemStorage (через `shutil.rmtree`)
   - Поддержка S3Storage (через `delete_objects` батчами)

### ✅ Настроить запуск cron job

1. **Создан скрипт** `api/retention_cleanup.py`:
   - Точка входа для запуска очистки
   - Инициализация Redis и Storage
   - Логирование результатов

2. **Добавлен endpoint** `POST /api/v1/admin/retention/cleanup`:
   - Ручной запуск очистки через API
   - Требует API Key аутентификации
   - Возвращает детальные результаты

3. **Настроен docker-compose**:
   - Сервис `dataprocessor-retention-cleanup` с cron
   - Ежедневный запуск в 2:00 UTC
   - Альтернативные варианты (Kubernetes CronJob, systemd timer)

### ✅ Тесты для retention policy

Созданы unit тесты в `api/tests/unit/test_retention.py`:
- Тесты для `cleanup_redis_state()` (нормальное использование, старые ключи, ошибки)
- Тесты для `cleanup_storage()` (старые run'ы, недавние run'ы)
- Тесты для `delete_run_storage()` (FileSystemStorage, S3Storage)
- Тесты для `run_retention_cleanup()` (успешная очистка, ошибки)

## Технические детали

### Константы retention

```python
REDIS_STATE_RETENTION = 24 * 3600  # 1 день
STORAGE_RETENTION = 7 * 24 * 3600  # 7 дней
```

### Удаление из S3

Используется батчевое удаление (до 1000 объектов за раз):
```python
storage._client.delete_objects(
    Bucket=storage.bucket,
    Delete={"Objects": delete_keys}
)
```
