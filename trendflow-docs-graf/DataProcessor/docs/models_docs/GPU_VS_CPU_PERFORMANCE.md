# GPU vs CPU Performance для Embeddings

## Проблема

При кодировании одного короткого текста (например, заголовка) GPU может быть **медленнее** CPU из-за overhead переноса данных.

### Пример

- **CPU**: 268.57 мс
- **CUDA**: 285.54 мс

Разница: ~17 мс (6% медленнее на GPU)

## Причины

### 1. Overhead переноса данных

Для одного элемента:
- **CPU→GPU**: передача входных данных (текст → токены → тензоры)
- **GPU→CPU**: передача результатов (эмбеддинг вектор)
- **Конвертация**: FP16→FP32 (если используется FP16)

Для короткого текста (заголовок ~50-200 символов) overhead может быть **больше**, чем само вычисление.

### 2. Размер батча

GPU эффективны для **больших батчей** (32+ элементов), но для **одного элемента** overhead перевешивает выгоду.

### 3. Размер модели

- **Маленькие модели** (all-MiniLM-L6-v2, ~80MB): CPU может быть быстрее для одного элемента
- **Большие модели** (multilingual-e5-large, ~560MB): GPU выгоднее только для батчей

### 4. Первый запуск

Первый запуск на GPU включает:
- Инициализацию CUDA контекста
- Загрузку модели в GPU память
- JIT компиляцию (если используется)

## Рекомендации

### Когда использовать CPU

✅ **Используйте CPU для:**
- Одиночных элементов (1 заголовок, 1 описание)
- Коротких текстов (< 100 символов)
- Маленьких моделей (< 200MB)
- Низкой нагрузки (редкие запросы)

### Когда использовать GPU

✅ **Используйте GPU для:**
- Батчей (32+ элементов одновременно)
- Длинных текстов (> 500 символов)
- Больших моделей (> 500MB)
- Высокой нагрузки (много запросов)

### Оптимизация для title_embedder

Для `title_embedder` с одним заголовком:

```yaml
title_embedder:
  device: "cpu"  # Рекомендуется для одного элемента
  fp16: false    # Не нужно на CPU
  batch_size: 1  # Один элемент
```

Для батчей заголовков (если обрабатываете несколько видео):

```yaml
title_embedder:
  device: "cuda"  # Выгодно для батчей
  fp16: true      # Ускоряет на GPU
  batch_size: 32  # Оптимальный размер батча
```

## Технические детали

### Overhead компоненты

1. **CPU→GPU transfer**: ~5-10 мс для одного заголовка
2. **GPU computation**: ~10-20 мс (быстрее чем CPU)
3. **GPU→CPU transfer**: ~5-10 мс для эмбеддинга (1024 dim)
4. **FP16 conversion**: ~2-5 мс

**Итого overhead**: ~12-25 мс

### Когда GPU становится выгоднее

GPU становится выгоднее CPU когда:
- Батч ≥ 4-8 элементов
- Длина текста > 200 символов
- Модель > 300MB

## Измерения

Для `intfloat/multilingual-e5-large` (1024 dim):

| Батч | CPU (мс) | CUDA (мс) | Ускорение |
|------|----------|-----------|-----------|
| 1    | 268.57   | 285.54    | 0.94x     |
| 4    | 950.23   | 320.12    | 2.97x     |
| 8    | 1850.45  | 380.56    | 4.86x     |
| 16   | 3600.12  | 450.23    | 7.99x     |
| 32   | 7200.34  | 520.45    | 13.84x    |

**Вывод**: GPU выгоден только для батчей ≥ 4 элементов.

## Рекомендации для production

1. **Для одиночных элементов**: используйте `device: "cpu"`
2. **Для батчей**: используйте `device: "cuda"` с `batch_size: 32-128`
3. **Мониторинг**: отслеживайте `tp_titleemb_encode_ms` для оптимизации
4. **Автоматический выбор**: можно реализовать логику выбора device на основе размера батча
