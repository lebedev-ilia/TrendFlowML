# Сравнение моделей эмбеддингов для TextProcessor

## Текущие модели в проекте

### 1. `sentence-transformers/all-MiniLM-L6-v2` (текущая по умолчанию)

**Характеристики:**
- **Размерность**: 384
- **Размер модели**: ~90MB
- **Скорость**: ⚡⚡⚡ (очень быстрая)
- **Качество**: ⭐⭐⭐ (хорошее для базовых задач)
- **Многоязычность**: ⚠️ (ограниченная поддержка русского)

**Плюсы:**
- Очень быстрая обработка
- Низкое потребление памяти
- Хорошо работает для английского текста
- Подходит для baseline и быстрого прототипирования

**Минусы:**
- Ограниченная поддержка русского языка
- Меньшая размерность → меньше семантической информации
- Хуже работает на длинных текстах

**Использование в проекте:**
- По умолчанию для всех embedding extractors (`title_embedder`, `description_embedder`, и т.д.)

---

### 2. `intfloat/multilingual-e5-large` (уже используется)

**Характеристики:**
- **Размерность**: 1024
- **Размер модели**: ~560MB
- **Скорость**: ⚡⚡ (быстрая, но медленнее MiniLM)
- **Качество**: ⭐⭐⭐⭐⭐ (отличное)
- **Многоязычность**: ✅✅✅ (отличная поддержка 100+ языков, включая русский)

**Плюсы:**
- Отличная поддержка русского языка
- Высокое качество эмбеддингов
- Большая размерность → больше семантической информации
- Лучше работает на длинных текстах
- Оптимизирована для retrieval и semantic search

**Минусы:**
- Больше размер модели и памяти
- Медленнее обработка (но все еще быстро)
- Требует больше VRAM на GPU

**Использование в проекте:**
- `semantics_topics_keyphrases` extractor

---

## Рекомендации по выбору модели

### Для production с русским текстом:

**Рекомендуется: `intfloat/multilingual-e5-large`**

**Почему:**
1. ✅ Отличная поддержка русского языка
2. ✅ Высокое качество эмбеддингов для semantic search и similarity
3. ✅ Уже используется в проекте для важных задач
4. ✅ Лучше работает на длинных текстах (description, transcript)

**Когда использовать:**
- `title_embedder` - для лучшего качества
- `description_embedder` - для длинных текстов
- `transcript_chunk_embedder` - для транскриптов
- `comments_embedder` - для комментариев на русском

### Для быстрых baseline тестов:

**Рекомендуется: `sentence-transformers/all-MiniLM-L6-v2`**

**Когда использовать:**
- Быстрое прототипирование
- Ограниченные ресурсы (CPU-only)
- Английский текст преимущественно

## Альтернативные модели (для рассмотрения)

### 3. `intfloat/multilingual-e5-base`

**Характеристики:**
- **Размерность**: 768
- **Размер модели**: ~280MB
- **Скорость**: ⚡⚡⚡ (быстрее large)
- **Качество**: ⭐⭐⭐⭐ (хорошее)
- **Многоязычность**: ✅✅✅ (отличная)

**Компромисс между MiniLM и e5-large:**
- Быстрее чем e5-large, но качественнее чем MiniLM
- Хорошая поддержка русского
- Средний размер модели

### 4. `sentence-transformers/paraphrase-multilingual-mpnet-base-v2`

**Характеристики:**
- **Размерность**: 768
- **Размер модели**: ~420MB
- **Скорость**: ⚡⚡
- **Качество**: ⭐⭐⭐⭐
- **Многоязычность**: ✅✅ (хорошая поддержка 50+ языков)

**Особенности:**
- Оптимизирована для paraphrase detection
- Хорошо работает для semantic similarity

### 5. `sentence-transformers/LaBSE`

**Характеристики:**
- **Размерность**: 768
- **Размер модели**: ~470MB
- **Скорость**: ⚡⚡
- **Качество**: ⭐⭐⭐⭐
- **Многоязычность**: ✅✅✅ (109 языков)

**Особенности:**
- Language-agnostic BERT Sentence Embedding
- Хорошо работает для cross-lingual tasks

## Рекомендации для вашего проекта

### Текущая ситуация:
- ✅ `all-MiniLM-L6-v2` - уже сохранена в `text/embeddings/all-MiniLM-L6-v2`
- ✅ `multilingual-e5-large` - уже зарегистрирована в spec_catalog

### Рекомендуемые действия:

1. **Для production с русским текстом:**
   ```yaml
   title_embedder:
     model_name: "intfloat/multilingual-e5-large"  # Вместо all-MiniLM-L6-v2
   ```

2. **Сохранить e5-large локально:**
   ```bash
   python scripts/save_sentence_transformer_model.py \
     --model-name "intfloat/multilingual-e5-large" \
     --output-dir "dp_models/bundled_models/text/embeddings/multilingual-e5-large"
   ```

3. **Обновить spec файл** (если нужно изменить путь):
   - `dp_models/spec_catalog/text/intfloat_multilingual-e5-large.yaml`
