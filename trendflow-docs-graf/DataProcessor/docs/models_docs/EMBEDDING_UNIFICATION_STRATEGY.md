# Стратегия унификации моделей эмбеддингов

## Анализ идеи: поставить все embedder'ы на `multilingual-e5-large`

### ✅ Плюсы унификации

1. **Единая размерность (1024)**
   - Все эмбеддинги имеют одинаковую размерность
   - Упрощает downstream обработку (cosine metrics, aggregators)
   - Не нужно делать PCA/projection для унификации

2. **Лучшая поддержка русского языка**
   - `e5-large` оптимизирована для 100+ языков, включая русский
   - `MiniLM-L6-v2` имеет ограниченную поддержку русского
   - Критично для вашего проекта (русский контент)

3. **Единая модель в памяти**
   - ModelRegistry кеширует модель один раз
   - Все extractors используют одну и ту же загруженную модель
   - Экономия памяти при использовании нескольких embedders

4. **Упрощение конфигурации**
   - Один `model_name` для всех
   - Проще управление и обновление
   - Меньше путаницы в конфигах

5. **Высокое качество для всех задач**
   - Лучше работает на длинных текстах (description, transcript)
   - Лучше для semantic search и similarity
   - Больше семантической информации (1024 vs 384 dim)

### ⚠️ Минусы и соображения

1. **Производительность**
   - **Скорость**: e5-large медленнее MiniLM (~2-3x на CPU, ~1.5x на GPU)
   - **Память**: ~560MB vs ~90MB (но загружается один раз через ModelRegistry)
   - **VRAM**: больше требований на GPU (но fp16 помогает)

2. **Избыточность для коротких текстов**
   - Для title/hashtags (короткие тексты) может быть избыточно
   - Но качество все равно лучше, особенно для русского

3. **Время обработки**
   - Первый запуск: загрузка модели ~2-5 секунд
   - Обработка: ~1-3 секунды на title (vs ~0.5-1 сек для MiniLM)
   - Но кеширование нивелирует разницу при повторных запусках

### 📊 Сравнение по use cases

| Extractors | Текст | Длина | Рекомендация e5-large | Причина |
|------------|-------|-------|----------------------|---------|
| `title_embedder` | Заголовок | Короткий (~10-50 слов) | ✅ **Да** | Важен для поиска, русский текст |
| `description_embedder` | Описание | Длинный (~100-500 слов) | ✅✅ **Очень да** | Длинный текст, нужна хорошая модель |
| `transcript_chunk_embedder` | Транскрипт | Очень длинный (чанки) | ✅✅ **Очень да** | Много чанков, качество критично |
| `comments_embedder` | Комментарии | Короткие (много) | ✅ **Да** | Батчинг эффективен, русский текст |
| `hashtag_embedder` | Хэштеги | Очень короткие | ⚠️ **Можно** | Минимум текста, но русский важен |

### 🎯 Рекомендация

**ДА, стоит унифицировать на `multilingual-e5-large` для production**

**Причины:**
1. ✅ Проект работает с русским текстом → e5-large значительно лучше
2. ✅ ModelRegistry кеширует модель → память используется эффективно
3. ✅ Единая размерность упрощает downstream обработку
4. ✅ Качество важнее скорости для production
5. ✅ Кеширование нивелирует разницу в скорости

**Когда НЕ стоит:**
- ❌ Быстрые baseline тесты (используйте MiniLM)
- ❌ Очень ограниченные ресурсы (CPU-only, мало RAM)
- ❌ Обработка только английского текста (MiniLM достаточно)

### 🔧 План миграции

1. **Сохранить e5-large локально** (уже делается)
   ```bash
   python scripts/save_sentence_transformer_model.py
   ```

2. **Обновить global_config.yaml**
   ```yaml
   text:
     extractors:
       title_embedder:
         model_name: "intfloat/multilingual-e5-large"
       description_embedder:
         model_name: "intfloat/multilingual-e5-large"
       transcript_chunk_embedder:
         model_name: "intfloat/multilingual-e5-large"
       comments_embedder:
         model_name: "intfloat/multilingual-e5-large"
       hashtag_embedder:
         model_name: "intfloat/multilingual-e5-large"
   ```

3. **Проверить производительность**
   - Запустить на тестовом наборе
   - Измерить время обработки
   - Проверить использование памяти

4. **Опционально: оставить MiniLM как fallback**
   - Для быстрых тестов
   - Для baseline сравнения

### 💡 Альтернативный подход (компромисс)

Если e5-large слишком медленная, можно использовать **`multilingual-e5-base`**:
- Размерность: 768 (компромисс между 384 и 1024)
- Размер: ~280MB (между 90MB и 560MB)
- Скорость: быстрее large, но качественнее MiniLM
- Поддержка русского: отличная

### 📈 Ожидаемые результаты

**После миграции на e5-large:**
- ✅ Лучшее качество эмбеддингов для русского текста
- ✅ Единая размерность (1024) для всех embedders
- ✅ Упрощенная конфигурация
- ⚠️ Немного медленнее (~1.5-2x), но приемлемо
- ⚠️ Больше памяти, но ModelRegistry оптимизирует

**Вывод: Для production с русским текстом - определенно стоит!**
