## 0. КОНТЕКСТ ПРОЕКТА

TrendFlow — это SaaS-платформа для анализа видеоконтента и предсказания его популярности на основе ML.

**Система состоит из двух частей:**
1. **Сайт** (то, что мы строим) — frontend (Next.js) + backend (FastAPI, уже частично реализован)
2. **MLService** (уже реализован) — Fetcher + DataProcessor (75+ компонентов: TextProcessor/AudioProcessor/VisualProcessor) + Models (Transformer-based fusion)

**Что MLService делает:**
- Принимает YouTube URL или загруженное видео
- Fetcher скачивает видео и метаданные
- DataProcessor обрабатывает видео по 75+ компонентам (текст/аудио/визуал)
- Models предсказывают просмотры и лайки через 7/14/21 дней
- Результат: детальный отчёт по каждому компоненту + прогноз + интерпретации

**Аудитория:**
- Тип A — Обычный YouTuber: хочет загрузить видео и получить прогноз + рекомендации, не знает о ML
- Тип B — Профессиональный аналитик: хочет детальные графики, распределения, факторы влияния, сравнения

**Backend уже реализован (FastAPI + PostgreSQL + Celery + Redis + WebSocket):**
- Auth (email/password + OAuth)
- Workspaces (multi-tenant)
- Channels, Videos, IngestionRuns
- AnalysisJobs, Predictions
- Subscriptions (базовая структура)
- WebSocket stream событий обработки (`/api/runs/{run_id}/events`)

**Реализация:**
```tsx
// React Three Fiber canvas, position: absolute, right side of hero
// Карточка — это HTML через drei <Html> внутри canvas
// Или: полностью WebGL card texture с анимированным Canvas 2D контентом
```

### 4.5 Страница прогресса (/dashboard/analysis/{job_id}/progress)

**Архитектура:**
- Подключается к WebSocket: `WS /api/runs/{run_id}/events?token=<JWT>`
- Обновляет UI в реальном времени
- При потере соединения — автоматический reconnect (exponential backoff)

**Layout:**
```
Верхняя часть:
- Название видео + thumbnail
- Общий прогресс-бар (0-100%)
- Статус текстом: "Обработка аудио... 45%"
- Время прошло + ожидаемое оставшееся

Центр — Live кадры:
- Сетка 2x2 или 3x3 последних обработанных кадров
- На каждом кадре — overlay с bbox (YOLO-боксы объектов, лиц)
- Под каждым кадром: "Кадр 0:42 — обнаружено: 2 человека, 1 ноутбук"
- Кадры появляются анимированно по мере обработки

Правая панель — Лог компонентов:
- Список всех компонентов с иконками статуса:
  ○ Ожидает  ◐ В процессе  ✓ Готово  ✗ Ошибка
- Группировка: TextProcessor / AudioProcessor / VisualProcessor
- При клике на компонент — показывает детальный лог

Нижняя панель:
- [Отменить анализ] (с подтверждением)
```

**Push-уведомление при завершении:**
- Browser Notification API (если разрешено)
- Telegram (если подключён)
- Email (всегда, если включено в настройках)

## 5. БИЛЛИНГ: ГИБРИДНАЯ МОДЕЛЬ

### Концепция
- У каждого workspace есть `credit_balance` (в кредитах)
- Тарифный план определяет:
  - Количество включённых кредитов в месяц (автопополнение)
  - Доступные компоненты (free tier — ограниченный набор)
  - Лимиты (max размер видео, хранение результатов)
- Дополнительные кредиты можно докупить в любое время

### Расчёт стоимости анализа
```
Для каждого выбранного компонента:
стоимость = цена_компонента_за_ед × количество_единиц_обработки

Для VisualProcessor компонентов: ед = кадры (зависит от длины видео и FPS)
Для AudioProcessor компонентов: ед = секунды аудио
Для TextProcessor компонентов: ед = блоки текста (≈1000 символов)

Итого = сумма по всем компонентам
Показывается пользователю ДО запуска
```

### Новые таблицы в PostgreSQL
```sql
-- Баланс кредитов (добавить к Workspace или отдельно)
ALTER TABLE core.workspaces ADD COLUMN credit_balance INT DEFAULT 0;

-- Транзакции кредитов
CREATE TABLE core.credit_transactions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    workspace_id UUID REFERENCES core.workspaces(id),
    amount INT NOT NULL, -- положительное = пополнение, отрицательное = списание
    type TEXT NOT NULL, -- 'purchase', 'deduction', 'refund', 'bonus', 'plan_renewal'
    analysis_job_id UUID REFERENCES core.analysis_jobs(id) NULLABLE,
    payment_provider TEXT, -- 'yookassa', 'manual'
    provider_payment_id TEXT,
    comment TEXT,
    created_at TIMESTAMP DEFAULT NOW()
);

-- Тарифы планов (дополнить subscription_plans)
ALTER TABLE core.subscription_plans
    ADD COLUMN credits_per_month INT DEFAULT 0,
    ADD COLUMN max_video_duration_sec INT DEFAULT 1200,
    ADD COLUMN results_retention_days INT DEFAULT 30,
    ADD COLUMN allowed_components TEXT[]; -- null = все

-- Invite-only система
CREATE TABLE core.invite_codes (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    created_by UUID REFERENCES core.users(id),
    email TEXT, -- null = любой email
    used_by UUID REFERENCES core.users(id) NULLABLE,
    used_at TIMESTAMP NULLABLE,
    created_at TIMESTAMP DEFAULT NOW()
);
```

## 13. СПЕЦИАЛЬНЫЕ ИНСТРУКЦИИ ДЛЯ CLAUDE

> Это раздел написан специально для Claude AI, который будет реализовывать сайт.

1. **Читай backend перед написанием API calls.** Все эндпоинты в `/TrendFlowML/backend/docs/API.md`. Типы — в `/TrendFlowML/backend/app/schemas.py` и `dbv2/models.py`. Не выдумывай структуры — бери из источника.

2. **WebSocket.** Протокол событий описан в `/TrendFlowML/backend/docs/EVENTS_AND_LOGGING.md`. При подключении обязательно передавать JWT как query param `?token=`.

3. **Дизайн — строго по этому документу.** Не используй светлые фоны в тёмной теме. Акцент только фиолетовый (#7c3aed). Никаких синих кнопок.

4. **Числа всегда через JetBrains Mono.** Virality score, кредиты, просмотры, лайки — всё через `font-mono`.

5. **Компоненты результатов анализа** — бери только те, что выбраны в `processing_config`. Не показывай компоненты которые не были запрошены.

6. **Кредиты** — всегда показывай текущий баланс в sidebar. При недостатке — блокировать запуск, показывать кнопку "Пополнить".

7. **Loading states** — каждый запрос должен иметь skeleton/spinner. Никаких "пустых" экранов.

8. **Error states** — каждый запрос должен обрабатывать ошибки. Toast уведомления через sonner.

9. **3D фон** — lazy load через `next/dynamic` с `ssr: false`. На мобиле (`useMediaQuery`) — отключать.

10. **Все тексты в messages/ru.json.** Не хардкодить строки в компонентах.

11. **TypeScript strict mode.** Никаких `any`. Все типы из `/types/` директории.

12. **Backend URL** — брать из `.env.local`: `NEXT_PUBLIC_API_URL=http://localhost:8000`. Не хардкодить.

13. **Virality Score** — не реализован в backend v1. Считать на frontend как взвешенное среднее из prediction outputs (пока заглушка, детали уточнить позже).

14. **Демо на лендинге** — полностью мок, никаких реальных запросов к MLService. Просто красивая анимация с hardcoded данными.

15. **Admin доступ** — проверять `user.is_superuser` (добавить поле в User модель backend). Редирект на /dashboard если не superuser.
