## 2. Технические детали

### 2.1. Checksum вычисление

**Файл**: `fetcher/checksums.py`

```python
def compute_sha256(file_path: str | Path, chunk_size: int = 8192) -> str:
    """Computes the SHA256 checksum of a file."""
```

- Использует SHA256 для вычисления хеша файла
- Читает файл по частям (chunk_size=8192) для эффективности
- Возвращает hexdigest строку

### 2.2. Интеграция в YouTubeAdapter

**Изменённые файлы**:
- `fetcher/platforms/youtube/adapter.py`

**Изменения**:
- В `fetch_metadata`: после upload `meta.json` вычисляется checksum и size_bytes, сохраняется в `Artifact`
- В `download_video`: после upload `video.mp4` вычисляется checksum и size_bytes, сохраняется в `Artifact`
- В `fetch_comments`: после upload `comments.json` вычисляется checksum и size_bytes, сохраняется в `Artifact`

**Формат checksum**: `sha256:<hexdigest>` (например, `sha256:abc123...`)

### 2.3. Manifest validation

**Файл**: `fetcher/manifest_validator.py`

**Функции**:
- `validate_manifest(manifest: FetcherManifest) -> tuple[bool, Optional[str]]`:
  - Проверяет обязательные поля (`video_id`, `platform`, `duration_seconds`)
  - Проверяет наличие и корректность артефактов (`video_file`, `meta_file`, `comments_file`)
  - Проверяет формат checksums (должны начинаться с `sha256:`)
  - Проверяет `size_bytes > 0` для всех артефактов
- `validate_manifest_artifacts_exist(manifest, bucket) -> tuple[bool, Optional[str]]`:
  - Проверяет, что все артефакты из manifest существуют в storage

**Интеграция**: В `fetcher/workers/artifacts.py`, функция `run_artifact_builder`:
- Валидация выполняется перед сохранением manifest в storage
- При ошибке валидации выбрасывается `ValueError` с описанием проблемы

## 3. Тестирование

### 3.1. Ручные проверки

- ✅ Checksum вычисляется корректно для файлов разного размера
- ✅ Manifest валидация обнаруживает отсутствующие обязательные поля
- ✅ Manifest валидация обнаруживает некорректные checksums
- ✅ Manifest валидация обнаруживает артефакты с size_bytes <= 0

### 3.2. Edge cases

- ✅ Обработка пустых файлов (size_bytes = 0)
- ✅ Обработка очень больших файлов (chunk-based reading)
- ✅ Обработка отсутствующих опциональных артефактов (comments_file)

## 4. Известные ограничения
