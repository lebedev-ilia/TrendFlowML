## 2. Технические детали

### 2.1. Circuit Breaker States

**Файл**: `fetcher/circuit_breaker.py`

**Состояния**:
- **CLOSED**: Нормальная работа, операции разрешены
- **OPEN**: Блокировка операций (превышен порог ошибок)
- **HALF_OPEN**: Тестовый режим после cooldown (ограниченное количество запросов)

**Переходы**:
- CLOSED → OPEN: При превышении `failure_threshold` ошибок за `failure_window_sec`
- OPEN → HALF_OPEN: После `cooldown_seconds`
- HALF_OPEN → CLOSED: После `success_threshold` успешных запросов
- HALF_OPEN → OPEN: При ошибке в HALF_OPEN

### 2.2. Configuration

**Файл**: `fetcher/config.py`

```python
enable_circuit_breaker: bool = Field(True, ...)
circuit_breaker_failure_threshold: int = Field(5, ...)
circuit_breaker_failure_window_sec: int = Field(60, ...)
circuit_breaker_cooldown_seconds: int = Field(300, ...)
circuit_breaker_success_threshold: int = Field(3, ...)
circuit_breaker_half_open_max_attempts: int = Field(5, ...)
```

### 2.3. Integration

**Изменённый файл**: `fetcher/platforms/youtube/adapter.py`

**Изменения**:
- В `fetch_metadata`, `download_video`, `fetch_comments`:
  - Вызов `check_circuit_breaker(operation)` перед операцией
  - Вызов `record_circuit_breaker_result(operation, success, reason)` после операции
  - Обработка `CircuitBreakerOpenError` с логированием

### 2.4. Metrics

**Метрика**: `circuit_breaker_tripped_total` (Counter)
- Инкрементируется при каждом срабатывании circuit breaker
- Labels: `operation` (metadata/download/comments), `reason` (failure_threshold_exceeded, half_open_failure)

## 3. Технические детали

### 3.1. Redis Storage

Circuit breaker использует Redis для хранения состояния:
- `circuit:{operation}` — текущее состояние (CLOSED/OPEN/HALF_OPEN)
- `circuit:{operation}:error_count` — счётчик ошибок (с TTL = failure_window_sec)
- `circuit:{operation}:open_ts` — timestamp открытия circuit breaker
- `circuit:{operation}:half_open_attempts` — счётчик попыток в HALF_OPEN

### 3.2. Error Detection

Circuit breaker реагирует на:
- HTTP 429 (Too Many Requests)
- HTTP 403 (Forbidden) — может быть временным
- Network errors
- Timeout errors

## 4. Тестирование

### 4.1. Ручные проверки

- ✅ Circuit breaker открывается при превышении failure_threshold
- ✅ Circuit breaker блокирует операции в состоянии OPEN
- ✅ Circuit breaker переходит в HALF_OPEN после cooldown
- ✅ Circuit breaker закрывается после success_threshold успешных запросов в HALF_OPEN
- ✅ Метрика обновляется при каждом срабатывании

### 4.2. Edge cases

- ✅ Обработка ошибок Redis (fail-open подход)
- ✅ Обработка случая, когда circuit breaker отключен через флаг
- ✅ Обработка множественных операций (metadata, download, comments имеют отдельные circuit breakers)

---

## 5. Известные ограничения

- Circuit breaker не различает типы ошибок (429 vs 403 vs network) — все считаются одинаково
- Нет автоматического восстановления после длительного периода хорошей работы (требует ручного сброса)
- Нет поддержки разных порогов для разных операций (все используют одни настройки)
