## 2. Технические детали

### 2.1. Database Schema

**Таблица `proxies`** (`fetcher/models.py`):
- `id` (UUID, PK)
- `url` (Text, unique) — URL прокси
- `country` (String(2), nullable) — ISO 3166-1 alpha-2 код страны (например, "US", "GB")
- `enabled` (bool, default=True) — флаг включения/выключения прокси
- `notes` (Text, nullable) — заметки о прокси
- `created_at`, `updated_at` (DateTime)

**Таблица `proxy_usage`** (`fetcher/models.py`):
- `id` (UUID, PK)
- `proxy_id` (UUID, FK к `proxies.id`)
- `operation` (String(50)) — тип операции (metadata, download, comments)
- `success` (bool) — успешность запроса
- `latency_ms` (Integer, nullable) — время ответа в миллисекундах
- `error_message` (Text, nullable) — сообщение об ошибке (ограничено 500 символов)
- `created_at` (DateTime)

### 2.2. Proxy Usage Logging

**Изменённый файл**: `fetcher/proxies.py`

**Функция `record_proxy_result()`**:
- Теперь принимает дополнительные параметры: `operation`, `latency_ms`, `error_message`
- Логирует каждое использование в таблицу `proxy_usage` (best-effort, не блокирует основной поток)
- Обновляет метрику `proxy_failure_rate` с правильным `country` из БД

### 2.3. Geographic Rotation

**Функция `get_next_proxy(country: Optional[str] = None)`**:
- Если указана `country`, ищет прокси из этой страны в БД
- Выбирает здоровый прокси (failure_rate < threshold) из указанной страны
- Fallback на обычный round-robin из настроек, если прокси из указанной страны не найдены или нездоровы

### 2.4. Integration

**Изменённый файл**: `fetcher/platforms/youtube/adapter.py`

**Изменения**:
- В `fetch_metadata`, `download_video`, `fetch_comments`:
  - Измеряется `latency_ms` для каждого запроса
  - Вызывается `record_proxy_result()` с `operation`, `latency_ms`, `error_message`
  - `error_message` ограничивается 500 символами для БД

## 3. Технические детали

### 3.1. Best-Effort Logging

- Логирование в БД выполняется в try-except блоке
- Ошибки БД логируются как warning, но не прерывают основной поток
- Это гарантирует, что проблемы с БД не влияют на ingestion pipeline

### 3.2. Country Code Format

- Используется ISO 3166-1 alpha-2 формат (2 символа, uppercase)
- Примеры: "US", "GB", "DE", "FR"
- Если `country` не указан, используется обычный round-robin

## 5. Известные ограничения
