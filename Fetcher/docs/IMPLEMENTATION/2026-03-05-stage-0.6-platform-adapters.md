## 2. Технические детали

### 2.1. Базовый `PlatformAdapter`

- Интерфейс:
  - `fetch_metadata(source: str, *, run_id: str) -> None`
  - `download_video(source: str, *, run_id: str) -> None`
  - `fetch_comments(source: str, *, run_id: str, limit: int = 100) -> None`
- Общие требования:
  - **идемпотентность**:
    - перед выполнением операции адаптер проверяет наличие соответствующих записей/артефактов (`video_metadata`, `artifacts`, `comments`);
    - при наличии валидных данных операция может быть no‑op;
  - **ошибки**:
    - адаптеры различают retryable и non‑retryable ошибки;
    - используют нормализованные коды ошибок (`YOUTUBE_429`, `VIDEO_NOT_FOUND`, `DOWNLOAD_TIMEOUT`, и т.п.);
  - **лимиты**:
    - уважают timeout’ы, лимиты попыток и будущий Redis‑rate‑лимитер/proxy‑pool.

### 2.2. YouTubeAdapter

- Метаданные (`fetch_metadata`):
  - получает данные через `yt-dlp --dump-json` и/или YouTube API;
  - заполняет:
    - таблицы `video_metadata`, `channel_metadata` (нормализованные поля + `raw_json`);
    - опционально `meta.json` в `video-analytics-raw` + запись в `artifacts`.
- Видео (`download_video`):
  - использует `yt-dlp` для скачивания лучшего доступного видео ≤720p и `ffmpeg` для нормализации;
  - pipeline: download → `/tmp` → upload в `video-analytics-raw` → удаление `/tmp`;
  - регистрирует артефакт в `artifacts` (`video_file`, `storage_path`, `size_bytes`, `checksum`, `status`).
- Комментарии (`fetch_comments`):
  - собирает top‑N комментариев (по умолчанию ≤100) с агрегированными полями и `raw_json`;
  - сохраняет либо:
    - в таблицу `comments`; либо
    - в `comments.json` в `video-analytics-raw` c записью в `artifacts`.

Все операции YouTubeAdapter’а должны генерировать события (`FetcherEvent`) и логи для observability.

### 2.3. Feature‑флаги платформ

- Конфигурация:
  - глобальный список `FETCHER_ENABLED_PLATFORMS = ["youtube", ...]`;
  - индивидуальные флаги `FETCHER_YOUTUBE_ENABLED`, `FETCHER_TIKTOK_ENABLED`, и т.п.
- Поведение:
  - Orchestrator при выборе адаптера по `platform` проверяет конфиг;
  - при отключённой платформе:
    - ingestion не запускается;
    - run помечается как `FAILED` с кодом `PLATFORM_DISABLED`;
    - публикуется соответствующее событие.

## 4. Известные ограничения и TODO

- **Нет реальной реализации адаптеров**:
  - YouTubeAdapter и другие платформы пока не реализованы в коде;
  - требуется отдельный этап для добавления Python‑классов, интеграции с `yt-dlp`, прокси и rate‑лимитом.
- **Нет строгой типизации платформ и кодов ошибок**:
  - платформы и коды ошибок представлены строками; в будущем рекомендуется ввести enum‑типы;
  - потребуется единый реестр кодов ошибок (например, в отдельном модуле).
- **Нет конфигурационного документа Fetcher**:
  - флаги `FETCHER_ENABLED_PLATFORMS`, `FETCHER_YOUTUBE_ENABLED`, и т.п. пока описаны только текстом;
  - переменные окружения и конфиг‑файлы должны быть задокументированы отдельно.
