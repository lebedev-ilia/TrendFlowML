## 2. Технические детали

### 2.1. Snapshot Creation

**Файл**: `fetcher/snapshots.py`

**Функция**:
```python
def create_initial_snapshot_from_info(
    platform: str, platform_video_id: str, info: Dict[str, Any]
) -> None:
```

**Логика**:
- Находит `Video` по `platform` и `platform_video_id`
- Проверяет, существует ли уже `snapshot_index=0`
- Создаёт новый `VideoSnapshot` с данными из `yt-dlp info_dict`:
  - `view_count`
  - `like_count`
  - `comment_count`
  - `subscriber_count` (из `channel_follower_count`)
  - `collected_at = datetime.now(timezone.utc)`

### 2.2. Интеграция в YouTubeAdapter

**Изменённый файл**: `fetcher/platforms/youtube/adapter.py`

**Изменения**:
- В `fetch_metadata`: после успешного получения metadata вызывается `create_initial_snapshot_from_info` если `settings.enable_snapshots == True`
- Snapshot создаётся только один раз (при первом fetch), последующие fetch не создают дубликаты

### 2.3. Конфигурация

**Файл**: `fetcher/config.py`

```python
enable_snapshots: bool = Field(
    True,
    description="Включить запись initial video_snapshots (snapshot_index=0).",
)
```

## 3. Технические детали

### 3.1. Database Schema

Используется существующая таблица `video_snapshots`:
- `video_id` (FK к `videos`)
- `snapshot_index` (0 для initial snapshot)
- `view_count`, `like_count`, `comment_count`, `subscriber_count`
- `collected_at` (timestamp)

### 3.2. Идемпотентность

- Проверка существования `snapshot_index=0` предотвращает создание дубликатов
- Если snapshot уже существует, функция просто возвращается без ошибки

---

## 4. Тестирование

### 4.1. Ручные проверки

- ✅ Snapshot создаётся при первом fetch metadata
- ✅ Snapshot не создаётся повторно при последующих fetch
- ✅ Все метрики корректно сохраняются в БД
- ✅ Snapshot может быть отключен через флаг

### 4.2. Edge cases

- ✅ Обработка случая, когда видео не найдено в БД (логируется warning)
- ✅ Обработка отсутствующих полей в `info_dict` (None значения)

## 5. Известные ограничения
