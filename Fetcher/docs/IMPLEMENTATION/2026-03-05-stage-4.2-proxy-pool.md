## 2. Технические детали

### 2.1. Health Scoring

**Файл**: `fetcher/proxies.py`

**Функции**:
- `get_proxy_health_score(proxy_url: str) -> float`:
  - Возвращает health score от 0.0 (плохой) до 1.0 (отличный)
  - Формула: `1.0 - failure_rate`
  - Если нет данных (0 запросов), возвращает 1.0 (считаем хорошим)
- `is_proxy_healthy(proxy_url: str) -> bool`:
  - Проверяет, не превышен ли порог failure rate (50% по умолчанию)
  - Учитывает минимальное количество запросов (10 по умолчанию) для оценки

**Параметры**:
- `_proxy_failure_threshold = 0.5` (50% failure rate)
- `_proxy_min_requests = 10` (минимум запросов для оценки)

### 2.2. Automatic Bad Proxy Eviction

**Изменения в `get_next_proxy()`**:
- Теперь пропускает нездоровые прокси (failure_rate > 50%)
- Пробует найти здоровый прокси (максимум N попыток, где N = len(proxies) * 2)
- Если все прокси нездоровы, возвращает None (fallback на прямой запрос)

### 2.3. Metrics Integration

**Метрика**: `proxy_failure_rate` (Gauge)
- Обновляется при каждом вызове `record_proxy_result()`
- Labels: `proxy_id` (URL прокси), `country` (пока "unknown")

## 3. Технические детали

### 3.1. In-Memory Statistics

**Хранение**: `_proxy_stats: Dict[str, Dict[str, int]]`
- Ключ: proxy URL
- Значение: `{"successes": int, "failures": int}`

**Обновление**: При каждом вызове `record_proxy_result(proxy_url, success)`

### 3.2. Thread Safety

- Используется `threading.Lock()` для защиты `_proxy_stats` и `_proxy_cycle`
- Все операции с shared state защищены lock'ом
