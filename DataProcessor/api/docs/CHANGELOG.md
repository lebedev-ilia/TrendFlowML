  - Graceful shutdown для API: остановка приема новых запросов через `shutdown_event`, ожидание завершения текущих запросов (5 секунд), очистка ресурсов (Redis клиент)
  - Graceful shutdown для Worker: остановка приема новых задач через `shutdown_event`, завершение текущих задач с таймаутом 5 минут, обновление состояния активных run'ов на "recovering", удаление heartbeat записей из Redis, ACK pending сообщений в очереди для перераспределения другим worker'ам
  - Отслеживание активных задач через `active_tasks` dict в Worker для корректного завершения
  - Проверка `shutdown_event` перед обработкой новых сообщений в worker loop
  - Метод `_process_run_task()` вынесен для отслеживания активных задач
  - См. детали в архитектуре: `DATAPROCESSOR_API_ARCHITECTURE.md` (строка 2271-2317)
