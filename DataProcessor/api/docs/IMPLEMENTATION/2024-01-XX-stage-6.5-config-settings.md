## Технические детали

### Префикс переменных окружения

Все переменные используют префикс `TF_BACKEND_`:
```python
class Config:
    env_prefix = "TF_BACKEND_"
    env_file = ".env"
```

### Маппинг переменных окружения

- `TF_BACKEND_DATAPROCESSOR_API_URL` → `dataprocessor_api_url`
- `TF_BACKEND_DATAPROCESSOR_API_KEY` → `dataprocessor_api_key`
- `TF_BACKEND_DATAPROCESSOR_POLL_INTERVAL` → `dataprocessor_poll_interval`
- `TF_BACKEND_DATAPROCESSOR_TIMEOUT_SECONDS` → `dataprocessor_timeout_seconds`

### Значения по умолчанию

**Development**:
- `dataprocessor_api_url`: `http://localhost:8001`
- `dataprocessor_api_key`: `None` (development mode разрешает доступ без ключа)
- `dataprocessor_poll_interval`: `5` секунд
- `dataprocessor_timeout_seconds`: `3600` секунд (1 час)

**Production** (рекомендуется):
- `dataprocessor_api_url`: `http://dataprocessor:8000` (если в docker-compose)
- `dataprocessor_api_key`: Установить секретный ключ
- `dataprocessor_poll_interval`: `5` секунд (можно увеличить для снижения нагрузки)
- `dataprocessor_timeout_seconds`: `3600` секунд (можно увеличить для длительных обработок)

## Использование

### Установка переменных окружения

**Через .env файл**:
```bash
cp backend/.env.example backend/.env
# Отредактировать backend/.env
```

**Через переменные окружения**:
```bash
export TF_BACKEND_DATAPROCESSOR_API_URL=http://dataprocessor:8000
export TF_BACKEND_DATAPROCESSOR_API_KEY=your-secret-key
export TF_BACKEND_DATAPROCESSOR_POLL_INTERVAL=5
export TF_BACKEND_DATAPROCESSOR_TIMEOUT_SECONDS=3600
```

**В docker-compose**:
```yaml
services:
  backend:
    environment:
      - TF_BACKEND_DATAPROCESSOR_API_URL=http://dataprocessor:8000
      - TF_BACKEND_DATAPROCESSOR_API_KEY=${DATAPROCESSOR_API_KEY}
      - TF_BACKEND_DATAPROCESSOR_POLL_INTERVAL=5
      - TF_BACKEND_DATAPROCESSOR_TIMEOUT_SECONDS=3600
```
