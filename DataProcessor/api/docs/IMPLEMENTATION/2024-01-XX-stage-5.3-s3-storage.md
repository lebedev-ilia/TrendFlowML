## Технические детали

### Streaming чтение JSONL

```python
# S3Storage.stream_jsonl()
async def stream_jsonl(self, key: str) -> AsyncIterator[dict]:
    """Async streaming чтение JSONL файла построчно."""
    resp = self._client.get_object(Bucket=self.bucket, Key=k)
    body = resp["Body"]
    
    buffer = ""
    for chunk in body.iter_chunks(chunk_size=8192):
        buffer += chunk.decode("utf-8", errors="replace")
        
        while "\n" in buffer:
            line, buffer = buffer.split("\n", 1)
            if line.strip():
                try:
                    yield json.loads(line)
                except json.JSONDecodeError:
                    continue
```

### Presigned URL генерация

```python
# S3Storage.generate_presigned_url()
def generate_presigned_url(
    self,
    key: str,
    expiration: int = 3600,
    http_method: str = "GET"
) -> str:
    """Генерация presigned URL для прямого доступа к объекту."""
    url = self._client.generate_presigned_url(
        http_method.lower(),
        Params={"Bucket": self.bucket, "Key": k},
        ExpiresIn=expiration
    )
    return url
```

## Производительность

### Преимущества streaming

1. **Экономия памяти**
   - Не загружает весь файл в память
   - Особенно важно для больших JSONL файлов (state_events.jsonl)

2. **Быстрый старт обработки**
   - Начинает обработку сразу после получения первой строки
   - Не ждет загрузки всего файла

3. **Масштабируемость**
   - Работает с файлами любого размера
   - Не зависит от размера файла

### Presigned URL

1. **Снижение нагрузки на API**
   - Клиенты получают прямой доступ к S3
   - Не проходит через API сервер

2. **TTL 1 час**
   - Баланс между безопасностью и удобством
   - Достаточно для загрузки артефактов

1. Установить переменные окружения:
   - `TREND_STORAGE_BACKEND=s3`
   - `S3_ENDPOINT=http://localhost:9000`
   - `S3_BUCKET=trendflow`
   - `S3_PREFIX=trendflowml`
   - `AWS_DEFAULT_REGION=us-east-1`

2. API автоматически использует S3Storage
3. Worker автоматически использует S3Storage
