# Этап 6.2: Polling для статуса

**Дата**: 2024-01-XX  
**Этап**: 6.2 из [API_DEVELOPMENT_CHECKLIST.md](../../../docs/API_DEVELOPMENT_CHECKLIST.md)  
**Статус**: ✅ Завершено

## Обзор

Реализована функция polling статуса обработки run'а для отслеживания прогресса до завершения. Это позволяет backend отслеживать статус обработки без необходимости постоянного опроса вручную.

## Выполненные задачи

### ✅ Реализовать `poll_run_status()`

Создана функция в `backend/app/services/dataprocessor.py`:

```python
async def poll_run_status(
    run_id: str,
    timeout_seconds: Optional[int] = None,
    poll_interval: Optional[int] = None
) -> Dict[str, Any]:
```

**Особенности**:
- Периодические запросы к `/api/v1/runs/{run_id}/status`
- Poll каждые 5 секунд (настраивается через `poll_interval`)
- Timeout 3600 секунд (настраивается через `timeout_seconds`)
- Обработка финальных статусов (success, error, empty, skipped, cancelled)
- Обработка ошибок (404, timeout, HTTP ошибки)
- Логирование прогресса и статуса

### ✅ Poll каждые 5 секунд

Интервал polling настраивается через:
- `settings.dataprocessor_poll_interval` (по умолчанию 5 секунд)
- Переменная окружения `TF_BACKEND_DATAPROCESSOR_POLL_INTERVAL`

```python
if poll_interval is None:
    poll_interval = settings.dataprocessor_poll_interval
```

### ✅ Timeout 3600 секунд

Timeout настраивается через:
- `settings.dataprocessor_timeout_seconds` (по умолчанию 3600 секунд = 1 час)
- Переменная окружения `TF_BACKEND_DATAPROCESSOR_TIMEOUT_SECONDS`

```python
if timeout_seconds is None:
    timeout_seconds = settings.dataprocessor_timeout_seconds
```

При истечении timeout выбрасывается `TimeoutError` с детальной информацией.

### ✅ Обработка финальных статусов

Функция завершается при получении финального статуса:

```python
final_statuses = ["success", "error", "empty", "skipped", "cancelled"]
if current_status in final_statuses:
    return status_data
```

**Обработка статусов**:
- `success` - обработка завершена успешно
- `error` - обработка завершена с ошибкой
- `empty` - обработка завершена, но результат пустой
- `skipped` - обработка пропущена
- `cancelled` - обработка отменена

### ✅ Интегрировать в Celery задачу

Обновлена Celery задача `process_analysis_job` в `backend/app/tasks.py`:

**Было** (subprocess):
```python
proc = subprocess.Popen(cmd, ...)
exit_code = proc.wait()
```

**Стало** (HTTP + polling):
```python
# Отправить запрос на обработку
run_paths = asyncio.run(run_dataprocessor_async(...))

# Polling статуса до завершения
final_status = asyncio.run(poll_run_status(...))

# Обработать финальный статус
if status == "success":
    # Успешное завершение
elif status == "error":
    # Ошибка обработки
```

**Обработка ошибок**:
- `TimeoutError` → failed с error_code="timeout"
- `ValueError` → failed с error_code="validation_error"
- Другие исключения → failed с error_code="api_error"

### Логирование

Функция логирует:
- Начало polling с параметрами
- Обновления статуса (debug уровень)
- Завершение с финальным статусом (info уровень)
- Ошибки (error/warning уровень)

### Обработка transient ошибок

При transient ошибках (connection errors) polling продолжается:
- Ошибки соединения не прерывают polling
- Ожидание перед повтором (poll_interval)
- Логирование ошибок для диагностики
