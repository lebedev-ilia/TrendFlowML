### ⚠️ Ограничения MVP

1. **In-memory registry** - данные теряются при перезапуске сервера
2. **Нет персистентности** - для production нужен Redis (Этап 2)
3. **Нет масштабирования** - нельзя запустить несколько worker'ов
4. **Нет очереди** - задачи выполняются в том же процессе что и API

### 📋 Следующие шаги

Для production рекомендуется:
1. **Этап 2**: Redis интеграция для персистентности и масштабируемости
2. **Этап 3**: Разделение API и Worker процессов
3. **Этап 4**: Добавление недостающих endpoints (метаданные, артефакты)
