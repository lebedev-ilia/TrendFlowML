## Выполненные задачи

### ✅ Реализовать идемпотентность processors

**Реализация:**

1. **Модуль `api/services/idempotency.py`**:
   - Функция `check_existing_result()` для проверки существующего результата run'а
   - Функция `check_component_result()` для проверки результата отдельного компонента
   - Логика проверки:
     - Проверка существования manifest.json в Storage
     - Проверка статуса run'а (должен быть "success")
     - Проверка наличия всех артефактов компонентов в Storage
     - Возврат существующего результата если все проверки пройдены

2. **Интеграция в worker**:
   - Проверка существующего результата перед запуском subprocess
   - Использование кэша при повторном запуске уже обработанного run'а
   - Обновление статуса на SUCCESS при использовании кэша
   - Добавление события `processing_completed_from_cache` в event stream
   - ACK сообщения без запуска subprocess при использовании кэша

**Файлы**:
- `api/services/idempotency.py` - новый модуль для проверки идемпотентности
- `api/services/worker.py` - интеграция проверки идемпотентности перед запуском обработки

### Архитектурные решения

1. **Проверка существующих результатов**
   - Проверка выполняется в worker перед запуском subprocess
   - Используется manifest.json как источник истины
   - Проверяется наличие всех артефактов в Storage

2. **Fail-safe подход**
   - При ошибке проверки кэша run обрабатывается заново
   - Это гарантирует, что ошибки проверки не блокируют обработку

3. **Использование кэша**
   - При обнаружении существующего результата:
     - Обновляется статус на SUCCESS
     - Добавляется событие в event stream
     - ACK сообщения в очереди
     - Subprocess не запускается
