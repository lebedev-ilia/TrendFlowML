## Выполненные задачи

### ✅ Реализовать проверку длины queue

1. **Функция `get_total_queue_length()`** в `api/services/queue.py`
   - Суммирует длины всех очередей (high + normal + low)
   - Использует `XLEN` для каждой очереди
   - Возвращает 0 если Redis не доступен (не блокирует обработку)
   - Логирует ошибки, но не прерывает выполнение

2. **Конфигурация `MAX_QUEUE_LENGTH`** в `api/config.py`
   - Добавлено поле `max_queue_length: int` в `APIConfig`
   - Значение по умолчанию: 100
   - Настраивается через переменную окружения `MAX_QUEUE_LENGTH`

3. **Проверка в `POST /api/v1/process`**
   - Проверка выполняется после проверки idempotency lock
   - Проверка выполняется только если Redis доступен
   - Если очередь заполнена, выбрасывается `BackpressureError` с `retry_after=300` (5 минут)

4. **Обработка ошибки 503**
   - Используется существующий `BackpressureError`
   - Exception handler в `api/main.py` возвращает 503 с header `Retry-After`
   - Header `Retry-After` устанавливается в 300 секунд (5 минут)

## Технические детали

### Измененные файлы

1. **`api/services/queue.py`**
   - Добавлена функция `get_total_queue_length()`
   - Суммирует длины всех очередей через `XLEN`
   - Обрабатывает ошибки gracefully (возвращает 0)

2. **`api/config.py`**
   - Добавлено поле `max_queue_length: int = 100`
   - Настраивается через `MAX_QUEUE_LENGTH` environment variable

3. **`api/endpoints/process.py`**
   - Добавлен импорт `get_total_queue_length`
   - Добавлена проверка длины очереди перед enqueue
   - Выбрасывает `BackpressureError` при превышении лимита

### Архитектурные решения

1. **Проверка только при доступности Redis**
   - Если Redis не доступен, проверка пропускается
   - Это позволяет системе работать в fallback режиме без Redis

2. **Graceful degradation**
   - Если не удалось проверить длину очереди, логируется предупреждение, но обработка продолжается
   - Это предотвращает блокировку системы при временных проблемах с Redis

3. **Retry-After header**
   - Установлен в 300 секунд (5 минут)
   - Это дает системе время обработать накопившиеся задачи
   - Клиент может повторить запрос через указанное время

4. **Порядок проверок**
   - Проверка длины очереди выполняется после проверки idempotency lock
   - Это предотвращает установку lock для задач, которые не могут быть обработаны

## Использование

### Автоматическое использование

Backpressure работает автоматически:

```python
# При запросе POST /api/v1/process:
# 1. Проверяется длина очереди через get_total_queue_length()
# 2. Если очередь заполнена - возвращается 503 с Retry-After
# 3. Если очередь не заполнена - обработка продолжается
```

### Ручное использование

Для ручной проверки длины очереди:

```python
from api.services.queue import get_total_queue_length

# Получить общую длину очереди
total_length = await get_total_queue_length()
if total_length >= config.max_queue_length:
    # Очередь заполнена
    raise BackpressureError(
        f"Queue is full: {total_length}/{config.max_queue_length}",
        retry_after=300
    )
```

### Конфигурация

Настройка через environment variable:

```bash
# Установить лимит очереди
export MAX_QUEUE_LENGTH=100

# Или в docker-compose.yml
environment:
  - MAX_QUEUE_LENGTH=100
```

## Известные проблемы и ограничения

1. **Проверка только при доступности Redis**
   - Если Redis не доступен, проверка пропускается
   - Это может привести к переполнению очереди в fallback режиме
   - TODO: Добавить проверку длины очереди в fallback режиме (TaskManager)

2. **Нет учета pending сообщений**
   - Проверяется только длина очереди, но не учитываются pending сообщения
   - Pending сообщения также занимают место в очереди
   - TODO: Учитывать pending сообщения при проверке backpressure

3. **Статический Retry-After**
   - Retry-After всегда 300 секунд
   - Не учитывается текущая нагрузка и скорость обработки
   - TODO: Динамический расчет Retry-After на основе скорости обработки

4. **Нет метрик**
   - Нет метрик для мониторинга длины очереди
   - Сложно отслеживать перегрузку системы
   - TODO: Добавить метрики для Prometheus/Grafana
