## 1. Проблемы при тестировании (исторические)

### 1.1 Основная проблема: недоступность Triton

**Проблема**: Большинство тестов падают на этапе `cut_detection` из-за недоступности Triton сервера для CLIP модели.

**Ошибка**:
```
TritonError: POST http://localhost:8000/v2/models/clip_image_224/versions/1/infer failed: 
<urlopen error [Errno 111] Connection refused>
```

**Цепочка зависимостей**:
- `color_light` требует `scene_classification`
- `scene_classification` требует `cut_detection`
- `cut_detection` требует `core_clip` через Triton (когда `use_clip=true`)

**Решение**: 
- Для успешного тестирования требуется запущенный Triton сервер
- Или использовать inprocess режим для `core_clip` (но `cut_detection` все равно требует Triton для CLIP)

### 1.2 Исправленные проблемы

1. **Ошибка валидации схемы scene_classification**:
   - Проблема: `dominant_places_topk_ids` и `dominant_places_topk_probs` создавались как 2D массивы вместо 1D object arrays
   - Решение: Явное создание 1D object arrays через `np.empty(n, dtype=object)`

2. **Отсутствие `meta` в схемах**:
   - Проблема: BaseModule автоматически добавляет `meta` в NPZ, но схемы не разрешали этот ключ
   - Решение: Добавлен `meta` в схемы `scene_classification_npz_v2.json` и `color_light_npz_v2.json`

3. **Ошибка в рендере**:
   - Проблема: f-string с обратным слешем вызывал синтаксическую ошибку
   - Решение: Исправлена строка в `render.py` (строка 491)
