## Качество данных

### Валидация схемы

Все артефакты соответствуют схеме `action_recognition_npz_v2`:
- ✅ Все обязательные ключи присутствуют
- ✅ Метаданные корректны
- ✅ Структура данных согласована

### Проверка согласованности

- ✅ Количество треков = количество embeddings = количество results_json
- ✅ Размерность embeddings: 256d (корректно)
- ✅ L2 нормализация эмбеддингов: корректна
- ✅ Все обязательные поля в results_json присутствуют

### Распределение метрик

**Наблюдения:**
- Большинство треков имеют `num_clips=1` (короткие сегменты)
- `stability=1.0` для всех треков (один клип = максимальная стабильность)
- `temporal_jumps=0.0` (нет соседних клипов для сравнения)
- Это ожидаемо для коротких видео с редкими person детекциями

## Производительность

Среднее время обработки:
- **Короткие видео** (12-60 сек): ~60-130 сек
- **Средние видео** (60-135 сек): ~55-150 сек
- **Длинные видео** (236-759 сек): ~105-150 сек

**Зависит от:**
- Количества person детекций
- Количества треков/сегментов
- Загрузки GPU

---

## Известные ограничения

1. **Короткие сегменты**: Большинство треков имеют только 1 клип, поэтому temporal jumps = 0 (ожидаемо)
2. **Редкие детекции**: На некоторых видео мало person детекций → мало треков
3. **Время обработки**: Зависит от количества треков и длины видео
