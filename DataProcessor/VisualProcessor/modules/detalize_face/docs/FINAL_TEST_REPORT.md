## 4. Сравнительный анализ

### 4.1. Сравнение с другими компонентами

**detalize_face** показывает:
- ✅ Высокую информативность метрик (CV > 0.99 для большинства метрик)
- ✅ Корректную обработку случаев без лиц (пропуск с `no_faces_in_video`)
- ✅ Стабильную работу на всех тестовых видео
- ✅ Корректную обработку частичного покрытия кадров (warning вместо error)

### 4.2. Особенности компонента

1. **Зависимость от core_face_landmarks:**
   - Компонент полностью зависит от `core_face_landmarks`
   - Если лиц нет, компонент пропускается оркестратором (не создает empty результат)
   - Если лица есть, но не для всех кадров axis, компонент продолжает работу с предупреждением

2. **Compact features:**
   - Фиксированная размерность: 40
   - Высокая информативность `compact_l2_mean` и `compact_l2_std`
   - Некоторые видео имеют `compact_l2_mean=0`, что может указывать на проблемы с фильтрацией или трекингом

3. **Face tracking:**
   - Компонент использует IoU-based tracking для множественных лиц
   - `primary_tracking_id` показывает ID основного лица
   - `primary_valid` показывает, найдено ли основное лицо после фильтров
