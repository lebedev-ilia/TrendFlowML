# Отчёт: core_clip — ✅ ЗАШТАМПОВАН (2026-07-05)

Автор: Claude (автономная сессия). GPU-под A4500. **CLIP-хаб проекта** — базовый визуальный
эмбеддинг + text-эмбеддинги промптов для многих downstream (scene/shot_quality/cut_detection/
similarity/…). Прогон inprocess (ViT-B/32, `clip.load`) — Triton не нужен.

## Выход (валидирован)
- **`frame_embeddings (N, 512)` L2-норм, finite** — базовый визуальный токен. Переиспользуется как
  scene-эмбеддинг (scene_classification), для CLIP-quality (shot_quality), сегментации (cut_detection),
  similarity и др.
- **Text-эмбеддинги промптов:** `places365_text_embeddings (365,512)`, `scene_aesthetic/luxury/
  atmosphere (6,512)`, `shot_quality`, `cut_detection_transition`, `popularity_topic (10,512)` — zero-shot
  базы для downstream (не пересчитывать CLIP в каждом компоненте — один раз здесь).
- **Proxy-фичи (аналитика/бэкенд):** `consecutive_cosine_prev`, `*_scores` (shot/scene/atmosphere/
  popularity), `places365_topk_indices/scores` (кадровые + video-level).

## Оценка по 4 осям
- **Корректность:** ✅ выходной валидатор (`validate_core_clip_npz.py`) — VALID schema; эмбеддинги
  L2=1.0, 100% finite.
- **Стабильность:** ✅ **golden идентичен** — 2 прогона на тех же кадрах → `frame_embeddings` побитово
  совпадают (max diff 0.0). CLIP детерминирован.
- **Различимость:** эмбеддинги 512-d L2 — стандартное CLIP-пространство; text-эмбеддинги в том же
  пространстве (косинус осмыслен). Подтверждено downstream (scene дал осмысленные сцены на этих эмбеддингах).
- **Модель-fit:** ✅ `frame_embeddings` — базовый seq-токен для Encoder И источник эмбеддингов для
  других компонентов (переиспользование — ключевая оптимизация цепочек).

## Оптимизация
Inprocess ViT-B/32, batch 16 — ~14–34 c на видео (зависит от N кадров); Triton-путь (CLIP ONNX) есть,
но inprocess проще/достаточен. Ключ для 200k: **эмбеддинги считаются ОДИН раз** и переиспользуются
всеми потребителями (scene/shot_quality/cut_detection/similarity) — не дублировать CLIP-инференс.
