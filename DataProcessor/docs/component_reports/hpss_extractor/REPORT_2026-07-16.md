## 6. Баги (найдены и исправлены)

### Bug-1: hpss_separation_quality ломается при margin≥2
**Причина:** librosa.decompose.hpss при margin>1 масштабирует маски → H+P > S_mag → `residual = S_mag - H - P < 0` → `residual² >> S_mag²` → `separation_quality << 0` (было до -3.9e15).  
**Фикс:** `np.clip(1.0 - residual_energy/energy_total, 0.0, 1.0)` в `main.py` `_compute_hpss_metrics()`.  
**Эффект:** Старые NPZ — исторические артефакты; новые прогоны всегда в [0,1].

### Bug-2: Empty-path feature_values = 8 NaN
**Причина:** `save_hpss_npz` всегда вызывал `add("sample_rate", None)` и т.д. → as_float(None) = NaN в feature_values.  
**Фикс:** Условный add: `if _mv is not None: add(_mk, _mv)` в `npz_savers/hpss.py`.  
**Эффект:** Empty-path: F=0, feature_names=[], feature_values=[].

### Bug-3: validate_structure() ложная WARN для empty NPZ
**Причина:** проверка `if n0 == 0: out.append(...)` не различала empty-статус.  
**Фикс:** `if n0 == 0 and status != "empty":` в `validate_hpss.py`.  
**Эффект:** 17/17 PASS (включая empty).
