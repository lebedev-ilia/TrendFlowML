# Отчёт: shot_quality — сессия 2026-07-05 (история доводки)

Автор: Claude (автономная сессия). GPU-под RunPod A4500. Static-review — `RUN_SPEC.md`.
Самая тяжёлая цепочка проекта — **5 hard-deps** с обязательным выравниванием frame_indices.

## Логика (разобрано + решения)
- **Сам shot_quality — numpy/CPU** (нет своей GPU-модели). Фичи покадрово: sharpness (tenengrad/
  laplacian/smd2), exposure (under/over/skewness), contrast (global/local/dynamic/clarity/microcontrast),
  lens (drop-off/glare), **depth** (mean/std/grad из core_depth_midas), CLIP-quality (промпты core_clip),
  face-ROI (из core_face_landmarks; пустота → NaN by design). + shot-агрегации по границам cut_detection.
- **Model-fit (решение):** `frame_features (N,F)` — **готовый seq-токен качества для Encoder**, отдельный
  эмбеддинг НЕ нужен (в отличие от scene/AR): вектор технических фич и есть представление. Face-ROI
  фичи = NaN by design (не «сломано»). shot-агрегации + CLIP-quality — аналитика.
- **Зависимости (5, hard, ОДИНАКОВЫЕ frame_indices):** core_clip, core_depth_midas, core_object_detections,
  core_face_landmarks, cut_detection. `_ensure_same_indices` → Segmenter aligned sampling group обязателен.

## Инфра (сделано)
- **Входной валидатор** `validate_shot_quality_input.py`: наличие+форма 5 deps, depth_maps (N,H,W),
  face-пустота допустима. **Контракты** (5 deps→/→Models) — в `COMPONENT_CONTRACTS.md`.

## Прогон — обход Triton (как в scene)
Блокер: **core_depth_midas — Triton-only (MiDaS)**. Обход: `scripts/midas_depth_inprocess.py`
(torch.hub MiDaS_small → `core_depth_midas/depth.npz` формата shot_quality, на тех же frame_indices).
Раннер `scripts/run_shot_quality_local.py`: Segmenter(shot-профиль) → core_clip(inprocess) +
object_detections(ultralytics) + face_landmarks(mediapipe) + cut_detection(farneback,no-clip) +
depth(bypass) → shot_quality → валидаторы. **Всё inprocess, без Triton.**

**Прогон на GPU (2026-07-05) — 5 из 6 компонентов ✅:** Segmenter 45c, core_clip inprocess 34c,
core_object_detections 57c, cut_detection farneback 137c (узкое место), **core_depth_midas
inprocess-обход (MiDaS) 44c ✅ — Triton обойдён!** Блокер — **только core_face_landmarks**:
`mediapipe==0.10.35` удалил `mp.solutions` (компонент на старом API) → `AttributeError: module
'mediapipe' has no attribute 'solutions'`. Фикс: `pip install "mediapipe<0.10.15"` ИЛИ порт компонента
на mediapipe Tasks API. После этого — face + shot_quality (numpy) поверх УЖЕ посчитанных deps
(кэшированы в rs/ на Network Volume) → ~1 мин. Тайминги/детали — ledger.
