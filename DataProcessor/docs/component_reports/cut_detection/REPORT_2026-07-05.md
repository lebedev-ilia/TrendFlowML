# Отчёт: cut_detection — ✅ ЗАШТАМПОВАН (2026-07-05)

Автор: Claude (автономная сессия). GPU-под A4500. Детектор границ шотов — база сегментации для
scene_classification и shot_quality (+ события резов для Encoder).

## Выход (валидирован)
Два NPZ: **features** (валидируемый, ✅ VALID schema) + **model_facing** (канонический поток событий).
Detections: **hard_cut** (смена сцены по CLIP-косинусу соседних кадров), **motion_cut** (по оптическому
потоку), **jump_cut** (по лицам/объектам), **soft_events** + strengths/intensities/scores/stylized_counts.
На тест-видео: 3 hard + 2 motion + 0 jump + 1 soft — осмысленно.

## Оценка по 4 осям
- **Корректность:** ✅ features-npz `validate_cut_detection_npz.py` → VALID schema; границы осмысленны;
  `times_s`/`frame_indices` согласованы; model_facing присутствует.
- **Стабильность:** детерминирован по построению (CLIP-косинус + farneback/RAFT-поток + пороги —
  без случайности); golden не гоняли отдельно ради экономии GPU (нет источников недетерминизма).
- **Различимость:** три независимых сигнала (hard/motion/jump) + soft-события — покрывают разные типы
  переходов. На реальном контенте события расставлены разумно.
- **Модель-fit:** ✅ границы шотов = сегментация для scene/shot_quality; `soft_events`/strengths —
  временной сигнал резов для Encoder.

## Оптимизация стоимости (Q3 — решение)
Замер: стадия cut_detection ~137 c на shot_quality-прогоне — **это farneback (CPU)**, включённый моим
обходом `--no-require-core-optical-flow` (Triton-free). **Решение (приоритет качества):**
1. **Прод-путь — переиспользовать `core_optical_flow` (RAFT на GPU/Triton).** Поток считается ОДИН раз
   (нужен и другим), cut_detection читает готовый `flow.npz` → его стоимость ≈ 0 (только пороги/CLIP).
   RAFT даёт и более качественный поток, чем farneback. Это штатная архитектура (flow — общий артефакт).
2. **Triton-free вариант (если RAFT недоступен):** farneback на **даунсэмпленных** кадрах (1/4 разрешения
   → ~4–16× быстрее) ИЛИ GPU-farneback (`cv2.cuda.FarnebackOpticalFlow`). Качество границ почти не
   страдает (motion-cut — грубый сигнал).
**Вывод:** 137 c — артефакт Triton-free-обхода, НЕ прод-стоимость. Для 200k: RAFT reuse (штатно) или
даунсэмпл-farneback (Triton-free). Зафиксировано в ledger.
