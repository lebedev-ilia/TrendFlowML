## 2. Резюме

`embedding_stats_extractor` — **derived-компонент статистик эмбеддингов**: считает две вещи — (1) **дисперсию по
чанк-эмбеддингам транскрипта** (`l2_variance` + top-k per-dim variances) = «насколько плывёт содержание по ходу
видео» (консистентность vs разброс темы); (2) **энтропию/перплексию topic-распределения** (реюз `topic_probs` из
semantics_topics_keyphrases) = «фокус на одной теме vs размытость». Оба concept'а осмысленны, но **на реальных данных
оба мертвы:** variance посчитана **лишь у 1/22 видео** (21a279af, 3 чанка, l2=0.0052) — у остальных транскрипт = **1
чанк < min_chunks=2** → NaN; topic_entropy = **константа 1.6 / norm 0.996 / perplexity 4.97 у всех 21** (наследует
**stub-таксон** semantics_topics → почти равномерное softmax-распределение). Кросс-видео сигнала нет ни по одной оси.
Инженерия аккуратна (safe-relpath, dim/nan-inf флаги, ddof).

## 3. Функционал

Работает после chunk-эмбеддера и топик-компонента:

1. **Загрузка чанков** — chunk_embeddings_relpath транскрипта (приоритет whisper→youtube_auto; legacy-fallback).
2. **Variance across chunks** — `var(chunks, axis=0)` → l2-норма (`l2_variance`) + top-k наибольших per-dim
   дисперсий (`topvar_1..8`). Нужно ≥ `min_chunks_required` (=2) чанков.
3. **Topic entropy** — берёт `topic_probs` из `tp_artifacts.topics.topk_distribution` (softmax semantics_topics) →
   энтропия / norm / перплексия.
4. Пишет 39 tp_embstats_* (метрики + present/флаги/config).

**Зачем продукту:** l2_variance = **дрейф контента** (фокусированное видео vs скачет по темам — влияет на удержание).
topic_entropy = **тематический фокус** (нишевое vs размытое). Обе — оси «структурной цельности» контента.

## 4. Вход

- **chunk-эмбеддинги транскрипта** (`transcript_chunk_embedder`, whisper) — реальные, но **чаще 1 чанк** (короткие
  транскрипты) → <2 → variance не считается.
- **topic_probs** (`semantics_topics_keyphrases`) — **stub-таксон** → распределение почти равномерное.
- Нет входа → present=0 / entropy NaN.

## 5. Выход

- **39 tp_embstats_* фич:** n_chunks, dim, l2_variance, topvar_1..8, topic_entropy/norm/perplexity, present-флаги
  (present, topic_probs_present/invalid, topic_entropy_present), диагностика (dim_mismatch/nan_inf/unsafe_relpath/
  used_legacy), source_used (whisper/youtube_auto), config (min_chunks/topk/ddof/slots), тайминги.
- **NaN-политика:** <2 чанков → variance NaN, present=0; нет topic_probs → entropy NaN.

## 6. Фичи (важное/неочевидное)

- **l2_variance present=1 только у 1/22 (главное)** — 21a279af (3 чанка, l2=0.0052). У остальных whisper-транскрипт =
  **1 чанк** (`source_used_whisper=1`, но n_chunks<2) → variance не считается. Нет кросс-видео сигнала.
- **topic_entropy = 1.6 / 0.996 / 4.97 КОНСТАНТА у всех 21** — реюз stub-топиков (5 слотов, почти равномерный softmax
  → энтропия≈log 5=1.6, norm≈max 0.996). Мёртвая константа, наследует блокер semantics_topics.
- **topvar_1..8** — крупнейшие per-dim дисперсии (какие «оси смысла» гуляют сильнее); только у 1 видео.
- **Диагностические флаги все 0** — чисто (нет dim/nan_inf/unsafe).
- **min_chunks_required=2** — разумно (дисперсия <2 точек бессмысленна), но на коротких транскриптах = смерть variance.

## 7. Архитектура / алгоритм

- **Чистый numpy** — `np.var` по оси чанков + энтропия готовых probs. Своей модели нет.
- **Безопасность:** safe-join relpath (path-traversal guard), dim/nan-inf проверки.
- **Сложность:** тривиальная (variance O(n_chunks×dim)).
- **Детерминизм:** тривиально детерминирован.

## 8. Оптимизации

- **Reuse** chunk-эмбеддингов и topic_probs (ничего не пересчитывает).
- **ddof-параметр** variance (0/1) — контроль смещения.
- **Приоритет источников** + legacy-fallback (устойчивость к схеме).
- **Slot-clamp** top-k (стабильная схема).
- **emit_extra_metrics gate** — тайминги опционально.

## 9. Слабые места

- **variance мертва на коротких транскриптах (главное)** — 1 чанк < min_chunks=2 → 21/22 NaN. Нужны либо длинные
  транскрипты, либо более мелкая нарезка чанков.
- **topic_entropy — константа (наследует stub-таксон)** — semantics_topics на e2e_stub даёт равномерное
  распределение → энтропия одинакова у всех. Пока таксон фиктивен — сигнала нет.
- **Тонкая надстройка второго порядка** — статистики поверх уже-слабых upstream'ов; двойная зависимость.
- **l2_variance опаков** — единое число «разброса»; трудно интерпретировать без нормировки/референса.
- **Нет нормировки variance** — абсолют зависит от масштаба эмбеддингов; сравнимость между видео под вопросом.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Более мелкая нарезка чанков** (или снизить min_chunks) — чтобы variance считалась не у 1/22. Дрейф
   контента — ценный сигнал удержания, но нужен ≥2 чанка.
2. **[выс., блокер] Реальный таксон** (semantics_topics) — тогда topic_entropy станет реальным «фокусом темы», а не
   константой.
3. **[сред.] Нормировать l2_variance** (на dim / на масштаб) — сравнимость между видео.
4. **[низ.] Добавить temporal-структуру** — не только дисперсия, но и тренд/скачки по времени (где именно дрейф).

## 11. Рекомендации по архитектуре / связям

- **transcript_chunk_embedder → embedding_stats** — критично число чанков; согласовать нарезку.
- **semantics_topics → embedding_stats (topic_entropy)** — оживёт с реальным таксоном.
- **Объединить с другими derived-статистиками** (cosine_metrics, embedding_shift) в единый «структурный» блок текста.

## 13. Интерпретируемость

- **Потенциально хорошая** — «контент дрейфует/цельный», «видео про одну тему/размыто» — понятные структурные оси.
- **Сейчас нечего показывать** — variance у 1 видео, энтропия константа. После мелкой нарезки + реального таксона —
  осмысленный «структурный профиль» контента.

## 14. Польза для моделей

**Низкая.** l2_variance (дрейф контента) и topic_entropy (фокус темы) — правдоподобные структурные предикторы
удержания, но фактически мертвы: variance у 1/22 (короткие транскрипты), entropy — константа (stub-таксон). Модель
получает почти нулевой сигнал. Потенциал умеренный (после мелкой нарезки + реального таксона).

## 15. Польза для аналитиков

**Низкая.** «Ваш контент цельный/скачет по темам», «видео сфокусировано/размыто» — понятные инсайты структуры, но на
данных недоступны (1 variance-значение, константная энтропия). После апстрим-починок — умеренно полезная аналитика
структуры; сейчас — нечего показать.
