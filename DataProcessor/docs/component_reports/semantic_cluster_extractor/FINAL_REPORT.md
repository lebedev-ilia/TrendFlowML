## 2. Резюме

`semantic_cluster_extractor` — **кластеризация видео по семантической нише**: относит видео к 1 из **32
кластеров** через faiss (ближайший центроид) на основе эмбеддинга заголовка/описания, выдаёт `cluster_id`,
similarity к центроиду, distance, `margin_top2` (уверенность) + config/fallback-флаги (31 поле). Механизм солидный
(faiss, golden=0, similarity∈[−1,1], distance=1−sim). **Но наследует mock-заголовки:** `cluster_id`=[5,5,5,5,5,26]
— 5/6 видео → кластер 5 (mock-идентичные title), только -Q6fnPIy → кластер 26; при этом similarity **низкая
(0.137)** — mock-заголовок далёк от центроида. **Кластеры не размечены** (просто ID 5/26 — без семантики «gaming/
beauty»). Идея (позиционировать видео в пространстве ниш для сравнения/рекомендаций/cold-start) ценна, но на mock-
данных и без меток кластеров нереализована.

## 3. Функционал

Работает в TextProcessor, после title/description-эмбеддеров:

1. Берёт primary-эмбеддинг (source=title по умолчанию; fallback description/hashtag).
2. **faiss nearest-centroid** к 32 пред-построенным кластер-центроидам → `cluster_id`.
3. Метрики: `similarity` (косинус к центроиду), `distance`=1−sim, `margin_top2` (разрыв топ-1 vs топ-2 = уверенность).
4. Флаги: `dim_mismatch`, `fallback_used`, `unsafe_relpath`, source/config.

**Зачем продукту:** семантическая ниша/тема — **позиционирование контента**: к какому типу видео относится (по
заголовку/описанию), для сравнения с нишей, рекомендаций, cold-start (новое видео без истории → похожие по нише).

## 4. Вход

- **title-эмбеддинг** (hard, primary) + description/hashtag (fallback) — из соответствующих эмбеддеров (mock).
- **32-кластерная модель** (пред-построенные центроиды, faiss).

## 5. Выход

- **31 поле tp_semclust_*:** `id` (0–31), `similarity`, `distance`, `margin_top2`, `n_clusters`, `embedding_dim`,
  `present`, source (title/description/hashtag), backend_faiss, `fallback_used`, `dim_mismatch_flag`, config-флаги.
- **NaN-политика:** нет эмбеддинга → present=0; extra_block NaN by design (emit_extra=False).

## 6. Фичи (важное/неочевидное)

- **`cluster_id` [5,5,5,5,5,26] (наследует mock)** — 5/6 в кластере 5 (mock-идентичные заголовки), -Q6 в 26.
  На реальных заголовках были бы разные ниши.
- **`similarity`=0.137 (низкая)** — mock-заголовок далёк от назначенного центроида (0.137 косинус); слабая
  принадлежность. На реальных заголовках similarity была бы выше/осмысленнее.
- **`margin_top2` — уверенность** назначения (разрыв топ-1/топ-2 кластеров); хорошая идея для «уверенно ли в нише».
- **Кластеры НЕ размечены** — cluster_id=5 не несёт смысла без меток («что за кластер 5?»). Нужна интерпретация центроидов.
- **32 кластера** — грубая таксономия ниш (YouTube-ниш сотни); coarse.
- **Fallback-цепочка** (title→description→hashtag) — устойчивость к отсутствию источника.

## 7. Архитектура / алгоритм

- **faiss nearest-centroid** к 32 пред-построенным кластерам (kmeans-подобная модель) на 1024-d эмбеддинге.
- **Детерминизм:** golden max|Δ|=0.0 (семантика; compute_ms исключён).
- **Сложность:** faiss-поиск тривиален (32 центроида).

## 8. Оптимизации

- **faiss** — быстрый nearest-centroid.
- **Fallback title→description→hashtag** — устойчивость.
- **margin_top2** — дешёвая уверенность.
- **Детерминизм**, config-флаги наблюдаемости.

## 9. Слабые места

- **Наследует mock-заголовки (главное)** — 5/6 в одном кластере (mock-идентичны); реальная кластеризация не задействована.
- **Кластеры не размечены** — cluster_id без семантики → непонятно ни модели (просто категория), ни аналитику
  («кластер 5» = ?). Нужны метки/репрезентативные примеры центроидов.
- **Низкая similarity (0.137)** — даже назначенный кластер далёк; либо mock-заголовок нерелевантен модели, либо
  32 кластера грубы для покрытия.
- **32 кластера грубы** — YouTube-ниш сотни; coarse таксономия.
- **Зависит от качества эмбеддера + кластер-модели** — обе под вопросом (mock title, неизвестное качество кластеров).

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Реальные заголовки** — сейчас 5/6 в одном кластере (mock); без реальных title кластеризация фиктивна.
2. **[выс.] Разметить кластеры** — дать метки/репрезентативные примеры центроидам (иначе cluster_id бесполезен для интерпретации).
3. **[сред.] Больше кластеров / иерархия** — 32 грубо для ниш; рассмотреть 100+ или иерархическую таксономию.
4. **[сред.] Проверить качество кластер-модели** — на чём обучены центроиды, покрывают ли реальные ниши.
5. **[низ.] Использовать transcript-эмбеддинг** (реальный) как источник кластеризации вместо mock-title.

## 11. Рекомендации по архитектуре / связям

- **Кластеризовать по transcript** (реальные данные) вместо/вместе с title — устойчивее к mock-метаданным.
- **cluster_id → Fusion** как категориальная ниша-фича (после реальных данных + меток).
- **Разметка кластеров** — общий словарь ниш для рекомендаций/сравнения.

## 13. Интерпретируемость

- **Низкая** — cluster_id=5 без метки бессмыслен; margin/similarity — служебны.
- **Добавить:** метки кластеров («ваше видео в нише X»), репрезентативные примеры; тогда станет понятным позиционированием.

## 14. Польза для моделей

**Низкая-умеренная.** Кластер-ниша — потенциально полезная категориальная семантическая фича (позиционирование/
cold-start), но наследует mock-заголовки (5/6 один кластер) + низкая similarity + coarse 32 кластера. Как
one-hot/embedding ниши мог бы помочь, но на данных фиктивен. После реальных заголовков + качественной кластер-модели — умеренно полезен.

## 15. Польза для аналитиков

**Умеренная (потенциал).** «В какой нише ваше видео / похожие ниши» — ценный позиционирующий инсайт, но **только с
размеченными кластерами** (сейчас ID без смысла) и реальными заголовками (mock). После этого — полезно для
сравнения/позиционирования; сейчас непригодно.

  требует размеченных кластеров (сейчас cluster_id без смысла) и реальных заголовков (mock). Потенциал сильный
  (сравнение/рекомендации), факт — непригоден до реальных данных и меток.
