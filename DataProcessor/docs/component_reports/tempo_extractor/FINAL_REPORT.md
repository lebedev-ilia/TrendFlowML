## 2. Резюме

`tempo_extractor` — **детектор темпа (BPM)** через `librosa.beat.beat_track`: оценивает удары в минуту всего аудио
и по сегментам (`bpm_by_segment`), плюс mean/median/std и `tempo_confidence` (11 фич). **Полностью живой**
(все 6 ok, 0 NaN, golden diff=0.0), но с **серьёзным содержательным дефектом:** на **не-музыкальном/речевом
аудио librosa возвращает дефолтный ~129.199 BPM** (артефакт beat_track с прайором у 120), и ровно это видно на
данных — **3 речевых видео закреплены на 129.199** (одинаковый дефолт), различаются лишь музыкальный ролик
(-Q6fnPIy=152) и -4WRepA=117. Хуже: **`tempo_confidence` остаётся высокой (~0.9) даже на речи без бита** — не
отражает, есть ли реальный ритм → вводит в заблуждение. Различимость слабая (tempo_bpm CV 8.3%). Часть
музыкального под-семейства (chroma/key/tempo/onset), осмыслен только на музыке.

## 3. Функционал

Работает после Segmenter (аудио). `librosa.beat.beat_track`:

1. Onset-энергия → автокорреляция → **оценка BPM** (`tempo_bpm`) + `bpm_by_segment (N,)`.
2. Статистики: `tempo_bpm_mean/median/std`, `tempo_bpm_by_segment_mean/median/std`.
3. **`tempo_confidence`** — уверенность оценки темпа; `tempo_estimates`/`warnings`.

**Зачем продукту:** темп музыки — **сигнал энергии/динамики** контента: быстрый BPM (танцевальная/экшн-музыка)
↔ энергичность, медленный ↔ спокойствие. Для музыкальных/гейминг-видео это релевантный ритмический признак.

## 4. Вход

- **Аудио** (Segmenter) — нет аудио → `status=empty` (fv NaN×11).
- **beat_track-параметры** (hop, start_bpm-прайор).

## 5. Выход

- **11 табличных фич:** `tempo_bpm`, `tempo_bpm_mean/median/std`, `tempo_confidence`,
  `tempo_bpm_by_segment_mean/median/std`, `duration_sec`, `sample_rate`, `segments_count`.
- **Per-segment:** `bpm_by_segment (N,)`, `tempo_estimates`, `segment_*_sec`, `segment_mask`, `warnings`.
- **NaN-политика:** empty → fv NaN×11; сегмент без ритма → NaN by design.

## 6. Фичи (важное/неочевидное)

- **`tempo_bpm` — дефолт ~129.199 на речи (главный дефект).** librosa beat_track при отсутствии реального бита
  возвращает значение около прайора → **3 речевых видео = ровно 129.199** (и `tempo_bpm`==`tempo_bpm_median`).
  Реальный темп определяется только на музыке (-Q6fnPIy=152). То есть на речевом контенте BPM — артефакт, не сигнал.
- **`tempo_confidence` не отражает валидность** — ~0.86–0.92 **даже на речи без бита**; высокая уверенность на
  спурьёзном дефолте вводит в заблуждение. Должна падать при отсутствии ритма (напр. по onset-силе/периодичности).
- **`tempo_bpm_std` / `by_segment_std`** — вариативность темпа: осмысленна на музыке (rubato/смена темпа), шум на речи.
- **Слабая различимость** (tempo_bpm CV 8.3%) — из-за кластеризации на дефолте.
- **Согласуется с музыкальным семейством** — -Q6fnPIy (152 BPM) = тот же музыкальный ролик, что chroma-тональный/
  mel-энергичный; на нём tempo реален.

## 7. Архитектура / алгоритм

- **librosa `beat.beat_track`** (onset-envelope → tempo autocorrelation, прайор ~120 BPM), агрегация (numpy).
- **Сложность:** дёшев, CPU.
- **Детерминизм:** golden max|Δ|=0.0 (librosa CPU без стохастики, 10 прогонов).

## 8. Оптимизации

- **Чистый librosa** — без моделей/GPU.
- **Per-segment BPM** + агрегаты — темп-профиль во времени.
- **tempo_estimates/warnings** — задел под диагностику (но confidence не используется как gate).

## 9. Слабые места

- **Спурьёзный дефолт-BPM на речи (главное).** На не-музыкальном аудио beat_track возвращает ~129.199 → 3/6 видео
  на одном дефолте; BPM бессмыслен на речевом контенте (доминирует в корпусе). Фактически полезен только на музыке.
- **`tempo_confidence` вводит в заблуждение** — высокая (~0.9) даже при отсутствии бита; не служит gate'ом «есть
  ли ритм». Downstream/аналитик может принять дефолт-129 за реальный темп.
- **Слабая различимость** (CV 8.3%) — прямое следствие дефолт-кластеризации.
- **Нет «музыкальность»-gate** — стоило бы не выдавать BPM (или NaN) при beat-less аудио (можно из chroma_entropy/
  onset-силы).
- **Пересечение с onset_extractor/rhythmic** — ритмическое под-семейство, дубли.
- **Только аудио, смешивает источники** — темп музыки на фоне речи размывается.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Gate «есть ли ритм»** — при низкой onset-периодичности/музыкальности (chroma_entropy высок) выдавать
   `tempo_bpm=NaN`/флаг «нет бита» вместо дефолт-129; сейчас спурьёзный BPM на всей речи.
2. **[выс.] Починить `tempo_confidence`** — привязать к реальной силе/периодичности бита, а не держать ~0.9;
   сейчас не отражает валидность.
3. **[сред.] Консолидировать с onset/rhythmic** — общий beat-анализ.
4. **[низ.] Октавные ошибки BPM** — beat_track часто путает half/double tempo; добавить коррекцию.

## 11. Рекомендации по архитектуре / связям

- **Ритмическое под-семейство** (tempo/onset/rhythmic) + **музыкальность-gate** из chroma — единый beat-модуль.
- **Связка chroma+tempo** = «музыкальность + темп»: не выдавать темп там, где нет тональности/бита.
- **CLAP** для семантики музыки; tempo — числовой ритм на верифицированной музыке.

## 13. Интерпретируемость

- **Хорошая на музыке:** «темп 152 BPM» понятно. На речи — вводит в заблуждение (129 не реален).
- **Добавить:** флаг «нет бита/не музыка» → не показывать BPM; «энергичный/спокойный темп» из верифицированного BPM.

## 14. Польза для моделей

**Умеренная (только на музыке).** Темп — реальный сигнал энергии для музыкального контента, но на речевом (доминирует)
BPM спурьёзен (дефолт-129) и confidence не отличает валидное от мусора → модель получит шум на большинстве данных.
Полезен при наличии музыкальности-gate; без него — слабый/обманчивый.

## 15. Польза для аналитиков

**Умеренная (нишевая).** «Темп музыки N BPM» понятен и полезен музыкальным/танцевальным креаторам. Но на речевом
контенте показывает спурьёзный 129 с высокой уверенностью — риск дезинформации. Нужен gate «есть ли музыка».

### Итоговые оценки

- **Польза для моделей: 3/5.** Темп/BPM — реальный сигнал энергии для музыкального контента (быстрый ↔ динамика).
  Но на речевом (доминирует в корпусе) BPM спурьёзен (librosa-дефолт ~129), а `tempo_confidence` не отличает
  валидное от мусора — модель получает шум без музыкальность-gate. Полезен лишь с фильтром «есть ли бит».
- **Польза для аналитиков: 3/5.** «Темп музыки N BPM» — понятный полезный инсайт для музыкальных/танцевальных
  креаторов. Ограничивают спурьёзный дефолт-129 с обманчиво высокой уверенностью на речи (риск дезинформации),
  слабая различимость и нишевость (нужен gate музыкальности).
