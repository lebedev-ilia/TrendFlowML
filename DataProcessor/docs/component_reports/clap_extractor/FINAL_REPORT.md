## 2. Резюме

`clap_extractor` — **семантический аудио-хаб**: прогоняет **LAION-CLAP** по аудио-сегментам и выдаёт 512-мерные
контрастные аудио-эмбеддинги — прямой аудио-аналог `core_clip` (визуального CLIP). Отдаёт per-segment
`embedding_sequence (N,512)` (seq-токен для Encoder) + video-level pooled `embedding (512)` + маску сегментов +
5 табличных фич. **Это один из самых здоровых компонентов на реальных данных:** все 6 видео `status=ok`, 512-d,
L2-нормы 0.80–0.98, `embedding_sequence` выровнен по сегментам, **0 NaN**, golden Δ=0 (CUDA autocast float32 +
inference_mode). CLAP кодирует «что звучит» семантически (музыка/речь/шумы, жанр, настроение) — фундамент аудио-
ветки Encoder и кросс-модального сравнения. 2 бага исправлены (validate empty-путь, PyTorch 2.12 `weights_only`).

## 3. Функционал

Работает после Segmenter (аудио-сегменты). Пайплайн:

1. Берёт аудио-сегменты (start/end/center, sample_rate 48kHz).
2. Прогоняет **LAION-CLAP audio encoder** по каждому сегменту → 512-d эмбеддинг.
3. Собирает `embedding_sequence (N,512)` (seq по сегментам) + `segment_mask`/`segment_*_sec` (ось).
4. Пулит video-level `embedding (512)`; считает `clap_norm`, `clap_magnitude_mean/std`, `segments_count`.

**Зачем продукту:** CLAP — **семантическое представление звука** (обучено выравнивать аудио с текстовыми
описаниями). Это «о чём звучит видео»: музыка vs речь vs эффекты, жанр/настроение музыки, тип звуковой дорожки.
Фундаментальный вход для аудио-ветки Encoder (аудио-семантический токен, как core_clip для видео) и для кросс-
модального сравнения (`similarity_metrics.audio_clap`, `high_level_semantic`).

## 4. Вход

- **Аудио-сегменты** (Segmenter `segments.json`) — окна с start/end/center; нет аудио → `status=empty`
  (`audio_missing_or_extract_failed`); ошибка загрузки → `status=error`.
- **LAION-CLAP checkpoint** (dp_models, доверенный локальный источник).
- **Sample rate 48kHz** (CLAP-контракт).

## 5. Выход

- **Model-facing seq:** `embedding_sequence (N,512)` — CLAP по сегментам (готовый seq-токен для Encoder).
- **Video-level:** `embedding (512)` (pooled), `embedding_present` (флаг).
- **Ось сегментов:** `segment_start/center/end_sec`, `segment_mask`, `segment_embedding_norm`.
- **Табличные (5):** `embedding_dim`, `clap_norm`, `clap_magnitude_mean/std`, `segments_count`.
- **NaN-политика:** masked/empty сегменты → NaN by design; error/empty → `embedding_present=False`, fv NaN×5.

## 6. Фичи (важное/неочевидное)

- **512-d CLAP-эмбеддинг = несущая ценность** — семантический аудио-вектор, не интерпретируемый напрямую, но
  богатый для Encoder (аудио-аналог penultimate/CLIP). Всё остальное (5 стат) вторично.
- **L2-нормы 0.80–0.98 (не строго 1.0)** — CLAP не отдаёт unit-норму; на данных mean≈0.93. Интересно: короткое
  видео (5 сегментов) → norm 0.98, длинное (30 сегм.) → 0.80 — вероятно эффект пулинга (усреднение разнородных
  сегментов снижает норму). Downstream должен L2-нормировать при использовании как similarity.
- **`embedding_sequence` выровнен по сегментам** — каждый сегмент = один 512-вектор + временная ось
  (start/center/end) → Encoder получает аудио-семантику во времени (как CLIP-кадры).
- **Контрастная природа (audio-text)** — CLAP можно использовать zero-shot (описания звука текстом), но сейчас
  компонент отдаёт только эмбеддинг, без zero-shot аудио-меток (в отличие от core_clip zero-shot в scene).
- **48kHz sample rate** — CLAP-специфика; ресэмплинг сегментов обязателен.

## 7. Архитектура / алгоритм

- **LAION-CLAP** (HTSAT/audio encoder + projection → 512-d), предобученная, не дообучается. GPU-инференс.
- **Детерминизм:** golden max|Δ|=0.0 при CUDA `autocast(float32)` + `inference_mode` (важно: float32, не fp16,
  для побайтовой воспроизводимости).
- **Сложность:** CLAP-forward на сегмент (GPU); стоимость ∝ числу сегментов.

## 8. Оптимизации

- **Per-segment батчинг** CLAP-инференса.
- **autocast(float32) + inference_mode** — детерминизм + скорость без потери воспроизводимости.
- **Pooled video-embedding** отдельно от seq — компактный video-level токен.
- **`weights_only=False` monkey-patch** (баг-фикс) — совместимость laion_clap checkpoint с PyTorch 2.12.
- **segment_mask** — чистая семантика невалидных сегментов (NaN + маска).

## 9. Слабые места

- **Тонкая аналитика (5 фич)** — наружу только эмбеддинг + 5 стат; нет zero-shot аудио-меток (музыка/речь/жанр
  словами), хотя CLAP это позволяет. Для аналитика сырой эмбеддинг непонятен.
- **Нормы не unit** (0.80–0.98) + зависимость нормы от числа сегментов (пулинг) — downstream обязан L2-нормировать
  для similarity, иначе video-embedding систематически «слабее» на длинных видео.
- **Полностью зависит от аудио** — нет звука → empty/error (2+2 из 16 NPZ); немое видео сигнала не получает.
- **Не разделяет источники звука** — один эмбеддинг на сегмент смешивает музыку+речь+шум (разделение — в
  source_separation/hpss, отдельно).
- **48kHz + CLAP-инференс на GPU** — дороже классических аудио-фич (mel/mfcc CPU).
- **v1 схема** — без per-segment zero-shot, без video-level семантических меток.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Добавить zero-shot аудио-метки** — CLAP умеет матчить аудио с текстом; выдать «музыка/речь/тишина/
   аплодисменты/жанр» словами (как scene zero-shot) — резко поднимет аналитическую ценность.
2. **[сред.] L2-нормировать video-embedding** (или задекларировать) — устранить зависимость нормы от длины.
3. **[сред.] Проверить пулинг** — mean-pool сегментов снижает норму; рассмотреть attention/нормированный пул.
4. **[низ.] Явно связать с source_separation** — CLAP на разделённых дорожках (музыка/речь отдельно) точнее.
5. **[низ.] fp16-опция** для скорости прода (golden оставить float32).

## 11. Рекомендации по архитектуре / связям

- **Аудио-аналог core_clip** — закрепить как единый источник аудио-эмбеддингов для всех downstream (similarity_
  metrics.audio_clap, high_level_semantic audio, Encoder) — не считать CLAP дважды.
- **Согласовать нормировку** с потребителями (similarity ожидает unit-норму).
- **Единый семантический слой** (CLIP + CLAP) для кросс-модального сравнения видео.

## 13. Интерпретируемость

- **Сейчас низкая** — сырой 512-d эмбеддинг + 5 стат непонятны аналитику.
- **Добавить (высокий приоритет):** zero-shot CLAP-метки словами — «преимущественно музыка / речь / тишина»,
  жанр/настроение музыки; timeline типа звука по сегментам. CLAP это позволяет из коробки.

## 14. Польза для моделей

**Высокая.** CLAP-эмбеддинг — **фундаментальный аудио-семантический токен** для Encoder (аудио-ветка), прямой
аналог core_clip: `embedding_sequence (N,512)` — готовый dense seq по времени, чистый (0 NaN), детерминированный,
различимый. Аудио-семантика (музыка/речь/настроение) правдоподобно сильна для предсказания вовлечённости (тип
саундтрека ↔ удержание). Одна из самых информативных аудио-фич — наравне с core_clip по роли.

## 15. Польза для аналитиков

**Умеренная (по факту), высокая (потенциал).** Сырой эмбеддинг аналитику не показать; наружу только 5 стат
(норма/магнитуда/число сегментов) — тонко. Но CLAP **позволяет zero-shot** («какой звук в видео») — после
добавления меток станет наглядным «музыка/речь/жанр». Сейчас косвенно ценен через аудио-similarity (сравнение
звучания видео).
