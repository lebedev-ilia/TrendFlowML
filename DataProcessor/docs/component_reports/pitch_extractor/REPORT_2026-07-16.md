## Найденные и исправленные баги (3 штуки)

### Баг 1: pitch_skewness/kurtosis = NaN при std(f0)=0 (run_segments)
- **Файл:** `main.py:614–620`
- **Причина:** scipy.stats.skew/kurtosis возвращает NaN при нулевой дисперсии (все сегменты одинаковый f0_mean), нет guard
- **Симптом:** 10/15 ok-NPZ в storage имели NaN в pitch_skewness/pitch_kurtosis
- **Фикс:** `sk = stats.skew(f0_arr); features["pitch_skewness"] = float(sk) if np.isfinite(sk) else 0.0`
- **Примечание:** В `_calc_additional_metrics()` (для run()) guard `if std > 0` уже был — рассинхрон исправлен

### Баг 2: _validate_output() consistency check без eps
- **Файл:** `main.py:235–240`
- **Причина:** строгое `f0_min <= f0_mean <= f0_max` нарушается при float32 рounding при агрегации одинаковых значений (np.mean vs np.min/max)
- **Симптом:** voiced 220 Hz синтетика → ValidationError "f0_min > f0_mean" на ровном pitch
- **Фикс:** добавлен `_eps=1e-3` → `f0_min - _eps <= f0_mean <= f0_max + _eps`

### Баг 3: YIN возвращает out-of-range f0 на тихом/нулевом сигнале
- **Файл:** `main.py:839–840`
- **Причина:** librosa.yin на нулевом сигнале → f0=2004.5 Hz (artéfact выше fmax=2000). Фильтрация только NaN, не диапазона
- **Симптом:** нулевой сигнал → YIN voiced=True с f0>fmax → финальный f0_mean=2004 → ValidationError в run_segments
- **Фикс:** добавлена фильтрация `& (f0 >= fmin) & (f0 <= fmax)` для обоих PYIN и YIN
