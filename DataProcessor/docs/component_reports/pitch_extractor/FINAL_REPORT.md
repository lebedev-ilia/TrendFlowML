## 2. Резюме

`pitch_extractor` — **трекер высоты тона (F0)** через librosa PYIN: оценивает основную частоту голоса/тона и её
статистики — `f0_mean/std/min/max/median`, гладкость контура, число скачков высоты, skew/kurtosis, октавное
распределение (16 фич). **Живой и различимый:** все 6 видео ok, f0_mean 78–441 Hz (CV=35%) осмысленно разделяет
глубокий голос (-4WRepA=78 Hz), высокий (-4RHVBIik=286 Hz) и выдержанный музыкальный тон (-Q6fnPIy=441 Hz, std=0 =
чистая нота ≈A4). `f0_std` — **выразительность подачи** (монотонность vs интонационный размах). golden diff=0.0
(PYIN детерминирован). **Оговорка данных:** видео с выдержанным тоном (std=0) имеет **2 pre-fix NaN** в skew/kurt —
фикс (0.0 при std=0) закоммичен, но этот storage-NPZ пре-фикс. Полезный голосовой/мелодический сигнал.

## 3. Функционал

Работает после Segmenter (аудио). librosa PYIN:

1. **PYIN F0-трекинг** → контур высоты во времени → `f0_mean/std/min/max/median`.
2. **Динамика контура:** `pitch_contour_smoothness` (плавность интонации), `pitch_jump_count` (резкие скачки высоты).
3. **Распределение:** `pitch_skewness`, `pitch_kurtosis`, `pitch_octave_distribution` (доли по октавам, сумма≈1).

**Зачем продукту:** высота голоса — **характер спикера и выразительность**: низкий/высокий голос (пол/возраст-
прокси), интонационный размах (`f0_std`: монотонная скучная подача vs живая выразительная) — **прямой фактор
удержания** (монотонность утомляет). Для музыки — мелодия/нота. Скачки высоты = эмоциональность/акценты.

## 4. Вход

- **Аудио** (Segmenter) — нулевой/невокализованный сигнал → `status=empty` (`pitch_all_segments_empty`).
- **PYIN-параметры** (`fmin`, `fmax`, `frame_length`, `hop_length`).

## 5. Выход

- **16 табличных фич:** `f0_mean/std/min/max/median`, `pitch_contour_smoothness`, `pitch_jump_count`,
  `pitch_skewness`, `pitch_kurtosis` + мета.
- **`pitch_octave_distribution`** (доли по октавам, ∑≈1). `segment_*_sec`, `segment_mask`.
- **NaN-политика:** empty → NaN; skew/kurt при std=0 → 0.0 (после фикса; в pre-fix NPZ = NaN).

## 6. Фичи (важное/неочевидное)

- **`f0_mean` — характер голоса (различим, CV 35%)** — 78 Hz (глубокий мужской/бас) … 286 Hz (высокий) … 441 Hz
  (выдержанная нота). Прокси пола/возраста спикера или мелодии.
- **`f0_std` — выразительность (несущий инсайт)** — размах интонации: -4WRepA std=13.5 (монотонно), -8WeWWOp
  std=108.8 (очень выразительно). **Монотонность vs живость подачи** — сильный сигнал удержания.
- **`pitch_jump_count` / `contour_smoothness`** — резкость/плавность интонации (эмоц. акценты vs ровная речь).
- **std=0 → выдержанный тон** — -Q6fnPIy f0_std=0 (одна нота 441 Hz = музыкальный тон, не голос); согласуется с
  chroma-тональным/mel-энергичным профилем этого ролика.
- **`pitch_skewness/kurtosis` — pre-fix NaN при std=0** — на выдержанном тоне (std=0) старый код давал NaN; фикс
  (0.0) есть, но storage-NPZ пре-фикс → 2 NaN у -Q6fnPIy. Мелкий data-gap.
- **`pitch_octave_distribution`** — распределение по октавам (∑≈1) — профиль диапазона.

## 7. Архитектура / алгоритм

- **librosa PYIN** (probabilistic YIN — робастный F0-трекинг с voiced/unvoiced), статистики (numpy/scipy).
- **Сложность:** PYIN дороже простых DSP, но CPU-приемлемо на коротком аудио.
- **Детерминизм:** golden max|Δ|=**0.0** (PYIN детерминирован, 3 прогона).

## 8. Оптимизации

- **PYIN** (робастнее autocorrelation/YIN) — качественный F0 с voiced-детекцией.
- **skew/kurt guard при std=0** (фикс) — 0.0 вместо NaN.
- **Октавное распределение** — компактный профиль диапазона.
- **Empty-путь** `pitch_all_segments_empty` — честный маркер невокализованного аудио.

## 9. Слабые места

- **Pre-fix NaN в storage** — skew/kurt=NaN на выдержанном тоне (std=0); фикс закоммичен, но батч не пере-прогнан
  (2 NaN у -Q6fnPIy). Мелко, но пример «фикс есть, данные старые».
- **Смешивает голос и музыку** — F0 речи (78–286 Hz) и музыкальной ноты (441 Hz) в одной шкале; без разделения
  источников «pitch» неоднозначен (голос vs мелодия).
- **Одна концептуальная ось** (высота) + производные; skew/kurt/octave частично коррелируют/опаковы.
- **Зависит от вокализованности** — на не-речи/не-мелодии (шум/перкуссия) F0 неопределён → empty/шум.
- **PYIN октавные ошибки** — возможны half/double F0 (типично для pitch-трекеров).

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[сред.] Пере-прогнать storage с skew/kurt-фиксом** — устранить 2 pre-fix NaN на выдержанных тонах.
2. **[сред.] Флаг «голос vs тон»** — разделять речевой F0 от музыкальной ноты (по std/вокализованности), не мешать шкалы.
3. **[сред.] `f0_std` как «выразительность подачи»** — actionable-инсайт (монотонно/живо), сильный для удержания.
4. **[низ.] Октавная коррекция PYIN** — фикс half/double ошибок.
5. **[низ.] Считать на разделённой речи** (source_separation) — чище F0 голоса.

## 11. Рекомендации по архитектуре / связям

- **Связка с voice_quality/speech_analysis** — F0 + качество голоса + темп речи = полный вокальный профиль.
- **Связка с emotion (аудио)** — интонационный размах/скачки ↔ эмоциональность.
- **Разделение голос/музыка** через source_separation перед pitch.

- **Хорошая:** «глубокий/высокий голос» (f0_mean), «монотонная vs выразительная подача» (f0_std) — понятно и actionable.

## 14. Польза для моделей

**Умеренная.** F0/pitch — характер голоса + **выразительность** (`f0_std` = монотонность↔живость) — правдоподобный
фактор удержания, различимый (CV 35%). Прокси пола/возраста спикера. Ограничивают: одна ось (высота), смешение
голос/музыка, pre-fix NaN, зависимость от вокализованности. Полезная вокальная ось.

## 15. Польза для аналитиков

**Умеренно-хорошая.** «Глубокий/высокий голос», «монотонная vs выразительная подача» — понятные, actionable инсайты
(особенно `f0_std` — совет «разнообразьте интонацию»). Ограничивают смешение голос/музыка и опаковость skew/kurt/octave.
