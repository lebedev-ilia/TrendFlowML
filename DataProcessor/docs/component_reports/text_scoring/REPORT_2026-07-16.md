При исключении во время загрузки `optical_flow.npz` функция сразу возвращала `None, "optical_flow_error"`, не пробуя fallback на `core_optical_flow`.  
**Фикс:** `except Exception: return None, "optical_flow_error"` → `except Exception: pass` (fallback продолжается к блоку 2).  
**Подтверждение:** тест `fallback_motion_signal` PASS — при broken optical_flow.npz + валидном core_optical_flow → motion signal загружен из fallback.
