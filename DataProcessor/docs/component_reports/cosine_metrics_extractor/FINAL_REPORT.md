## 2. Резюме

`cosine_metrics_extractor` — **derived-компонент кросс-косинусов**: считает 5 косинусных близостей между эмбеддингами
частей текста — **title↔desc, title↔transcript, desc↔transcript, transcript↔comments (mean/median)**. Своей модели
нет — потребляет готовые эмбеддинги. **Ключевой продуктовый смысл: title↔transcript = «соответствие заголовка
контенту» — прямой прокси кликбейта/обмана** (низкое = заголовок не про то, что в видео). На **реальных данных
частично живой**: у **5/22 видео есть whisper-транскрипт** → все 5 метрик считаются и **реально варьируются**
(title↔transcript 0.73–0.82, transcript↔comments 0.79–0.87); у **17/22 транскрипта нет** → только title↔desc
(mock-константа 0.853). Ограничение: **title/desc/comments — mock**, поэтому вариация идёт только со стороны реального
транскрипта, а абсолютные значения полу-фиктивны. Механизм корректен и аккуратен (dim/zero-norm/unsafe-relpath флаги,
NaN by design).

## 3. Функционал

Работает в TextProcessor после эмбеддеров/агрегаторов:

1. Достаёт relpath'ы эмбеддингов из `doc.tp_artifacts` (title/desc/transcript_agg_mean/comments_agg).
2. **Транскрипт-источник по приоритету** (whisper→youtube_auto→combined).
3. Считает 5 косинусов (безопасно: dim-check, L2-norm-check).
4. **comments-режимы:** `aggregates` (косинус к mean/median-вектору) или `matrix` (косинус к каждому комментарию →
   mean/median/std/p95).
5. Пишет 39 tp_cos_* (метрики + present/enabled/empty/mismatch-флаги + источник + тайминги).

**Зачем продукту:** кросс-косинусы = **согласованность модальностей текста**. title↔transcript — «обещает ли заголовок
то, что в видео» (кликбейт-детектор). desc↔transcript — точность описания. transcript↔comments — обсуждает ли
аудитория сам контент. Прямые интерпретируемые сигналы честности/качества.

## 4. Вход

- **Эмбеддинги** из `doc.tp_artifacts`: title/desc (bge, **mock**), transcript_agg_mean (e5, **реальный**, 5/22),
  comments_agg (**mock**).
- Отсутствует операнд → соответствующий косинус NaN (by design).

## 5. Выход

- **39 tp_cos_* фич:** 5 метрик (title_desc, title_transcript, desc_transcript, transcript_comments_mean/median),
  present-флаги (title/desc/transcript/comments), enabled/require/empty-флаги, mismatch (dim/pair/tc/zero-norm/
  unsafe-relpath), transcript_agg_source (whisper/youtube_auto/combined one-hot), тайминги, matrix-статы
  (n_comments/sims_std/sims_p95, только при emit_extra_metrics).
- **NaN-политика:** нет операнда → NaN; dim-mismatch/zero-norm → NaN + флаг.

## 6. Фичи (важное/неочевидное)

- **title↔transcript (главная)** — 0.73–0.82 на 5 реальных видео; **прокси «заголовок ↔ контент»** = кликбейт-
  сигнал. Варьируется (реальный транскрипт), но title-операнд mock → абсолют полу-фиктивен.
- **title↔desc = 0.841/0.853 (mock-константа)** — оба операнда mock → одинаково по группам; неинформативно сейчас.
- **transcript↔comments 0.79–0.87** — «обсуждает ли аудитория контент»; варьируется через транскрипт, comments mock.
- **17/22 без транскрипта** — transcript_present=0 → 4 из 5 метрик NaN; только title↔desc. Полкорпуса урезано.
- **matrix-режим (sims_std/p95)** — распределение близости комментариев к контенту (разброс мнений); задел, но
  emit_extra_metrics off → NaN.
- **Флаги согласованности** (dim/pair/tc/zero-norm/unsafe-relpath) — аккуратная диагностика, все 0 на корпусе (чисто).

## 7. Архитектура / алгоритм

- **Чистый numpy** — L2-нормированный dot между парами векторов; свой модели нет.
- **Безопасность:** dim-guard, zero-norm-guard, path-traversal-guard (relpath escapes artifacts_dir → флаг).
- **Сложность:** тривиальная (косинусы; matrix-режим O(n_comments)).
- **Детерминизм:** тривиально детерминирован.

## 8. Оптимизации

- **Reuse эмбеддингов** — не пересчитывает, берёт готовые (дёшево).
- **Приоритет источников транскрипта** (whisper→youtube→combined) — устойчивость.
- **Два comments-режима** (aggregates/matrix) — гибкость: агрегат дёшево / matrix детальнее.
- **Safe-join relpath** — защита от path traversal.
- **emit_extra_metrics gate** — тяжёлые статы (std/p95) опционально.

## 9. Слабые места

- **Операнды mock (главное)** — title/desc/comments mock → title↔desc константа, остальные метрики варьируются лишь
  со стороны реального транскрипта; абсолюты полу-фиктивны. Смысл «alignment» проявится только на реальном title/desc.
- **17/22 без транскрипта** — 4/5 метрик NaN на большинстве; зависит от наличия whisper-agg (transcript_aggregator).
- **Leakage (transcript↔comments)** — комментарии пост-публикация; для предсказания просмотров — с осторожностью.
- **Тонкая надстройка** — по сути 5 dot-произведений; ценность полностью зависит от качества входных эмбеддингов.
- **matrix-статы off** — sims_std/p95 (разброс реакции) не пишутся при emit_extra_metrics=false.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Реальные title/desc/comments** — тогда title↔transcript станет полноценным кликбейт-детектором (сейчас
   операнд mock).
2. **[выс.] Выделить title↔transcript как named-фичу «clickbait_gap»** — это самый ценный интерпретируемый сигнал;
   продвинуть в аналитику/модель явно.
3. **[сред.] Включить matrix-статы** (sims_std/p95) — разброс релевантности комментариев (споры vs согласие).
4. **[сред.] Leakage-политика** для transcript↔comments в view-предсказании.
5. **[низ.] Дозаполнить транскрипт** (17/22 без него) — зависит от ASR-wiring/transcript_aggregator.

## 11. Рекомендации по архитектуре / связям

- **transcript_aggregator → cosine_metrics** — ключевая связь; чем больше видео с whisper-agg, тем полнее метрики.
- **title↔transcript → Fusion как явный «честность заголовка»** — сильный, дешёвый, интерпретируемый признак.
- **Объединить с `title_to_hashtag_cosine_extractor`** — родственный косинус-компонент; единый «alignment-блок».

## 13. Интерпретируемость

- **Одна из лучших в text-секции** — «заголовок совпадает с контентом на 0.73» → понятно и actionable (кликбейт).
- **Добавить:** «ваш заголовок обещает X, а видео про Y — рассинхрон N%» (title↔transcript), «описание точно/неточно»
  (desc↔transcript), «аудитория обсуждает контент/офтоп» (transcript↔comments). Готовый золотой интерпретируемый слой.

## 14. Польза для моделей

**Умеренная (факт), высокая (потенциал).** title↔transcript (честность заголовка) и transcript↔comments
(релевантность реакции) — правдоподобные предикторы CTR/удержания. На реальном транскрипте метрики варьируются
(реальный сигнал), но title/desc/comments mock → абсолюты полу-фиктивны, а 17/22 без транскрипта. При реальных
операндах — сильный дешёвый признак.

## 15. Польза для аналитиков

**Умеренная-высокая (концепт золотой).** «Заголовок соответствует контенту?», «описание точное?», «аудитория обсуждает
то, что в видео?» — прямые, понятные, actionable инсайты честности/качества (кликбейт-риск). Работают где есть
транскрипт; ограничены mock-title (абсолют) и отсутствием транскрипта у большинства. После реальных данных — топовая
интерпретируемая аналитика.
