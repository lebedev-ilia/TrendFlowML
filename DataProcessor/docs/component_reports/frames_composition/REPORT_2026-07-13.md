## 2. Логические исправления (2026-07-13)

### БАГ #1: `neg_space_balance_lr = 0.0` при no_obj (ИСПРАВЛЕН)
- **Место:** `balance_composition.py::_bbox_stats_for_frame` early-exit
- **Было:** `return 0.0, 0.0, 0.0` — нет объектов → balance=0.0 (неверно)
- **Стало:** `return 0.0, 0.0, 1.0` — нет объектов → balance=1.0 (идеальный баланс)
- **Подтверждение:** `neg_space_balance_lr at no_obj: unique=[1.]` ✅

### БАГ #2: depth нормализация в style_probs (ИСПРАВЛЕН)
- **Место:** `balance_composition.py::_work`, строки ds/bokeh_proxy
- **Было:** `ds = _clip01(depth_std)` — depth_std в raw MiDaS единицах (~250) → всегда 1.0
  `bokeh_proxy = _clip01(p95 - p05)` — разница ~756 → всегда 1.0
- **Эффект:** `cinematic` имел завышенный вклад depth (0.35×1.0 + 0.20×1.0 = 0.55 базово), все видео получали style_dominant_id=1 (cinematic)
- **Стало:** `ds = _clip01(depth_std / max(depth_mean, 1e-6))` (CV, dimensionless ∈[0,1])
  `bokeh_proxy = _clip01((p95 - p05) / 1024.0)` (MiDaS scale ~1024)
- **Подтверждение:** ds_new=0.43..0.84 (было 1.0), bokeh=0.36..0.97 (было 1.0)
- **Влияние на style:** cinematic 0.326→0.286 (для vid1), minimalist 0.275→0.302; после фикса style_dominant_id=0 (minimalist) для vid3 vs 1 (cinematic) для vid1 → **различимость появилась**
