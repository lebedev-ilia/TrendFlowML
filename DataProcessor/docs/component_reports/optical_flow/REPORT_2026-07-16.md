# REPORT — optical_flow (module) (2026-07-16)

Компонент: `optical_flow` (VisualProcessor, consumer модуль). Потребляет `core_optical_flow/flow.npz` (RAFT),
производит per-frame матрицу (N,16) + 9 табличных агрегатов.
Schema `optical_flow_npz_v3`, producer_version **2.0.2**.
Hard dep (no-fallback): `core_optical_flow`.
Критерии — `CRITERIA.md` (согласованы 2026-07-14).

## Исправления (2026-07-16)

В ходе валидации найдены и исправлены 3 бага:

1. **БАГ раннера**: `run_optical_flow_module_local.py` использовал конфиг `visual_core_optical_flow_only.yaml`
   → Segmenter не создавал секцию `optical_flow` в metadata → `get_frame_indices` возвращал [] → `ValueError: Нет кадров`.
   **Фикс**: конфиг изменён на `visual_optical_flow_only.yaml` (включает `optical_flow: true`).

2. **БАГ empty-пути (frame_feature_names)**: при `core_optical_flow.status=empty` модуль не возвращал
   `frame_feature_names` / `frame_feature_values` → валидатор rc=1 («INVALID schema»).
   **Фикс**: в empty-ветке `optical_flow.py` добавлены все 16 имён + `(N,16) NaN` матрица.

3. **БАГ empty feature_names (неполный)**: empty-путь возвращал только 5 из 9 `feature_names`
   → validate --struct rc=2 («нет ожидаемых: cam_shake_std_mean, cam_rotation_abs_mean, …»).
   **Фикс**: список расширен до всех 9 (`_FIXED_FEATURE_NAMES`), feature_values = NaN×9 + missing_ratio=1.0.

**Доп. фикс инфра**: RunPod API сменил поле `dataCenterId` → `dataCenterIds` (массив). Исправлено
в `automation/runner/tools.py` (строка `_create_with_menu`).
