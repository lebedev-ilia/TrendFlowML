## 2. Резюме

`spectral_extractor` — **классический спектральный анализ аудio**: считает **46 DSP-дескрипторов тембра/текстуры**
(spectral centroid, bandwidth, flatness, rolloff, ZCR, contrast, slope — каждый с mean/std/min/max/median +
производные: ratio/entropy/stability) + per-segment массивы. **Полностью живой и различимый** на реальных данных:
все 6 видео ok, 0 NaN из 46 фич, golden diff **строго 0.0** (чистый numpy), багов не найдено. Дескрипторы
осмысленно различают тембр: centroid 460–2979 Hz (глухой/басовый → яркий), flatness 0–0.10 (тональный → шумный),
ZCR (озвонченность). Это **низкоуровневый интерпретируемый акустический профиль** — сильный вход для модели
(дополняет обучаемый CLAP), но опаковый для аналитика без перевода в «яркий/глухой/шумный».

## 3. Функционал

Работает после Segmenter (аудио). librosa-STFT → спектральные дескрипторы:

- **`spectral_centroid`** — «центр масс» спектра = **яркость** звука (высокий = яркий/резкий, низкий = глухой/басовый).
- **`spectral_bandwidth`** — ширина спектра вокруг центроида (богатство/разброс частот).
- **`spectral_flatness`** — тональность vs шумность (0 = чистый тон/музыка, →1 = шум/белый шум).
- **`spectral_rolloff`** — частота, ниже которой 85% энергии (граница ВЧ-содержания).
- **`zcr`** (zero-crossing rate) — частота смены знака = озвонченность/шумность (речь vs фрикативы/шум).
- **`spectral_contrast`** — контраст пик-впадина по частотным полосам (тембровая структура).
- **`spectral_slope`** — наклон спектра (баланс НЧ/ВЧ; `slope_stability` — устойчивость наклона).

**Зачем продукту:** тембр/текстура звука — **акустическая семантика**: различает музыку/речь/шум, жанр, характер
саундтрека, качество записи. Для модели — интерпретируемые акустические фичи (комплемент к CLAP-эмбеддингу).

## 4. Вход

- **Аудио** (Segmenter) — нет аудио → `status=empty` (`audio_missing_or_extract_failed`, fn=5, fv NaN).
- **STFT-параметры** (`n_fft`, `hop_length`, `sample_rate`).

## 5. Выход

- **46 фич** — 7 дескрипторов × 5 статистик (mean/std/min/max/median) + производные (`bandwidth_ratio`,
  `rolloff_ratio`, `flatness_entropy`, `contrast_variance`, `slope_stability`, `centroid_median_metric`) + 5 мета.
- **Per-segment массивы:** `centroid/bandwidth/flatness/rolloff/zcr/contrast/slope_mean_by_segment` + `segment_*_sec`, `segment_mask`.
- **NaN-политика:** empty → fn=5 (мета), fv[0-3]=NaN, segments_count=0.

## 6. Фичи (важное/неочевидное)

- **`spectral_centroid` — самая различимая** (460–2979 Hz): «яркость». -Q6fnPIy=461 (глухой/басовый), -8WeWWOp=2979
  (яркий). Прямой тембровый сигнал; коррелирует с rolloff (оба про ВЧ-содержание).
- **`spectral_flatness`** — тональность: -Q6fnPIy=0.0 (чистый тон, вероятно музыка/гул), -8WeWWOp=0.104 (более
  шумный). Различает музыку от речи/шума.
- **`slope_stability`, `flatness_entropy`** — производные «устойчивости/предсказуемости» спектра во времени
  (осмысленные, но неочевидные — стабильность тембра).
- **Сильная взаимокорреляция** — centroid↔rolloff↔bandwidth (все про ВЧ); 46 фич реально описывают ~7 осей ×
  статистики → высокая избыточность.
- **Опаковость для человека** — «centroid 2554 Hz» ничего не говорит креатору без перевода в «яркий/резкий звук».

## 7. Архитектура / алгоритм

- **librosa STFT** → спектральные дескрипторы (numpy), агрегация mean/std/percentiles.
- **Сложность:** O(samples·log) для STFT, дёшев, CPU.
- **Детерминизм:** golden max|Δ|=**0.0** (строго — в отличие от loudness 6.7e-8; спектральные редукции здесь
  полностью детерминированы на этих данных).

## 8. Оптимизации

- **Чистый librosa/numpy** — без моделей/GPU, дёшево.
- **Per-segment + agg** — две гранулярности.
- **Empty-путь с мета-only** (5 фич) — компактно.

## 9. Слабые места

- **Высокая избыточность (главное)** — 46 фич вокруг ~7 дескрипторов × 5 статистик; centroid/rolloff/bandwidth
  сильно коррелируют. Много почти-дублей → шум для модели, раздувание.
- **Опаковость для аналитика** — сырые Hz/безразмерные величины непонятны без перевода в качественные ярлыки.
- **Низкоуровневость** — тембр без семантики (не «это рок-музыка», а «яркий шумный спектр»); семантика — в CLAP.
- **Смешивает источники** — один спектр на речь+музыку+шум (разделение — в source_separation/hpss).
- **Зависит от качества аудио** — шум записи искажает дескрипторы.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[сред.] Проредить/сжать 46 фич** — убрать почти-дубли (centroid↔rolloff), оставить ядро или применить PCA;
   меньше шума/раздувания для Encoder.
2. **[сред.] Качественные ярлыки для аналитика** — перевести centroid/flatness в «яркий/глухой/чистый/шумный»
   (пороговая карта) — сделает опаковые Hz понятными.
3. **[низ.] Считать на разделённых дорожках** (source_separation) — тембр музыки vs речи отдельно, точнее.
4. **[низ.] Связать с CLAP** — низкоуровневые дескрипторы + обучаемый эмбеддинг = полный акустический профиль.

## 11. Рекомендации по архитектуре / связям

- **Общий STFT** с mel/mfcc/chroma-экстракторами — не считать спектр многократно (общий спектрограмм-кэш).
- **Reuse аудио-декода** с loudness/quality — единый буфер.
- **Часть единого аудио-технического профиля** (spectral + loudness + quality + band_energy).

## 13. Интерпретируемость

- **Низкая для человека** — сырые Hz/безразмерные не понятны.
- **Добавить:** перевод в ярлыки — «яркий/резкий звук» (высокий centroid), «глухой/басовый» (низкий), «чистый
  тон/музыка» (flatness→0), «шумный» (flatness↑); тогда станет наглядно.

## 14. Польза для моделей

**Умеренно-высокая.** 46 акустических дескрипторов — **интерпретируемые низкоуровневые фичи тембра/текстуры**,
дополняющие обучаемый CLAP-эмбеддинг; различимы, чисты (0 NaN), строго детерминированы — надёжный табличный вход
для baseline/Encoder. Ограничивает избыточность (коррелирующие фичи) и низкоуровневость (нет семантики). Хороший
рабочий набор, особенно для baseline-моделей, которым нужны tabular-фичи.

## 15. Польза для аналитиков

**Низкая (без перевода).** Сырые спектральные величины (centroid Hz, flatness) непонятны креатору. Различимы, но
не actionable как есть. После перевода в качественные ярлыки («яркий/глухой/чистый/шумный звук») — станут умеренно
полезны; сейчас — служебные.
