## Наблюдение: mfcc_mean ≈ 1e-8 (by design, не баг)

**Симптом**: mfcc_mean_by_segment = 1e-8 (≈0), mfcc_centroid≈0, mfcc_stability≈0.5015 на ВСЕХ 15 видео.

**Причина**: `enable_audio_normalization=True` → `normalize_audio(waveform)` (строка 617 main.py) делает RMS-нормализацию waveform к 1.0. После этого:
- log-power ≈ 0 (log(RMS²) ≈ 0)
- MFCC[k].mean() ≈ 0 by construction (логарифм единичной мощности)
- std(MFCC) ≈ 1 (т.к. только временная дисперсия вокруг нуля) → stability = 1/(1+1) = 0.5
- mean(abs(MFCC[0])) ≈ 0.7 (std/√(2/π) для N(0,1)) — это mfcc_energy

**Вывод**: Физически корректно. Дискриминативность обеспечивается через temporal arrays (`mfcc_mean_by_segment`, `delta_mean_by_segment`), а не scalar aggregates. Согласуется с Golden=0.0.
