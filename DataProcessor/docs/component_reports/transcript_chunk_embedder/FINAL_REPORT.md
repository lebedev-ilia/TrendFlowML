## 2. Резюме

`transcript_chunk_embedder` — **семантический эмбеддинг транскрипта речи** (что произнесено в видео): разбивает
Whisper-транскрипт на чанки и эмбеддит **multilingual-e5-large** (1024-d, L2) → `(N,1024)` чанк-эмбеддинги.
**Это ПЕРВЫЙ Text-компонент с реальными различимыми данными:** в отличие от mock-эмбеддеров (title/description/
hashtag, где вход — фикстуры), здесь вход — **настоящий ASR-транскрипт** (Whisper реально работал). 5/6 видео
ok+present (чанков 1–3 по длине; -4RHVBIik=3), -Q6fnPIy present=0 (музыка, нет речи). **Мультиязычная модель**
(e5-large) — решает проблему RU, отмеченную у bge-en эмбеддеров. Реальные различимые косинусы (title↔transcript
0.73–0.82, transcript↔comments 0.79–0.87 — варьируются). Транскрипт — **богатейший текст-сигнал** (о чём говорят).
golden=0, conf NaN by design (Whisper без per-word confidence).

## 3. Функционал

Работает в TextProcessor, после `asr_extractor`:

1. Берёт Whisper-транскрипт (декод токенов ASR).
2. **Чанкинг** транскрипта (по длине/окнам) → N чанков.
3. **multilingual-e5-large** эмбеддинг каждого чанка → 1024-d → **L2** → `(N,1024)`.
4. Пишет `transcript_whisper_chunk_embeddings.npy` + tp_* мета (present, dim, chunks, source=whisper, policy).

**Зачем продукту:** транскрипт — **семантика произнесённого**: темы, ключевые слова, о чём видео на самом деле.
Один из **сильнейших текст-сигналов** (спикер говорит суть). Чанки → покрытие длинной речи; multilingual → RU/EN.
Кормит Fusion (текст) + cosine-метрики (соответствие title/desc/comments содержанию).

## 4. Вход

- **`asr_extractor`** (hard) — Whisper-транскрипт; нет ASR → `present=0` (напр. -Q6fnPIy музыка).
- **multilingual-e5-large checkpoint** (bundled).

## 5. Выход

- **`_artifacts/transcript_whisper_chunk_embeddings.npy` (N,1024)** L2 — чанк-эмбеддинги транскрипта.
- **tp_* мета:** present, embedding_dim=1024, whisper_chunks (1–3), source (whisper/youtube_auto/combined),
  policy (transcript_first/only), conf_present (0 — Whisper без confidence).
- **NaN-политика:** нет транскрипта → present=0; conf NaN by design.

## 6. Фичи (важное/неочевидное)

- **РЕАЛЬНЫЕ данные (главное отличие)** — вход = настоящий Whisper-транскрипт (5/6 present), не mock-фикстура.
  Косинусы title↔transcript варьируются (0.73–0.82) — реальная различимость (в отличие от идентичных title/desc/hashtag).
- **multilingual-e5-large** — мультиязычная модель (не bge-en); эмбеддит RU/EN-речь корректно. Решает проблему,
  отмеченную у title/description (en-only).
- **Чанкинг (N,1024)** — длинный транскрипт → несколько чанков (-4RHVBIik=3); покрытие всей речи, не обрезка.
- **`conf_present=0` (conf NaN by design)** — Whisper-payload без per-word confidence → conf-статистики NaN (честно).
- **-Q6fnPIy present=0** — музыкальный ролик без речи → нет транскрипта (корректно).
- **Policy `transcript_first`** — приоритет источника (whisper > youtube_auto) для embedding-id.

## 7. Архитектура / алгоритм

- **multilingual-e5-large** (BERT-like, mean-pool → 1024-d) + чанкинг + L2.
- **Сложность:** e5-large тяжелее MiniLM; CPU, дёшево на коротких транскриптах.
- **Детерминизм:** golden max|Δ|=0.0 (single- и multi-chunk).

## 8. Оптимизации

- **Мультиязычная модель** — RU/EN без потери качества (в отличие от bge-en).
- **Чанкинг** — покрытие длинной речи.
- **L2** — готово для similarity.
- **Source-policy** — гибкий выбор транскрипт-источника.
- **Детерминизм CPU**.

## 9. Слабые места

- **Наследует качество ASR** — Whisper-small ошибки/галлюцинации → шум в эмбеддинге; на не-речи present=0 (нет сигнала).
- **conf отсутствует** — нельзя взвесить чанки по уверенности ASR (Whisper без per-word conf).
- **Мало чанков на коротких видео** (1 чанк у 4/5) — на длинном контенте чанкинг раскроется сильнее.
- **1024-d непрозрачен** — ценность через темы/ключевые слова/косинусы (в производных компонентах).
- **Зависит от privacy** — транскрипт = потенциально PII (речь); хранение эмбеддинга ок, но сырой текст — вопрос.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[сред.] Улучшить ASR** (Whisper medium/large) — качество транскрипта → качество эмбеддинга.
2. **[сред.] Взвешивание чанков** — если появится ASR-confidence, взвешивать по надёжности.
3. **[низ.] Тематические производные** — темы/ключевые слова из транскрипт-эмбеддинга для аналитика.
4. **[низ.] Единая multilingual-модель** для всех text-эмбеддеров (title/desc → e5 вместо bge-en) — согласованность RU.

## 11. Рекомендации по архитектуре / связям

- **Транскрипт-эмбеддинг → Fusion (текст-ветка)** — сильнейший текст-сигнал (содержание речи); закрепить.
- **cosine-метрики** (title↔transcript, desc↔transcript, transcript↔comments) — «соответствие метаданных содержанию» (уже есть).
- **Единая multilingual-модель** для title/desc/hashtag/comments/transcript — согласованное RU-пространство.

## 14. Польза для моделей

**Высокая.** Транскрипт-эмбеддинг — **семантика произнесённого** = один из сильнейших текст-сигналов (о чём видео),
на **реальных данных** (не mock), в качественной мультиязычной форме (e5-large 1024-d L2), различим. Чанкинг покрывает
длинную речь. Ограничивают качество ASR (Whisper-small) и отсутствие conf-взвешивания. Топ-кандидат текст-ветки Fusion.

## 15. Польза для аналитиков

**Средняя-хорошая.** Сам эмбеддинг непрозрачен, но производные ценны: «о чём говорят», соответствие заголовка/
описания содержанию (cos 0.73–0.82 — реально), темы. На реальных данных (в отличие от mock-эмбеддеров) — доступно.
Ограничивает опаковость 1024-d (нужны темы/ключевые слова).
