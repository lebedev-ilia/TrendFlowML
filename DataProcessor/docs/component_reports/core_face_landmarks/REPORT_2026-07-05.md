# Отчёт: core_face_landmarks — ✅ ЗАШТАМПОВАН (2026-07-05)

Автор: Claude (автономная сессия). GPU-под A4500. Провайдер лицевых лендмарков (MediaPipe FaceMesh
468 точек) + поза/руки + person-mask. Питает shot_quality (face-ROI), emotion/detalize_face и др.

## Выход (валидирован)
- **`face_landmarks (N, F, 468, 3)`** — MediaPipe FaceMesh: 468 3D-точек на лицо/кадр (+ `face_landmarks_raw`).
- **`face_present (N, F)` bool**, **`has_any_face` () bool**, **`person_present`**, **`face_mesh_ran`**,
  **`has_any_pose`/`has_any_hands`** + `*_empty_reason` — богатая диагностика (v2, QA-friendly).
- Ось: `frame_indices` строго от Segmenter; `times_s = union[frame_indices]`.

## Оценка по 4 осям
- **Корректность:** ✅ выходной валидатор `validate_core_face_landmarks_npz.py` → VALID schema;
  структура 468×3 корректна; флаги присутствия согласованы.
- **Валидный empty (критично):** на видео **без лиц** → `has_any_face=False`, `face_present` все False,
  `empty_reason` заполнен, массивы правильной формы (не error). **Это ровно то, на что рассчитывает
  shot_quality** (face-ROI фичи → NaN by design). Подтверждено (shot_quality-цепочка отработала).
- **Face-present (подтверждено):** на видео с людьми → `has_any_face=True`, **245/245 кадров с лицами**,
  468 лендмарков детектятся. Т.е. оба пути работают: лица есть → детект, лиц нет → валидный empty.
- **Стабильность:** mediapipe FaceMesh детерминирован при фикс. входе.
- **Модель-fit:** лендмарки (468×3) + флаги → лицевые/поза-фичи для downstream (shot_quality face-ROI,
  emotion_face, micro_emotion, detalize_face). Для Encoder — присутствие/геометрия лица по времени.
