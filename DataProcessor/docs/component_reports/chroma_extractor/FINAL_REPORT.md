## 2. Резюме

`chroma_extractor` — **хромаграмма**: энергия по 12 классам высоты тона (C, C#, …, B) = **гармония/тональность**
аудио (какие ноты звучат). В отличие от mel/mfcc (тембр), chroma про **высоту/музыкальность**. Отдаёт
`chroma_mean_by_segment (N,12)` seq + 12 pitch-class-средних + `chroma_entropy/harmonic_stability/contrast/
dominant_energy` + `dominant_class`/`tuning_estimate` (16 фич). **Полностью живой** (все 6 ok, 0 NaN, golden=0 при
одном аудио) и **осмысленно различает музыку от речи**: `chroma_entropy` отделяет тональный/музыкальный ролик
(-Q6fnPIy: entropy 0.72, dominant_energy 0.86 — доминирует одна нота) от речевых/смешанных (остальные: entropy
~2.46 — плоская хрома, нет гармонии). Это **детектор музыкальности** — интерпретируемее mfcc. Одна фича почти
мертва (`harmonic_stability` ≈0.987 const). Пересекается с `key_extractor` (детекция тональности).

## 3. Функционал

Работает после Segmenter (аудио). librosa `chroma_cqt`:

1. CQT → **12-класс хромаграмма** (свёртка энергии в 12 полутонов октавы) → усреднение по сегменту →
   `chroma_mean_by_segment (N,12)`.
2. 12 pitch-class средних (C…B) — «профиль тональности» видео.
3. Дескрипторы: `chroma_entropy` (равномерность = музыкальность vs хаос), `dominant_class`/`dominant_energy`
   (доминирующая нота), `chroma_contrast`, `harmonic_stability`, `tuning_estimate` (отклонение настройки).

**Зачем продукту:** тональность/гармония — **сигнал музыкального контента**: есть ли выраженная музыка/мелодия,
её ключ, стабильность гармонии. Отделяет музыкальные видео (клипы, гейминг с саундтреком) от речевых (лекции/
влоги, где хрома плоская). Ключ/тональность — нишевый музыкальный инсайт.

## 4. Вход

- **Аудио** (Segmenter) — нет аудио → `status=empty` (chroma_mean=[NaN], dominant_class=−1 sentinel).
- **CQT-параметры** (частотный диапазон, разрешение).

## 5. Выход

- **Model-facing seq:** `chroma_mean_by_segment (N,12)` — хромаграмма по сегментам.
- **Video-level:** 12 `chroma_mean_<нота>`, `chroma_entropy`, `chroma_contrast`, `dominant_energy`,
  `harmonic_stability`, `dominant_class`, `tuning_estimate`.
- **16 табличных фич** + `segment_centers/durations_sec`, `segment_mask`.
- **NaN/sentinel:** empty → chroma_mean=[NaN], dominant_class=−1.

## 6. Фичи (важное/неочевидное)

- **`chroma_entropy` — детектор музыкальности (несущая)** — низкая (0.72) = чёткая тональность/доминирующая нота
  (музыка/гул), высокая (~2.46) = энергия размазана по 12 нотам (речь/шум, нет гармонии). Различает музыку от речи.
- **12 pitch-class профиль** — на тональном видео пиковый (ясный ключ), на речевом плоский (нет высоты). Основа
  детекции ключа.
- **`dominant_class`/`dominant_energy`** — какая нота доминирует и насколько; на -Q6fnPIy dominant_energy=0.86 (одна
  нота), у речи ~0.1 (нет доминанты).
- **`tuning_estimate`** — отклонение настройки инструмента от A440 (тонкий музыкальный признак).
- **`harmonic_stability` ≈ 0.987 (почти мёртвая)** — CV 0.6%, near-const; заявлена by design, но малоинформативна.
- **Пересечение с `key_extractor`** — детекция тональности/ключа; вероятно, key_extractor использует ту же хрому.

## 7. Архитектура / алгоритм

- **librosa `chroma_cqt`** (Constant-Q Transform → 12-класс свёртка), агрегация (numpy).
- **Сложность:** CQT дороже STFT, но CPU-дёшев на коротком аудио.
- **Детерминизм:** golden max|Δ|=0.0 при **одном аудио-файле** (расхождения 0.12 между «группами» — это разные
  скачанные файлы yt-dlp, не проблема детерминизма).

## 8. Оптимизации

- **Чистый librosa** — без моделей/GPU.
- **Seq + агрегаты + dominant/tuning** — музыкальный профиль на разных уровнях.
- **Sentinel −1** для empty dominant_class — чистая обработка нет-аудио.

## 9. Слабые места

- **Плоская/неинформативная на речи (главное для этого корпуса)** — хрома осмысленна на музыке; на речевых видео
  (доминируют в корпусе) entropy ~2.46, профиль плоский → фактически бинарный сигнал «музыка/не-музыка» на
  talking-head-контенте.
- **`harmonic_stability` почти мертва** (≈0.987 const) — фича-мусор.
- **Пересечение с key_extractor** — детекция ключа дублируется; возможна консолидация.
- **CQT отдельно от STFT-семейства** — ещё один независимый спектральный проход (spectral/mel/mfcc STFT + chroma CQT).
- **Опаковость pitch-class значений** — 12 чисел непонятны без перевода в «ключ C-мажор» и т.п.
- **Только аудио, смешивает источники** — музыка+речь в одной хроме (разделение — source_separation).

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[сред.] Консолидировать с `key_extractor`** — общий CQT/хрома → ключ + профиль; не дублировать.
2. **[сред.] Убрать `harmonic_stability`** (≈const) — фича-мусор.
3. **[сред.] Явный флаг «музыкальность»** из entropy/dominant_energy — actionable «в видео есть музыка/мелодия».
4. **[низ.] Перевести профиль в ключ** (`dominant_class`+tuning → «C-мажор») для аналитики.
5. **[низ.] Общий спектральный слой** (STFT+CQT переиспользовать).

## 11. Рекомендации по архитектуре / связям

- **Музыкальное под-семейство** (chroma + key + tempo + onset) — общий CQT/beat-анализ; консолидировать.
- **Связка с CLAP** — chroma (гармония) + CLAP (семантика звука) = музыкальный профиль.
- **Флаг музыкальности** как gate — не гонять музыкальные фичи на чисто речевом контенте.

## 13. Интерпретируемость

- **Средняя:** «в видео есть музыка/мелодия» (entropy/dominant) — понятно; ключ (после перевода) — музыкальный инсайт.
- **Добавить:** флаг «музыкальное видео», определённый ключ (C-мажор), «доминирует нота X»; 12 чисел скрыть.

## 14. Польза для моделей

**Умеренная.** `chroma_mean_by_segment (N,12)` + entropy/dominant — сигнал музыкальности/гармонии, различимый и
чистый. Полезен как «есть ли музыка и какая тональность» (комплемент к тембровым mel/mfcc и семантическому CLAP).
Ограничивает: на речевом контенте плоский (бинарный), пересечение с key_extractor, near-dead harmonic_stability.
Полезная нишевая (музыкальная) ось.

## 15. Польза для аналитиков

**Умеренная.** «Есть ли в видео музыка, какая тональность/ключ» — понятный музыкальный инсайт для музыкального/
гейминг-контента. Но на речевых видео неинформативен, а сырые 12 pitch-class значений опаковы без перевода в ключ.
Полезен для музыкальной вертикали, служебен для речевой.
