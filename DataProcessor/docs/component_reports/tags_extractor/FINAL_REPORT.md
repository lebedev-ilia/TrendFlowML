## 2. Резюме

`tags_extractor` — **лексический парсер тегов/хэштегов** из заголовка и описания: считает хэштеги (found/unique/
total/density/длина), выделяет **топ-5 тегов** (present/len/hash), задаёт privacy-режимы экспорта (hashed/none/raw)
и config-флаги группы (extract/merge/mutate) — 43 поля `tp_tags_*` (namespace общий с hashtag_embedder). Это
metadata-лексический слой (без эмбеддинга). **Данные 100% MOCK:** все 6 видео идентичны (`unique_count=3`,
`merged_count=3`, **`found_count=0`** — в реальном mock-тексте хэштегов нет, 3 тега инжектированы через autogen JSON).
**0 из 43 фич варьируются** между видео (39 констант, 16 — config/policy-флаги). Реальный парсинг тегов не задействован
(found_count=0). Теги = SEO/discovery-сигнал, privacy-режимы продуманы, но на mock всё фиктивно + сильный config-балласт.

## 3. Функционал

Работает в TextProcessor:

1. **Парсит** хэштеги/теги из title + description (regex `#\w+`) → found_count, density, длины.
2. **Мержит** с тегами из JSON-метаданных (merged_count).
3. Выделяет **топ-5** тегов (по частоте/важности) → `top{1..5}_present/len/hash01`.
4. Privacy-экспорт: режимы `hashed/none/raw` + policy-флаги (disabled_by_policy, truncated).

**Зачем продукту:** теги/хэштеги — **SEO/discovery-метаданные**: сколько тегов, релевантны ли, трендовые. Плотность
тегов в title/description = SEO-оптимизация. Кормит hashtag_embedder (семантика) + аналитику SEO.

- **NaN-политика:** нет текста → present=0.

## 6. Фичи (важное/неочевидное)

  (`merged_count=3`). Т.е. основной путь (парсинг `#tags` из текста) на данных не проверен.
- **0/43 варьируются** — все 6 mock-идентичны; нулевой различающий сигнал.
- **16/43 — config/policy-флаги** (export modes, group_enabled, require, truncated) — константы, не сигнал → раздувание контракта.

- **Privacy-режимы** (hashed/none/raw) — продуманный контроль приватности тегов.
