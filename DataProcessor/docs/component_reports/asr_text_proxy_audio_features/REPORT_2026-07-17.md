## Описание компонента

`ASRTextProxyExtractor` — CPU-only агрегатор TextProcessor. Принимает ASR-транскрипт из `VideoDocument.asr` (AudioProcessor → TextProcessor) и извлекает числовые прокси-признаки:
- **Basic**: segments_count, word_count, text_chars, confidence_*
- **Noise**: rare_word_ratio, oov_ratio, noise_proxy
- **Rhythm**: speech_rate_wpm, char_density, pause_density, filler_ratio
- **Intonation**: sentence_intonation_proxy

Схема: `asr_text_proxy_audio_features_output_v1`, 37 полей (`tp_asrproxy_*`), все scalar float32.

## Логика и оптимизация

**Логика:** Без нейросетей. Все вычисления — чистый Python + numpy. Детерминирован побайтово.

**NaN-политика задокументирована:**
- `conf_*` = NaN при `conf_present_rate=0` (Whisper не даёт confidence) — by design, C1
- `rhythm/intonation` = NaN при `present=0` — by design, C2
- `noise_proxy` = NaN когда нет ни текста, ни confidence

**Инфраструктура:** валидатор `validate_asr_text_proxy_text_npz.py` строгий (schema + struct + ranges). Поддерживает batch-режим (`--results-base`).

**Оптимизация:** компонент тривиально дёшев (строковые операции, numpy агрегации). Стоимость определяется upstream (ASR inference в AudioProcessor, не этот компонент).

## Замечания

1. **noise_proxy=0.0 при малом числе слов** — при 1-2 словах `_is_rare()` даёт 0 (слова короткие, без цифр). Нормально для коротких транскриптов.
2. **word_count=1–2** у 2 видео в датасете — валидно; в продакшне ожидается больше.
3. **Нет expected-empty NPZ** с status=empty в storage (component-level empty, не оркестраторский error) — тест U4 пройден синтетически (нет зависимости от пода).
