## Найденные и исправленные баги

### Баг 1: `_save_beat_times_npy` всегда возвращала None (КРИТИЧНО)

**Проблема:** тело метода (строки 513-528) оказалось мёртвым кодом внутри `_save_beat_segment_index_npy` (после всех `return`). Метод `_save_beat_times_npy` при `size >= 10000` не выполнял сохранение и молча возвращал None.

**Последствие:** при видео с > 10000 битов beat_times не сохранялись ни в NPZ, ни в .npy файл (данные терялись).

**Фикс:** перенесено тело метода в правильный def, убран мёртвый код.

**Тест фикса:** `_save_beat_times_npy(big_beats[10001], tmpdir)` → path `_artifacts/test_comp_beat_times_sec.npy`, file_exists=True ✓

### Баг 2: missing import `Any` в validate_rhythmic.py

**Проблема:** `from typing import Dict, List, Tuple` — без `Any`, а сигнатура `-> Dict[str, Any]` использует его.

**Фикс:** добавлен `Any` в импорт.
