## 2. Резюме

`title_embedder` — **семантический эмбеддинг заголовка видео** через sentence-transformer (BAAI/bge-large-en-v1.5,
**1024-d, L2-нормализованный**). Отдаёт эмбеддинг заголовка (`title_embedding.npy` + tp_titleemb_* мета) —
model-facing текстовый токен для Fusion/предсказания (заголовок — один из **сильнейших предикторов кликабельности/
CTR**: clickbait, ключевые слова, эмоц. триггеры). **Механизм образцовый:** 1024-d, L2-норма=1.0000, present=1,
детерминизм golden=0 (20 прогонов побайтово). **Но данные в батче — MOCK:** 5 из 6 видео получили **одинаковый
фикстур-заголовок** из E2E-autogen («E2E sample video 1: creator tips…») → их эмбеддинги **идентичны** (попарный
косинус 1.0); только -Q6fnPIy с другим текстом (0.84). То есть эмбеддер работает идеально, но **реальные YouTube-
заголовки не использованы** — вход фиктивный.

## 3. Функционал

Работает в TextProcessor. Пайплайн:

1. Берёт заголовок видео (из метаданных Fetcher; в E2E — autogen-фикстура).
2. **sentence-transformer bge-large** → 1024-d эмбеддинг → **L2-нормализация**.
3. Пишет `_artifacts/title_embedding.npy` + мета `tp_titleemb_dim/present/l2_norm/artifact_written` в общий text_features.npz.

**Зачем продукту:** заголовок — **топ-фактор CTR/кликов**: формулировка, ключевые слова, эмоц. триггеры, длина.
Семантический эмбеддинг заголовка кормит Fusion (текст-ветка) — потенциально один из самых информативных
предикторов просмотров. Также основа для «похожих заголовков»/кластеризации ниши.

## 4. Вход

- **title** (метаданные) — есть → эмбеддинг; нет → graceful empty (present=0). В батче — mock-фикстура.
- **bge-large checkpoint** (sentence-transformers, CPU).

## 5. Выход

- **`_artifacts/title_embedding.npy` (1024,)** L2 — model-facing семантический токен заголовка.
- **Мета в text_features.npz:** `tp_titleemb_dim=1024`, `tp_titleemb_present`, `tp_titleemb_l2_norm`,
  `tp_titleemb_artifact_written`.
- **NaN-политика:** нет title → present=0, graceful.

## 6. Фичи (важное/неочевидное)

- **1024-d bge-large L2-эмбеддинг** — качественная модель (bge-large — сильный retrieval-эмбеддер, лучше MiniLM-384).
  L2-норма=1.0 → косинус = скалярное произведение (готово для similarity/attention).
- **5/6 идентичны (MOCK)** — попарный косинус 1.0 у 5 видео: **один фикстур-заголовок на всех**. НЕ баг эмбеддера,
  а фиктивный вход (E2E autogen). Реальная различимость не проверяема на этих данных.
- **Детерминизм golden=0** — sentence-transformers на CPU побайтово воспроизводим (20 прогонов).
- **Заголовок ≠ контент** — эмбеддинг заголовка семантически независим от аудио/видео → уникальная текст-ось Fusion.
- **Английская модель (en-v1.5)** — на русских заголовках качество ниже (bge-large-en); для RU-контента нужна мультиязычная.

## 7. Архитектура / алгоритм

- **sentence-transformer bge-large-en-v1.5** (BERT-like, mean-pooling → 1024-d) + L2.
- **Сложность:** encode 305–932 мс/заголовок (CPU); дёшево.
- **Детерминизм:** golden max|Δ|=0.0.

## 8. Оптимизации

- **bge-large** (сильный эмбеддер) + L2 — качественный готовый токен.
- **Artifact .npy** отдельно от text_features.npz — не раздувает общий NPZ.
- **Graceful empty** при отсутствии title.
- **Детерминизм CPU** — воспроизводимо.

## 9. Слабые места

- **MOCK-заголовки в батче (главное).** 5/6 видео — одинаковая фикстура → идентичные эмбеддинги. Реальные YouTube-
  заголовки не подставлены (E2E autogen). Пока Fetcher не даёт реальные title, компонент даёт фиктивный сигнал.
- **Английская модель** — bge-large-**en**; русские/мультиязычные заголовки эмбеддятся хуже (нужна bge-m3/multilingual).
- **Не проверена различимость на реальных данных** — mock скрывает, насколько эмбеддинги различают реальные заголовки.
- **Заголовок только один** — нет вариантов/A-B (что естественно, у видео один title).
- **1024-d непрозрачен** — сам эмбеддинг не интерпретируем (нужны производные: similarity, кластер, ключевые слова).

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Подать реальные YouTube-заголовки** (из Fetcher) вместо mock-фикстуры — сейчас 5/6 идентичны; без
   реальных title компонент бесполезен на данных.
2. **[выс.] Мультиязычная модель** (bge-m3/multilingual-e5) — для RU/смешанных заголовков; bge-large-en субоптимален.
3. **[сред.] Проверить различимость** на реальных заголовках (CV/кластеры) после подачи реальных данных.
4. **[низ.] Интерпретируемые производные** — «похожие заголовки», ключевые слова, тема (для аналитика).

## 11. Рекомендации по архитектуре / связям

- **Title-эмбеддинг → Fusion (текст-ветка)** — сильный предиктор CTR; закрепить как model-вход.
- **Reuse** для topk_similar_titles/title_to_hashtag_cosine/title_cluster_entropy (все зависят от title-эмбеддинга).
- **Единый эмбеддер** для title/description/comments (одна модель, разные тексты) — согласованное пространство.

## 14. Польза для моделей

**Высокая (потенциал), ограниченная (факт).** Эмбеддинг заголовка — **один из сильнейших текст-предикторов CTR/
просмотров** (заголовок решает клик), в качественной форме (bge-large 1024-d L2) для Fusion. Механизм образцов
(детерминизм, L2). **Но** на данных 5/6 идентичны (mock) → реальный сигнал не задействован. После подачи реальных
заголовков — потенциально топ-фича текст-ветки; сейчас — фиктивно.

## 15. Польза для аналитиков

**Средняя (потенциал).** Сам 1024-d эмбеддинг непрозрачен, но производные («похожие видео по заголовку», тема,
ключевые слова) — ценны креатору. На mock-данных недоступно; английская модель ограничивает RU. После реальных
title + производных — полезно.
