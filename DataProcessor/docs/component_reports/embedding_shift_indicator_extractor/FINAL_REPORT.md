## 2. Резюме

`embedding_shift_indicator_extractor` — **derived-индикатор смещения темы начало→конец**: считает косинус между
средним эмбеддингом **начального окна** (первые `win` чанков) и **конечного окна** (последние `win`) транскрипта →
`cosine_begin_end` + `shift_flag` (косинус < 0.85 = тема ушла от начала к концу). Продуктовый смысл отдельный и
хороший — **нарративная арка**: связан ли финал видео с началом (intro→payoff когерентность vs «уплыло»). **На
реальных данных почти мёртв:** `present=1` **лишь у 1/22** видео (21a279af: 3 чанка, cosine=0.791, shift_flag=1 —
тема сместилась). 4 видео = **1 чанк < min_chunks=2** → NaN; 17 видео = **0 чанков** (нет транскрипта). Единственное
значение корректно, но кросс-видео сигнала нет. **Системно:** третий derived-компонент подряд (после embedding_stats)
гибнет на **грубой нарезке чанков** — корень в `transcript_chunk_embedder`.

## 3. Функционал

Работает после chunk-эмбеддера:

1. **Резолв relpath** chunk-эмбеддингов (приоритет whisper→youtube_auto, legacy-fallback).
2. **Окна:** `win = min(n_window_chunks=2, n_chunks//2)`; start = mean первых win, end = mean последних win.
3. **cosine_begin_end** = cos(start, end); `shift_flag` = 1 если < threshold (0.85); `margin` = cos − threshold.
4. **Extra (off):** cosine_first_last, mean_cosine_last_to_start_window.
5. Пишет 27 tp_embshift_* (метрики + present/6 диагност-флагов/config).

**Зачем продукту:** «уходит ли видео от своей темы к концу» — **структурная связность/нарративная арка**. Резкий
сдвиг может значить рассеянность (плохо для удержания) либо намеренную структуру intro→развязка.

## 4. Вход

- **chunk-эмбеддинги транскрипта** (`transcript_chunk_embedder`, whisper) — нужно **≥2 чанка**; на корпусе почти
  всегда 0–1 чанк.
- Нет/мало чанков → present=0, метрики NaN.

## 5. Выход

- **27 tp_embshift_* фич:** cosine_begin_end, shift_flag, margin, cosine_threshold, n_chunks/n_window_chunks/dim,
  extra-косинусы (first_last, mean_last_to_start), source_used (whisper/youtube_auto), 6 диагност-флагов
  (chunk_missing/dim_mismatch/zero_norm/nan_inf/unsafe_relpath/used_legacy), config, тайминги.
- **NaN-политика:** <2 чанков → present=0, метрики NaN.

## 6. Фичи (важное/неочевидное)

- **present=1 только у 1/22 (главное)** — 21a279af (3 чанка): cosine_begin_end=**0.791**, shift_flag=**1** (0.791 <
  0.85 → тема сместилась), margin=−0.059. Единственное реальное значение; корректно.
- **4 видео = 1 чанк** — источник whisper есть (source_used_whisper=1), но n_chunks=1 < min=2 → shift невычислим.
- **17 видео = 0 чанков** — нет транскрипт-чанков вовсе (нет ASR-chunk-эмбеддингов) → present=0.
- **win-clamp** — `min(2, n_chunks//2)`: у 3-чанкового видео win=1 (сравнивает первый и последний чанк).
- **shift_flag бинарен** — теряет градацию (0.791 и 0.5 → одинаковый flag=1); `cosine_begin_end`/margin информативнее.
- **6 диагност-флагов все 0** — чисто (нет dim/nan_inf/zero_norm/unsafe).

## 7. Архитектура / алгоритм

- **Чистый numpy** — mean по окнам + косинус. Своей модели нет.
- **Безопасность:** safe-join relpath, finite-check, zero-norm-check.
- **Сложность:** тривиальная.
- **Детерминизм:** тривиально детерминирован.

## 8. Оптимизации

- **Reuse** chunk-эмбеддингов.
- **Оконное усреднение** (устойчивее, чем один чанк на край).
- **win-clamp** к n_chunks//2 (защита от перекрытия окон).
- **Приоритет источников** + legacy-fallback.
- **emit_extra_metrics / compute_extra_cosines gates** — опциональная тяжесть.

## 9. Слабые места

- **Почти мёртв (главное)** — present=1 у 1/22; грубая нарезка чанков (0–1 чанк) убивает begin/end-сравнение.
- **Системный блокер нарезки** — общий с embedding_stats (и далее pair/qa); корень — `transcript_chunk_embedder`
  выдаёт слишком крупные чанки.
- **shift_flag бинарен** — потеря градации; порог 0.85 произволен без калибровки на реальных данных.
- **Тонкая надстройка 2-го порядка** — зависит от качества и числа чанков.
- **Только begin↔end** — не ловит дрейф в середине (для этого нужна полная траектория, как в embedding_stats variance).

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс., системный] Мельче нарезать чанки** (`transcript_chunk_embedder`) — чтобы у большинства видео было ≥2
   чанка. Оживит сразу embedding_shift + embedding_stats + pair/qa.
2. **[сред.] Использовать `cosine_begin_end`/margin как непрерывную фичу**, а не бинарный shift_flag (больше сигнала).
3. **[сред.] Калибровать порог 0.85** на реальном распределении (сейчас произволен).
4. **[низ.] Добавить mid-drift** (максимум попарного расстояния по траектории) — полнее, чем begin↔end.

## 11. Рекомендации по архитектуре / связям

- **transcript_chunk_embedder → shift/stats** — согласовать гранулярность нарезки (ключ ко всей derived-группе).
- **Объединить derived-статистики** (shift + stats + cosine_metrics) в единый «структурный профиль текста» — один
  проход по чанкам, набор согласованных метрик (variance, begin-end shift, drift-траектория).

## 13. Интерпретируемость

- **Хорошая и наглядная** — «видео заканчивается на той же теме, что началось» / «уплыло к концу» — понятный
  структурный инсайт.
- **Добавить:** «начало и конец совпадают на 79% — видео немного уходит от темы»; при мелкой нарезке — карта дрейфа
  по времени. Готовый интерпретируемый нарратив-сигнал (простаивает).

## 14. Польза для моделей

**Низкая.** Смещение темы начало→конец (нарративная связность) — правдоподобный предиктор удержания, но фактически
почти мёртв (present=1 у 1/22 из-за грубой нарезки), а shift_flag бинарен. Модель получает почти нулевой сигнал.
Потенциал умеренный (после мелкой нарезки + непрерывной фичи).

## 15. Польза для аналитиков

**Низкая.** «Ваше видео заканчивается на другой теме, чем началось» — понятный структурный инсайт, но на данных
доступен лишь у 1/22. После мелкой нарезки — наглядная аналитика нарративной арки; сейчас — нечего показать.
