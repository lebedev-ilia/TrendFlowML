## 2. Резюме

`transcript_aggregator` — **агрегация чанк-эмбеддингов транскрипта в video-level вектор** «о чём видео»: пулит
(mean/max/combined) чанк-эмбеддинги `transcript_chunk_embedder` по источникам (whisper/youtube_auto/combined) с
опциональным **temporal decay** (свежие/начальные чанки весомее) → 1024-d L2-эмбеддинги (19 tp_tragg_* фич + артефакты).
**Второй Text-компонент на РЕАЛЬНЫХ данных** (после transcript_chunk): 5 видео present (whisper), -Q6fnPIy absent
(музыка без речи). **Различим:** `combined_agg_mean` попарные косинусы **0.70–0.88 (distinct, не 1.0)** — видео
кластеризуются по содержанию (напр. -0In↔-4RH=0.87, -4WR↔-8We=0.88 — реальная семантическая структура). Это
консолидированный видео-уровневый токен содержания речи — **сильный вход Fusion** (текст-ветка).

## 3. Функционал

Работает после `transcript_chunk_embedder`:

1. Берёт чанк-эмбеддинги `(N,1024)` транскрипта.
2. **Пулинг:** mean / max / combined по чанкам → video-level 1024-d.
3. **По источникам:** whisper / youtube_auto / combined (present-флаги).
4. **Temporal decay** (`decay_rate`) — взвешивание чанков по позиции (опц.).
5. Пишет `transcript_{src}_agg_{mean,max}.npy` + tp_tragg_* (present, n_chunks, mean/max_std, compute-флаги).

**Зачем продукту:** видео-уровневый эмбеддинг транскрипта — **консолидированная семантика произнесённого** (о чём
видео целиком): для Fusion (одна текст-фича содержания), сравнения видео по теме, кластеризации. Сильнейший
текст-сигнал в компактной video-форме.

## 4. Вход

- **`transcript_chunk_embedder`** (hard) — чанк-эмбеддинги; нет транскрипта → present=0 (-Q6fnPIy).
- **decay_rate** (config) — temporal взвешивание.

## 5. Выход

- **Артефакты:** `transcript_{whisper,combined}_agg_{mean,max}.npy` (1024,) L2 — video-level эмбеддинги.
- **19 tp_tragg_* фич:** present (общий/whisper/youtube/combined), n_chunks (per source), mean/max_std (per source),
  compute-флаги (mean/max/std/combined), decay_rate, write_artifacts.
- **NaN-политика:** нет источника → present_<src>=0.

## 6. Фичи (важное/неочевидное)

- **REAL & discriminative (главное)** — combined_agg_mean cosines 0.70–0.88 (не 1.0!) → эмбеддинги различны, видео
  кластеризуются по содержанию. В отличие от mock-эмбеддеров (title/desc/hashtag/comments — идентичны).
- **Multi-source (whisper/youtube_auto/combined)** — гибкость источника транскрипта; combined = объединение.
  На корпусе только whisper present (youtube_auto absent).
- **mean vs max пулинг** — mean = усреднённая тема, max = пиковые/выделяющиеся семантические черты; оба полезны.
- **`decay_rate` (temporal weighting)** — взвешивание чанков по позиции (напр. начало важнее для хука) — умный задел.
- **`mean_std`/`max_std`** — разброс чанков (тематическая однородность видео: узкая тема vs разнородная).
- **Кластеризация по содержанию** — -0In↔-4RH=0.87, -4WR↔-8We=0.88: реальные тематические пары (в отличие от mock).

## 7. Архитектура / алгоритм

- **Пулинг** (numpy mean/max) чанк-эмбеддингов + L2 + опц. temporal decay. Модели нет.
- **Сложность:** тривиально (пулинг N×1024).
- **Детерминизм:** golden (заявлен PASS).

## 8. Оптимизации

- **Reuse чанк-эмбеддингов** transcript_chunk (не эмбеддит заново).
- **Multi-pooling** (mean/max/combined) — разные аспекты содержания.
- **Multi-source** — гибкость.
- **Temporal decay** — позиционное взвешивание.

## 9. Слабые места

- **Наследует ASR/чанк-качество** — Whisper-small ошибки → шум в агрегате; present=0 на не-речи.
- **Лёгкий пулинг** — по сути mean/max поверх chunk-эмбеддера; собственного расчёта мало (но video-токен нужен).
- **Aggregated 1024-d непрозрачен** — ценность через тему/сравнение (в производных).
- **youtube_auto absent** — только whisper-источник на корпусе (youtube-субтитры не используются).
- **decay_rate не проверен** на эффект (задел).

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[сред.] Attention/learnable pooling** вместо mean/max — если video-токен важен для Fusion (сейчас простое усреднение).
2. **[сред.] Проверить temporal decay** — оценить, помогает ли позиционное взвешивание (хук в начале).
3. **[сред.] Задействовать youtube_auto** (субтитры) как fallback/дополнение к whisper.
4. **[низ.] Тематические производные** (тема видео, ключевые слова) для аналитика.

## 11. Рекомендации по архитектуре / связям

- **transcript_aggregator → Fusion (текст-ветка)** — video-level содержание речи; сильный вход. Закрепить.
- **Reuse video-эмбеддинга** для semantic_cluster (кластеризовать по содержанию, не mock-title) и cosine-метрик.
- **combined-источник** (whisper+youtube) — устойчивость к ASR-ошибкам.
