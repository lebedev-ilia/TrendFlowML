## 1. Что делает компонент

`TopKSimilarCorpusTitlesExtractor` — ищет Top-K ближайших заголовков из corpus-pack по cosine similarity с title-эмбеддингом текущего видео.

- Corpus-pack (`embeddings.npy` + `ids.json`) загружается через ModelManager (`similar_titles_corpus_v1`)
- Title-эмбеддинг читается из `doc.tp_artifacts.embeddings.title.relpath` (`.npy` файл)
- Бэкенд: numpy (точный matmul) или faiss HNSW (приближённый)
- Выход: 29 ключей `tp_topktitles_*` в `text_features.npz`; payload `topk_similar_ids` / `topk_similar_scores` в `result.topk_similar_corpus_titles`

## 7. Найденные проблемы / логические наблюдения

1. **Stub corpus (8 нулевых векторов)** — не баг компонента; логика корректна. Но не позволяет проверить реальную различимость (top1=0.0 везде).
2. **6 error-NPZ** — TextProcessor упал до topk; не проблема topk_similar_titles_extractor.
3. **FAISS backend** не тестировался (не установлен); fallback на numpy — корректный. TODO: тест с faiss при масштабировании.
4. **batch-оптимизации** нет (каждый document обрабатывается независимо) — допустимо для TextProcessor.
5. **cache TTL** работает корректно (OrderedDict с TTL 3600с, max 2 entry).
