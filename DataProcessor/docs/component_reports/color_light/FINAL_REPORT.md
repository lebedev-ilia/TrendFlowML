## 2. Резюме

`color_light` — **классический CV-анализатор цвета и освещения** (без нейросетей). Для каждого
Segmenter-кадра он считает десятки характеристик: HSV/LAB-статистики, доминантную палитру (KMeans в Lab),
colorfulness, тёплые/холодные тона, skin-tone, контраст (global/local), клиппинг светов/теней, динамический
диапазон, равномерность освещения (grid 3×3), виньетирование, число источников света, мягкость/жёсткость
света. Наружу отдаёт два слоя: **компактный model-facing вектор `frame_compact_features (M,16)`** (стабильные
16 нормированных фич на кадр — для Encoder) и **аналитический `video_features` (543 ключа)** + scene/frame
debug. Компонент прод-готов по контракту (U1–U6/C1–C4 PASS), но у него две существенные оговорки данных:
(1) 6 aesthetic-полей = NaN by design (модели не подключены), (2) **в реальном storage-корпусе 7 video-level
temporal-фич мертвы** (getf-баг: gini=NaN, entropy≈0, все change-speed/strobe/periodicity=0) — фикс закоммичен
(f99742e), но батч им ещё не пере-прогнан.

## 3. Функционал

Стоит в Tier-2, **после `scene_classification`** (берёт `scenes[*].indices`, фильтрует по allowed
frame_indices Segmenter — пересэмплинг запрещён). Три уровня агрегации:

1. **Frame-level** (`extract_frame_features`) — на кадр: RGB mean/std, HSV (+взвешенная по насыщенности
   энтропия hue), LAB (L/contrast/ab-balance), палитра (KMeans Lab → доминантный a,b), colorfulness,
   warm/cold, skin-tone, гармонии (комплементарная/аналоговая), lighting (яркость/контраст/энтропия/клиппинг/
   динамический диапазон dB/локальный контраст/равномерность/виньетка), направление и мягкость света.
2. **Scene-level** (`extract_scene_features`) — усреднение по сцене + temporal: скорость смены яркости/цвета,
   flicker, flash-события, color stability, temporal entropy, периодичность.
3. **Video-level** (`extract_video_features`) — глобальные агрегаты (543 ключа): распределение цвета
   (entropy/gini), стили цветокоррекции (teal&orange/film/vintage/tiktok…), aesthetic (NaN-заглушки),
   глобальная динамика яркости/цвета, strobe, периодичность.

**Зачем продукту:** цвет и свет — **сильный драйвер эстетики и «кликабельности»** видео. Яркая насыщенная
картинка (tiktok-стиль), тёплая/холодная гамма, профессиональный свет vs любительский — всё это влияет на
первое впечатление и удержание. Компонент даёт и model-вход (компакт), и аналитику креатору («ваше видео
тёмное/пересвеченное/малонасыщенное»).

## 4. Вход

- **`scene_classification`** (hard, no-fallback) — `scenes` (dict со `indices` и `scene_label`); нет → `RuntimeError`
  (status=error). Ось кадров берётся из сцен, фильтруется по allowed frame_indices (строго Segmenter).
- **`union_timestamps_sec`** (из frames metadata) — ось времени; нет → status=error.
- **Кадры** — `FrameManager.get(idx)`, RGB uint8 (с `_ensure_rgb` по `color_space` meta).
- Конфиг (`max_frames_per_scene`, `stride`) — **deprecated**, сэмплинг только Segmenter.

## 5. Выход

- **Model-facing:** `frame_compact_features (M,16)` float32 + `frame_compact_feature_names (16,)` +
  `frame_compact_frame_indices` + `aggregated.frame_compact`. 16 фикс-фич (`FRAME_COMPACT_KEYS`): hue mean/std/
  entropy_weighted, sat/val/L mean, global/local contrast, colorfulness, skin_tone, over/under-exposed,
  vignetting, soft_light, dominant Lab a/b — все нормированы [0,1], стабильные размерности для Encoder.
- **Analytics:** `video_features` (543 ключа), `scenes`/`frames` (debug, тяжёлые), `sequence_inputs`
  (frames/scenes/global последовательности), `times_s`/`sequence_times_s`, `meta.ui_payload`.
- **NaN-политика:** 6 aesthetic-полей NaN by design (`nima_*`, `laion_*`, `cinematic_lighting_score`,
  `professional_look_score`) + маски `*_present=0`.

## 6. Фичи (важное/неочевидное)

- **`hue_entropy_weighted`** — энтропия оттенков, **взвешенная по насыщенности**: тусклые пиксели не засоряют
  меру «богатства палитры» (осознанно лучше простой hue-энтропии).
- **Доминантный цвет через KMeans в Lab (a,b)** — устойчивее к яркости, чем в RGB; наружу как нормированные
  `dominant_lab_a/b_norm`.
- **`colorfulness_norm`** (Hasler-Süsstrunk-подобная метрика std(rg)+std(yb)+0.3·mean) — «насколько сочная
  картинка», сильный сигнал tiktok/vlog-стиля.
- **`vignetting_score`** = 1−corner/center brightness (grid 3×3) — прокси кинематографичности/объектива.
- **`soft_light_prob`** из дисперсии Лапласиана (низкая → мягкий рассеянный свет, высокая → жёсткий) с
  подогнанными порогами 50…600 — эвристика без калибровки.
- **`skin_tone_ratio`** — доля «кожных» пикселей (hue∈[0,25], sat≥20, val≥50): для vlog/beauty-контента высок,
  на статике/коротких видео часто const (малоинформативен).
- **Стили цветокоррекции** (teal&orange/film/vintage/tiktok) — грубые пороговые эвристики по средним RGB/sat,
  скорее «сигнальные», чем точные.

## 7. Алгоритм / архитектура

- **Чистый CPU**: opencv (cvtColor/Sobel/Laplacian), numpy (гистограммы/статистики), sklearn (KMeans Lab),
  scipy (entropy/find_peaks/gaussian_filter/label). Нейросетей нет.
- **Сложность:** линейна по кадрам; KMeans на сэмпле ≤10k пикселей, локальный контраст скользящими окнами.
  Дёшев: 133 кадра ~1.5 c, 250 кадров ~3–4 c (CPU). В полной цепочке доминируют GPU-deps (CLIP+scene), не он.
- **Детерминизм:** golden diff=0 **только при `OMP_NUM_THREADS=1`** — иначе многопоточный BLAS даёт ≤1 ULP
  дрожание (2/2128 элементов, max|Δ|=1.19e-7). KMeans фиксирован `random_state=42`.

## 8. Оптимизации

- **Компактный (M,16) контракт** отдельно от 543-ключевой аналитики — Encoder не тянет тяжёлый debug.
- **Сэмплинг пикселей ≤10k** для палитры/KMeans — не гоняет весь кадр.
- **Взвешенная энтропия hue** и **Lab-KMeans** — осознанные улучшения качества фич.
- **`store_debug_objects`** флаг — можно не писать тяжёлые `frames`/`scenes` в NPZ.
- **Переиспользование HSV/LAB** между блоками фич в пределах кадра (частично; см. §10 — есть дубли cvtColor).
- **Reuse per-frame декодера** для 200k рекомендован в AGENT_CONTEXT (узкое место — scene/CLIP deps, не он).

## 9. Слабые места

- **7 video-level temporal-фич мертвы в реальном батче (главное).** В storage-артефактах (пре-фикс код):
  `color_distribution_gini=NaN`, `color_distribution_entropy≈8.4e-8`, `global_brightness_change_speed=0`,
  `global_color_change_speed=0`, `strobe_transition_frequency=0`, `global_color_periodicity=0`,
  `global_color_shift=0`. Это **getf-баг** (hue читался `frame.get` вместо `frame["features"]`), фикс
  закоммичен (`f99742e`), но **батч не пере-прогнан** → на реальных данных эти аналитические фичи бесполезны.
- **6 aesthetic-полей = NaN by design** — NIMA/LAION/cinematic/professional не подключены; есть `*_present=0`
  маски, но это «пустые слоты» контракта.
- **Эвристики без калибровки** — пороги soft_light (50/600), стили цветокоррекции, colorfulness/100 — подогнаны
  «на глаз», не валидированы на разметке. Стили скорее сигнальные, чем точные.
- **543 ключа аналитики — раздутость/дубли** (`*_mean/std/min/max` от scene-агрегатов), высокая корреляция;
  много малоинформативных полей.
- **Const-фичи на коротких/статичных видео** — vshort (M=12) даёт std>0 лишь у **7/16** компакт-dim
  (skin_tone/local_contrast/vignetting/soft_light/dominant_lab часто const) — валидно, но бедно.
- **Golden требует пиннинга потоков** (`OMP_NUM_THREADS=1`) — в проде надо явно выставлять воркерам.
- **Дубли cvtColor** (RGB→HSV/LAB/GRAY считаются в нескольких методах повторно) — мелкий CPU-техдолг.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Пере-прогнать storage на пост-фикс коде** — материализовать рабочие gini/entropy/temporal-фичи
   (сейчас на реальных данных 7 аналитических фич = NaN/0, хотя фикс в git). Общий с другими компонентами
   паттерн «фикс есть, батч не пере-прогнан».
2. **[выс.] Подключить aesthetic-модель (NIMA или LAION-aesthetic)** — 6 NaN-полей это «дыра качества»,
   а именно эстетическая оценка — самый ценный для пользователя сигнал цвета/света.
3. **[сред.] Откалибровать эвристики** (soft_light, стили, colorfulness) на небольшой разметке или хотя бы
   задокументировать распределения по реальному корпусу; сейчас пороги произвольны.
4. **[сред.] Проредить 543 аналитических ключа** — убрать сильно коррелирующие `min/max` дубли, оставить
   информативное ядро (меньше шума для дашборда и мельче NPZ).
5. **[низ.] Дедуп cvtColor** — один раз посчитать HSV/LAB/GRAY на кадр и передавать между блоками.

## 11. Рекомендации по архитектуре / связям

- **Reuse HSV/LAB/GRAY** с другими CV-компонентами (shot_quality/frames_composition тоже считают яркость/
  контраст) — общий per-frame цветовой кэш вместо повторного декодирования.
- **Зависимость от scene_classification** оправдана (агрегация по сценам), но делает color_light заложником
  качества сцен; рассмотреть fallback на «одна сцена = всё видео», если scene пустой, вместо жёсткого краха.
- **Компакт (M,16)** — согласовать с FEATURE_ENCODER_CONTRACT (стабильные имена/нормировки уже к этому готовы).

## 13. Интерпретируемость

- **Сильная сторона** компонента: почти все фичи человекочитаемы (яркость, насыщенность, тёплые/холодные тона,
  пересвет/недосвет, виньетка). `meta.ui_payload` (presentation) уже строится.
- **Добавить:** словесная сводка «видео тёплое/холодное, насыщенное/тусклое, пересвеченное/тёмное,
  мягкий/жёсткий свет»; палитра доминантных цветов (swatch) на превью; сравнение с «типичным» видео ниши.
- Эстетическую оценку (после подключения NIMA/LAION) — как понятный 1–10 балл «профессиональности картинки».

## 14. Польза для моделей

**Умеренно-высокая для компакта.** `frame_compact_features (M,16)` — лёгкий, чистый (0% NaN), нормированный
dense seq-вход: цвет/свет/контраст покадрово — правдоподобный предиктор эстетики/стиля, коррелирующей с
кликабельностью. Форма идеальна для Encoder (16 скаляров/кадр). Ограничивает: эвристичность (не выучено),
const-фичи на статике, отсутствие aesthetic-оценки (самое ценное — в NaN). Аналитические 543 ключа для модели
избыточны/шумны и частично мертвы. Оценка держится на надёжном компакте.

## 15. Польза для аналитиков

**Высокая по потенциалу, ограничена данными.** Цвет/свет — самая понятная креатору аналитика: «ваше видео
тёмное/пересвеченное/малонасыщенное/холодное», палитра, стиль цветокоррекции. Компакт и часть video_features
работают. Но: gini/entropy распределения цвета и все temporal-динамики (смена цвета/яркости, strobe) на
реальных данных сейчас NaN/0 (пре-фикс), а aesthetic-баллы — заглушки. После пере-прогона и подключения NIMA
это станет одним из самых полезных для креатора компонентов.
