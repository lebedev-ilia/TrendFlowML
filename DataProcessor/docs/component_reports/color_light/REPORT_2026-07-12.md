## Найденный баг (C4) — color_distribution_gini=NaN / entropy≈0
- **Симптом:** cl_run1/cl_gold → `color_distribution_gini=NaN`, `color_distribution_entropy≈8.4e-8`
  при валидном per-frame `hue_mean` (17.79–42.47, sum=4122, 0 NaN).
- **Причина:** в `processor.py::extract_video_features` hue_values собирались из frame_list через
  `frame.get("hue_mean")`, но фичи лежат в `frame["features"]["hue_mean"]` → всегда default 0.0.
  Все hue=0 → гистограмма в одном бине → entropy≈0; `_compute_gini_coefficient` при sum(values)=0 даёт
  деление на ноль → NaN. (Тот же дефолт занулял brightness/color change speed, strobe, periodicity, shift.)
- **Фикс:** safe-getter `getf(frame,key)` читает `frame["features"][key]` (уже в processor.py, строки ~995).
- **Проверка фикса:** переигровка color_light на тех же deps → gini=0.0726, entropy=2.489
  (совпадает с ручным пересчётом из `d["frames"]`), NaN-набор сузился с 7 до 6 by-design ключей.
- **СТАТУС КОДА:** фикс присутствует локально (`git status: M ...processor.py`, uncommitted) и на поде.
  **Требуется коммит в git DataProcessor** (source of truth), аналогично ocr_extractor-фиксу.
