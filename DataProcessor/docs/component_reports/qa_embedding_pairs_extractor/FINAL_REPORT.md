## 2. Резюме

`qa_embedding_pairs_extractor` — **извлекатель вопросоподобных фраз + их эмбеддинги** (имя историческое — Q–A пары
НЕ строятся). Ищет вопросы в title/description/transcript/comments по вопросительным словам (кто/что/где/how/what/why)
и знаку «?», эмбеддит их e5-large в матрицу N×D и считает связность (`mean_cosine_to_centroid`), плотность
(`questions_per_min`, `questions_per_1k_chars`). Продуктовый смысл хороший — **FAQ/curiosity-хуки**: контент,
задающий вопросы («как…», «что если…»), драйвит любопытство/вовлечение/CTR. **На данных ПУСТ на 100%:** `present=0`
у **всех 22 видео**, `num_questions=0` из **всех** источников. Причина — тройной-мёртвый вход: title/desc mock (без
вопросов), **transcript не прокинут** (policy asr_only + `doc.asr` пуст = L8; даже 5 видео с реальным транскриптом
дали 0), comments mock. e5-large ни разу не запускается. Концепт живой, вход мёртвый; фиксится ASR-wiring.

## 3. Функционал

Работает в TextProcessor:

1. **Сбор текста** из включённых источников (use_title/description/transcript/comments), с лимитами (max_comments,
   max_chars, max_transcript_chars).
2. **Извлечение вопросов** — фразы с вопросительными словами (RU/EN словари) или «?», фильтры длины, dedup.
3. **Эмбеддинг** вопросов e5-large → матрица N×D.
4. **Метрики:** num_questions, per-source счётчики (q_title/description/transcript/comments), questions_per_min/1k,
   mean_cosine_to_centroid (насколько вопросы про одно), опц. privacy-артефакты (хэши/source-id вопросов).
5. Пишет 34 tp_qa_* + матрицу.

**Зачем продукту:** вопросы в контенте — **сигнал интерактивности/вовлечения**: «как сделать X», «что если Y» —
curiosity gap, драйвер удержания/комментариев. Плотность и связность вопросов = стиль подачи.

## 4. Вход

- **title/description** (mock, без вопросов), **transcript** (asr_only — **doc.asr пуст, L8** → 0), **comments** (mock).
- e5-large — эмбеддинг (не запускается: 0 вопросов).
- Нет вопросов → present=0.

## 5. Выход

- **34 tp_qa_* фич:** present, num_questions, embedding_dim, per-source счётчики (q_title/description/transcript/
  comments), плотности (per_min/per_1k), mean_cosine_to_centroid(+present), config/лимиты/policy-флаги, artifact-флаги.
- **Матрица N×D** эмбеддингов вопросов (при present).
- **NaN-политика:** 0 вопросов → present=0, метрики NaN/0.

## 6. Фичи (важное/неочевидное)

- **present=0 у всех 22 (главное)** — num_questions=0 из всех источников; компонент не произвёл ничего.
- **q_transcript=0 даже у 5 видео с реальным транскриптом** — прямое доказательство L8-wiring: transcript не доходит
  (asr_only читает `doc.asr`, который пуст, а не tp_artifacts-агрегаты).
- **q_title/q_description=0** — mock-заголовки без вопросительных слов/«?».
- **q_comments=0** — mock-комментарии (3 фикстуры) без вопросов.
- **mean_cosine_to_centroid** — связность вопросов (про одно vs разное) — хорошая идея, не задействована.
- **questions_per_min/1k** — плотность вопросов (интерактивность) — не задействована.
- **privacy-артефакты** (хэши/source-id вопросов) — продуманный контроль утечки текста; off.

## 7. Архитектура / алгоритм

- **Regex/словарь-извлечение вопросов** + **e5-large эмбеддинг** + centroid-косинус. Гибрид эвристика+модель.
- **Сложность:** e5-large инференс (когда есть вопросы); сейчас 0.
- **Детерминизм:** извлечение детерминировано; эмбеддинг — как e5.
- **Проблема:** нет вопросов на входе → пусто.

## 8. Оптимизации

- **Мультиисточник** вопросов (4 источника) — комплексная FAQ-картина.
- **Лимиты стоимости** (max_questions/comments/chars) — защита от разрастания.
- **dedup вопросов** — без повторов.
- **RU+EN словари** — двуязычность.
- **privacy-артефакты** (хэши вместо текста) — контроль утечки.
- **centroid-связность** — компактная мера когерентности.

## 9. Слабые места

- **Пуст на 100% (главное)** — тройной-мёртвый вход (mock title/desc, transcript не прокинут, mock comments).
- **ASR-wiring (L8)** — transcript не доходит (asr_only + doc.asr пуст); главный блокер, т.к. вопросы в речи —
  основной источник.
- **Зависит от mock-метаданных** — title/desc/comments фиктивны.
- **Извлечение по словам/«?» грубое** — риск ложных срабатываний на реальном тексте (риторические вопросы и т.п.).
- **Comments-вопросы = leakage** (пост-публикация) — для предсказания с осторожностью.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс., блокер] Прокинуть transcript** (L8) — вопросы в речи = основной источник; asr_only читает пустой doc.asr.
2. **[выс.] Реальные title/desc/comments** — сейчас mock без вопросов.
3. **[сред.] Уточнить извлечение** — отсеять риторические/шумовые «вопросы»; возможно, лёгкий классификатор.
4. **[низ.] Leakage-политика** для comments-вопросов в view-предсказании.

## 11. Рекомендации по архитектуре / связям

- **transcript → qa** (asr_only fix) — ключ; речь содержит большинство вопросов.
- **Единый e5-инференс** — шэрить эмбеддинг с semantics_topics/semantic_cluster (тот же e5-large) — экономия.
- **FAQ-профиль** (плотность+связность вопросов) → аналитика стиля/интерактивности.
