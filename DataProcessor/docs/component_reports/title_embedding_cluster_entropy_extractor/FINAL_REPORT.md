## 2. Резюме

`title_embedding_cluster_entropy_extractor` — **энтропия кластерного назначения заголовка**: title-эмбеддинг → PCA-
редукция → мягкое назначение к таксону кластеров (semantic_clusters_v1) через центроиды/FAISS → softmax по
расстояниям → энтропия / перплексия / margin («однозначно ли тема заголовка ложится в один кластер vs размыта по
многим»). **На данных — двойная смерть:** (1) **clusters DB = e2e_stub** — 32 фейк-кластера `cluster_0..cluster_31`
(случайные центроиды) → заголовок «не принадлежит» ни одному → softmax почти равномерный → **entropy_norm 0.93–0.995
≈ максимум**; (2) операнд **title mock** — лишь **2 различных значения на 22** (1.6 у 5, 1.5 у 17, разница только от
двух mock-заголовков). Тот же stub-паттерн, что semantics_topics и embedding_stats.topic_entropy. Механизм (PCA +
centroid-softmax + энтропия) корректен, но таксон фиктивен и вход mock.

## 3. Функционал

Работает после title-эмбеддера:

1. **Загрузка** title-эмбеддинга (relpath из tp_artifacts).
2. **PCA-редукция** (ассеты кластеров) → reduced_dim.
3. **Назначение к кластерам** — расстояния до центроидов (FAISS или numpy) → softmax(temperature) → распределение.
4. **Метрики:** entropy_raw/norm, perplexity, margin_top2 (уверенность top1 vs top2), distinct_clusters_topk,
   top_k_used.
5. Пишет 24 tp_titleclent_* (метрики + present/config/backend-флаги).

**Зачем продукту:** энтропия темы заголовка — **однозначность позиционирования**: чёткий заголовок ложится в одну
тему (низкая энтропия), размытый/вагий — в много (высокая). Мягкий сигнал ясности/качества заголовка.

## 4. Вход

- **title-эмбеддинг** (bge, **mock** — 2 варианта).
- **clusters-ассеты** (PCA/центроиды, **e2e_stub 32 кластера**).
- Нет title → present=0.

## 5. Выход

- **24 tp_titleclent_* фич:** entropy_raw/norm, perplexity, margin_top2, distinct_clusters_topk, top_k_used,
  n_clusters, model_orig/reduced_dim, present/title_present, config (top_k_slots/temperature/faiss), backend_faiss,
  dim_mismatch, compute_ms.
- **NaN-политика:** нет title → present=0; margin/n_clusters частично NaN.

## 6. Фичи (важное/неочевидное)

- **entropy_norm 0.93–0.995 ≈ максимум (главное)** — заголовок распределён почти равномерно по top-k кластерам →
  «не принадлежит» ни одному. Прямое следствие **stub-кластеров** (случайные центроиды).
- **Только 2 различных значения на 22** — 1.6 у 5 (title A), 1.5 у 17 (title B); варьируется лишь из-за двух mock-
  заголовков. Нулевой реальный сигнал.
- **margin_top2 = NaN везде** — уверенность top1-vs-top2 не пишется (export/расчёт off); потеря полезной метрики.
- **backend_faiss=0** — FAISS не задействован (fallback numpy центроиды); FAISS-режим — балласт на малом таксоне.
- **n_clusters=NaN** в фичах, хотя таксон 32 — метаданное поле не заполнено.
- **perplexity 4.48–4.96** — ≈ e^entropy; при 5 top-k слотах ~5 = максимум (равномерно) — подтверждает вырождение.

## 7. Архитектура / алгоритм

- **PCA + centroid-softmax** (numpy, опц. FAISS) над title-эмбеддингом. Прямой модели нет; ассеты из dp_models.
- **Строгая загрузка** (fail-fast в __init__, legacy clusters_path запрещён — Audit v3).
- **Сложность:** тривиальная (PCA-проекция + расстояния до 32 центроидов).
- **Детерминизм:** детерминирован (по построению).

## 8. Оптимизации

- **PCA-редукция** — быстрее и устойчивее в низкой размерности.
- **FAISS-режим** (задел под большой таксон) — сейчас не нужен.
- **Строгая загрузка ассетов** (Audit v3) — воспроизводимость, запрет legacy-путей.
- **Slot-clamp** top-k + temperature — контроль распределения.

## 9. Слабые места

- **Stub-кластеры (главный блокер)** — 32 e2e_stub → энтропия ≈ максимум у всех; сигнала нет. Общий с
  semantics_topics/embedding_stats корень (фиктивный таксон).
- **Title mock** — 2 значения на 22; даже с реальным таксоном операнд фиктивен.
- **margin_top2/n_clusters не заполнены** — потеря метрик, недоделка контракта.
- **Надстройка над двумя слабыми входами** (mock title + stub clusters) — двойная зависимость.
- **FAISS-балласт** — режим/флаги без пользы на малом stub-таксоне.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс., блокер] Реальный таксон кластеров** — заменить e2e_stub на осмысленные семантические кластеры (жанры/
   темы). Тогда энтропия станет реальной «однозначностью темы заголовка».
2. **[выс.] Реальные заголовки** — сейчас mock (2 значения).
3. **[сред.] Заполнить margin_top2 / n_clusters** — полезные метрики уверенности, сейчас NaN.
4. **[низ.] FAISS — только при большом таксоне** (сейчас балласт).

## 11. Рекомендации по архитектуре / связям

- **Общий кластерный таксон** для title_cluster_entropy + semantic_cluster_extractor + embedding_stats — единый
  реальный DB, шэрить центроиды/PCA.
- **Энтропия заголовка → сигнал ясности** в аналитику («заголовок чёткий/размытый по теме»).
- **Единый e5/PCA-инференс** — не пересчитывать проекции в каждом кластерном компоненте.
