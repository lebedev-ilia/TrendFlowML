  (boxes/scores/class_ids/valid_mask/**track_ids**/centers/areas/counts) присутствуют, формы верны.

- **Стабильность:** ✅ **golden подтверждён этой сессией** — 2 прогона на тех же кадрах дали
  **идентичные `track_ids` И `boxes`** (детектор + трекер полностью детерминированы).
- **Различимость (трекинг):** **когерентные длинные треки** (mean 52–127 кадров, max до 295) при
  **низкой фрагментации** (frac_single в осн. 0.0). 2–3 трека/видео — реалистично. Это подтверждает,
  что appearance-трекер (эмбеддинг бокса + ассоциация) решает задачу на реальном контенте разной длины.
- **Модель-fit:** `boxes/centers/areas/class_ids/valid_mask` (N,M) + `track_ids` (N,M) — seq-поток
  детекций для downstream (AR строит клипы по person-трекам; shot_quality — по боксам). track_ids
  дают когерентность person-траекторий.

## Нюанс (не блокер)
Прогон выполнен на `yolo11l.pt` (COCO-80) → распределение классов COCO (top: potted_plant/chair/person).
**Канонично компонент использует `yolo11x_41_best.pt`** (41-класс таксономия проекта). На трекинг это
не влияет (person = class 0 в обоих). Для прод-таксономии downstream — использовать 41-класс модель.

## Оптимизация
object_det ~45–59s (YOLO + histogram-трекер) на разреженной выборке; Segmenter 30–72s (зависит от
длины). Трекер — overhead ~единицы % (histogram дёшев). Для 200k: YOLO батч (уже 16), fp16-опция,
OSNet-эмбеддер (ветка готова) для толпного/длинного контента. Тайминги — ledger.
