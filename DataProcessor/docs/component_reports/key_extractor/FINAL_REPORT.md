## 2. Резюме

`key_extractor` — **детектор музыкального ключа** (Krumhansl-Schmuckler по хроме): определяет тональность
(12 нот × major/minor = 24 класса), `key_id`, `key_name/mode` и **уверенность** оценки (5 фич + per-segment).
**Полностью живой** (все 6 ok, 0 NaN, golden=0). Два **образцовых для аудио-семейства решения:** (1) **переиспользует
хрому** (`chroma_reused=True`) — не считает свой STFT, правильное слоение (chroma = профиль высот → key =
интерпретация в ключ); (2) **честная уверенность** — на речевом контенте key спурьёзен, и компонент **честно
помечает `key_confidence_category='low'`** (confidence 0.19–0.51 на речи, 0.67–0.73 на тональной музыке -Q6fnPIy/
-8WeWWOp) — в отличие от `tempo`, который давал ложно-высокую уверенность. Ограничение: ключ осмыслен **только на
музыке**, а на речевом корпусе (доминирует) спурьёзен (честно с низкой уверенностью). Нишевый музыкальный признак.

## 3. Функционал

Работает после Segmenter/chroma:

1. **Reuse хромы** (12-класс профиль из chroma_extractor).
2. **Krumhansl-Schmuckler** — корреляция хрома-профиля с 24 эталонными профилями тональностей → лучший ключ +
   `key_confidence` (сила корреляции).
3. `key_id` (0–23), `key_name`/`key_mode`, `key_scores` (все 24), per-segment `key_id/confidence_by_segment`.

**Зачем продукту:** музыкальный ключ (D minor, A# major) — **нишевый музыкальный инсайт** (тональность саундтрека,
настроение: minor = грустно/драматично, major = весело). Осмысленно для музыкального/гейминг-контента.

## 4. Вход

- **Аудио/хрома** (Segmenter + reuse chroma) — нет аудио → `status=empty` (`audio_missing`).

## 5. Выход

- **5 табличных фич:** `key_id`, `key_confidence` + мета.
- **Meta:** `key_name`, `key_mode`, `key_confidence_category` (low/…), `key_low_confidence_warning`, `key_method=librosa`.
- **Per-segment:** `key_id/confidence_by_segment`, `key_scores` (24), `segment_*_sec`, `segment_mask`.
- **Sentinel:** mask=False → key_id=−1.

## 6. Фичи (важное/неочевидное)

- **`key_confidence` — честный gate (образцово)** — 0.19–0.51 на речи (спурьёзный ключ), 0.67–0.73 на тональной
  музыке. `key_confidence_category='low'` + `key_low_confidence_warning` **честно сигналят «нет реального ключа»**.
  Это правильная модель (в отличие от tempo-confidence, которая лгала).
- **`key_name/mode`** — D minor / A# major и т.д.: интерпретируемый музыкальный ярлык (minor↔настроение).
- **Reuse хромы (`chroma_reused=True`)** — не дублирует STFT; правильное слоение поверх chroma_extractor. Единственный
  из аудио-семейства, кто явно переиспользует upstream (образец для консолидации остальных).
- **-Q6fnPIy = A minor, conf 0.67** — тональный музыкальный ролик получает осмысленный ключ (согласуется с
  chroma-тональностью/mel-энергией/pitch-нотой этого видео).
- **`key_scores` (24)** — распределение по всем тональностям (не только топ-1) — для неоднозначных случаев.

## 7. Архитектура / алгоритм

- **Krumhansl-Schmuckler** (корреляция хрома-профиля с 24 профилями тональностей) — классический music-theory метод.
- **Reuse chroma** — не считает CQT заново.
- **Детерминизм:** golden max|Δ|=0.0 (детерминированная корреляция).

## 8. Оптимизации

- **Reuse хромы** — образцовая экономия (не дублирует спектр) — единственный такой в аудио-семействе.
- **Честная категория уверенности** (`low`/warning) — правильный gate качества.
- **key_scores (все 24)** — полное распределение для неоднозначности.

## 9. Слабые места

- **Нишевость + спурьёзность на речи (главное)** — ключ осмыслен только на музыке; на речевом корпусе (доминирует)
  key спурьёзен (честно low-confidence, но всё же выдаётся). Для большинства контента бесполезен.
- **Низкая уверенность у большинства** — 4/6 видео confidence <0.55 (речь) → ключ ненадёжен. Реально полезен на ~2 видео.
- **Зависит от качества хромы** — ошибки chroma_extractor наследуются.
- **Короткое/смешанное аудио** — key detection нестабилен (K-S требует достаточной тональной статистики).
- **Пересечение с chroma** концептуальное (тональность) — но реализовано правильно (reuse, не дубль).

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[сред.] Не выдавать key при low-confidence** (или `key_name='none'`) — на речи спурьёзный ключ вводит в
   заблуждение, если игнорировать warning; сделать confidence-gate жёстким.
2. **[сред.] Объединить в тональный модуль с chroma** — chroma → key как один компонент (reuse уже есть, слить контракт).
3. **[низ.] Показывать key только на верифицированной музыке** (chroma_entropy низкая + key confidence высокая).
4. **[низ.] Настроение из mode** (minor→драматично/major→весело) — как готовый аналитический ярлык.

## 11. Рекомендации по архитектуре / связям

- **Тональный модуль** = chroma (профиль) → key (интерпретация) + музыкальность-gate — уже почти так (reuse).
- **Образец reuse** для остального аудио-семейства (spectral/mel/mfcc/tempo/onset/rhythmic не шэрят — key шэрит).
- **Настроение музыки** (mode + chroma) для аналитики.

## 13. Интерпретируемость

- **Хорошая (на музыке):** «D minor / A# major» — понятный музыкальный ярлык; minor↔настроение.
- **Добавить:** «настроение: минорное/драматичное» из mode (при confidence); скрывать key при low-confidence.

## 14. Польза для моделей

**Низкая-умеренная.** Ключ + confidence — нишевый музыкальный признак; на речевом корпусе (доминирует) спурьёзен и
low-confidence → мало сигнала для большинства данных. Полезен на музыкальном контенте (тональность/настроение
саундтрека). Честная confidence позволяет модели дисконтировать спурьёзные ключи. Ограниченная ценность вне музыки.

## 15. Польза для аналитиков

**Умеренная (нишевая).** «Тональность/ключ саундтрека» (D minor) + настроение (minor/major) — понятный музыкальный
инсайт для музыкального/гейминг-контента. Честный low-confidence warning корректно предупреждает «нет ключа» на
речи. Но для большинства (речевого) контента неприменим.
