## 2. Резюме

`title_to_hashtag_cosine_extractor` — **самый лёгкий derived-компонент**: одна метрика — косинус между title-
эмбеддингом и hashtag-эмбеддингом (11 полей). Продуктовый смысл полезный и конкретный — **hashtag-релевантность (SEO)**:
соответствуют ли хэштеги теме заголовка/видео (нерелевантные/спам-теги → хуже ранжирование/discovery). **Механизм
здоров:** `present=1` у **всех 22**, все диагност-флаги чисты. Но оба операнда **mock** (title + hashtag) → cosine —
**mock-константа: лишь 2 значения** (0.846 у 5, 0.851 у 17, разница только от двух mock-заголовков). Входит в
«alignment-семейство» (cosine_metrics + pair_topk + этот) — кандидат на консолидацию. Корректный, чистый, но фиктивный
на mock-данных.

## 3. Функционал

Работает после эмбеддеров:

1. Достаёт relpath'ы title/hashtag эмбеддингов из `doc.tp_artifacts.embeddings`.
2. Грузит оба (safe-join, size-check).
3. **L2-нормирует** и считает косинус (dim-check, zero-norm-check).
4. Пишет 11 tp_titlehashcos_* (cosine + present/require/6 диагност-флагов).

**Зачем продукту:** «подходят ли хэштеги под заголовок» — **SEO-сигнал релевантности тегов**: высокий косинус = теги
про то же, что видео (хорошо для discovery); низкий = нерелевантные/спам-теги. Прямой actionable инсайт для креатора.

## 4. Вход

- **title-эмбеддинг** (bge, **mock** — 2 варианта).
- **hashtag-эмбеддинг** (bge, **mock**).
- Отсутствует операнд → present=0, cosine NaN.

## 5. Выход

- **11 tp_titlehashcos_* фич:** cosine, present, title_present, hashtag_present, require-флаги (title/hashtag),
  диагностика (unsafe_relpath/title_embed_missing/hashtag_embed_missing/dim_mismatch/zero_norm).
- **NaN-политика:** нет операнда → present=0, cosine NaN; dim-mismatch/zero-norm → NaN + флаг.

## 6. Фичи (важное/неочевидное)

- **cosine = 0.846/0.851 (главное)** — только 2 значения на 22; оба операнда mock → нет реальной вариации. Разница
  лишь между двумя mock-заголовками (title A/B).
- **present=1 у всех 22, флаги чисты** — механизм полностью работоспособен (в отличие от NaN-компонентов).
- **hashtag_present=1 у всех** — hashtag-эмбеддинг всегда есть (mock hashtag-эмбеддер отрабатывает).
- **Единственная content-метрика + 10 служебных** — очень лаконичный контракт (в отличие от 69-полей pair_topk).
- **Высокий абсолют 0.85** — mock title и mock hashtag семантически близки (mock-фикстуры похожи); на реальных данных
  ожидается более широкий диапазон.

## 7. Архитектура / алгоритм

- **Чистый numpy** — L2-норма + dot. Своей модели нет.
- **Безопасность:** safe-join relpath (path-traversal guard), dim/zero-norm проверки, missing-vs-unsafe различены.
- **Сложность:** тривиальная. **Детерминизм:** тривиально детерминирован.

## 8. Оптимизации

- **Reuse эмбеддингов** (не пересчитывает).
- **Различение unsafe/missing/bad_file** — точная диагностика (аккуратно).
- **Лаконичный контракт** (11 полей) — минимум балласта.

## 9. Слабые места

- **Оба операнда mock (главное)** — cosine mock-константа (2 значения); реальная hashtag-релевантность не задействована.
- **Дублирует «alignment-семейство»** — концептуально рядом с cosine_metrics/pair_topk (косинусы частей текста);
  кандидат на объединение.
- **Одна метрика** — ценность целиком зависит от качества title/hashtag эмбеддингов.
- **Нет градаций** (по числу хэштегов, per-tag релевантность) — только агрегатный косинус к общему hashtag-эмбеддингу.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Реальные title/hashtag** — тогда cosine станет полноценным SEO-сигналом релевантности тегов.
2. **[сред.] Per-tag косинусы** — не только общий hashtag-эмбеддинг, но релевантность каждого тега (найти
   нерелевантные/спам-теги точечно).
3. **[сред.] Консолидировать в alignment-блок** (cosine_metrics + pair_topk + этот) — единый набор косинусов.
4. **[низ.] Нормировать/калибровать** порог «релевантно» на реальном распределении.

## 11. Рекомендации по архитектуре / связям

- **Единый «title-alignment» блок** — title↔desc, title↔transcript (cosine_metrics), title↔hashtag (этот),
  best-segment (pair_topk) в один компонент без дублирования.
- **hashtag-релевантность → SEO-аналитика** — «ваши теги подходят/не подходят под видео» + подсказка тегов.
- **Per-tag → рекомендации тегов** (заменить нерелевантные на релевантные из таксона).

умеренный вспомогательный признак.
