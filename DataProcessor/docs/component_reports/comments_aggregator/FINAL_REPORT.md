## 2. Резюме

`comments_aggregator` — **агрегация эмбеддингов комментариев в video-level вектор** «мнение аудитории»:
взвешенный пулинг (mean/median) с масками **authority/likes/recency** (важные комментарии весомее) → 1024-d L2
+ 27 фич (дуальный namespace tp_commentsagg_/tp_cagg_). **Данные MOCK** (наследует comments_embedder): 3 фикстур-
комментария, **все 6 идентичны** (косинус 1.0), взвешивание не применено (`weights_applied`/`std_enabled=0`).
Продуманный дизайн (взвешивание, mean+median robust) не задействован на mock. Как и comments_embedder — **leakage-
риск** (комментарии пост-публикация → следствие просмотров) для предсказания. Нужны реальные комментарии +
leakage-политика.

## 3. Функционал

Работает после `comments_embedder`:

1. Берёт эмбеддинги отобранных комментариев.
2. **Взвешенный пулинг** (mean/median) с масками authority/likes/recency.
3. Пишет `comments_agg_{mean,median}.npy` + tp_commentsagg_* мета (count, dim, present, weights, timings, флаги).

**Зачем продукту:** video-level мнение аудитории — **консолидированный сентимент/реакция** (что думают зрители в
среднем/медиане, взвешенно по важности). Кормит Fusion (текст-ветка). Взвешивание по лайкам/авторитету → важные
комментарии определяют агрегат.

## 4. Вход

- **`comments_embedder`** (hard) — эмбеддинги; нет → present=0. В батче — 3 mock-комментария.

## 5. Выход

- **Артефакты:** `comments_agg_{mean,median}.npy` (1024,) L2.
- **27 фич (tp_commentsagg_/tp_cagg_):** count, dim, present, mean/median_std, weights_mask_{authority,likes,recency},
  weights_applied/align, compute-флаги, timings, dim_mismatch.
- **NaN-политика:** нет комментариев → present=0.

## 6. Фичи (важное/неочевидное)

- **Все 6 идентичны (MOCK)** — наследует mock comments; count=3, косинус 1.0. Нулевой реальный сигнал.
- **Дизайн взвешивания (authority/likes/recency)** — важные комментарии весомее; сильная идея, но `weights_applied=0`
  на mock (инфра готова, не проверена).
- **mean + median** — median робастен к outlier-комментариям (тролли/спам); хорошая идея.
- **Дуальный namespace (tp_cagg_ + tp_commentsagg_)** — 27 фич в двух префиксах = избыточность контракта.
- **weights_align_present/shape_ok** — проверки согласования весов с эмбеддингами (аккуратно).

## 7. Архитектура / алгоритм

- **Взвешенный пулинг** (numpy mean/median) эмбеддингов + L2. Модели нет.
- **Сложность:** тривиально.
- **Детерминизм:** golden (заявлен PASS).

## 8. Оптимизации

- **Reuse эмбеддингов** comments_embedder.
- **Взвешивание** (authority/likes/recency) — приоритет важных комментариев.
- **mean+median** — робастность.
- **Align-проверки** весов.

## 9. Слабые места

- **MOCK (главное)** — все 6 идентичны; реальный агрегат мнения не задействован.
- **LEAKAGE-риск** — комментарии пост-публикация → утечка при предсказании просмотров/лайков (как comments_embedder).
- **Взвешивание не применено** (weights_applied=0) — сильный дизайн не проверен.
- **Дуальный namespace** — избыточность (tp_cagg_ vs tp_commentsagg_).
- **Тонкая надстройка** — по сути пулинг поверх comments_embedder; собственного мало.

## 10. Рекомендации по улучшению алгоритма (приоритизированные)

1. **[выс.] Реальные комментарии** — сейчас mock (все идентичны).
2. **[выс.] Leakage-политика** — только доступные на момент прогноза комментарии (или исключить из view-предсказания).
3. **[сред.] Задействовать взвешивание** (authority/likes/recency) на реальных данных.
4. **[низ.] Убрать дуальный namespace** — консолидировать tp_cagg_/tp_commentsagg_.

## 11. Рекомендации по архитектуре / связям

- **comments_embedder → comments_aggregator → Fusion** — с leakage-контролем.
- **Взвешивание по лайкам/авторитету** — модель важности комментариев (закрепить на реальных данных).
- **Сентимент-производные** (позитив/негатив/споры) из агрегата — для аналитика.
