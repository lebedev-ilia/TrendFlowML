Компонент: `video_pacing` (VisualProcessor). Темп монтажа: shot/склейки + 3 кривые (motion/semantic/color) + 57 табличных скаляров.
Schema `video_pacing_npz_v3`, producer_version **2.0.1**. Hard deps (no-fallback, union-domain): `cut_detection`, `core_clip`, `core_optical_flow`.

## Источник чисел
Полный GPU-прогон (Segmenter→deps→video_pacing) в эту сессию **НЕ выполнен**: дневной бюджет исчерпан
($29.33 / soft-stop $22.50, осталось $0.00 → под поднимать нельзя). Валидация проведена на **22 реальных
NPZ** из `storage/result_store/youtube` (6 уникальных видео × runs), все schema v3 / producer 2.0.1.
⚠️ Эти NPZ произведены со СТАРЫМ гейтингом (entropy/hist/bursts = OFF, «вариант A»), т.е. 8 фич, которые
активирует согласованный дефолт B, здесь ещё NaN. Логика этих фич — простой numpy (entropy/gini/histogram),
низкий риск; их фактические значения нужно подтвердить коротким прогоном варианта B.

## Критерии под компонент
### C1 — различимость (порог CV>0.30 у ≥3 из 4 метрик) → **PASS (4/4)**
Across 6 уникальных видео:
| Метрика | CV | mean | std |
|---|---|---|---|
| cuts_per_10s | **0.377** | 1.979 | 0.746 |
| shot_duration_mean | **0.311** | 3.826 | 1.192 |
| mean_motion_speed_per_shot | **0.805** | 0.119 | 0.096 |
| frame_embedding_diff_mean | **0.694** | 0.241 | 0.167 |
Все 4 > 0.30 (нужно было 3). Пер-видео разброс: cuts10 0.94–3.45, sdm 1.45–5.35, motion 0.012–0.301, sem 0.010–0.554.

### C2 — согласованность деп → **PASS**
- 22/22: три кривые длины N (motion/semantic/color).
- `motion_norm_per_sec_mean` ≥ 0 для всех finite (0 отрицательных).
- Доли ∈ [0,1]: 0 нарушений (short_shot_fraction, share_of_high_motion_frames/shots, high_change_frames_ratio).
- `shots_count` = S (число shot-boundaries) согласовано: N=65→S4/shots4, N=119→S7/shots7, N=12→S2/shots2 и т.д.

### C3 — NaN-политика → **PASS**
Union NaN-фич по 22 NPZ = 15, все объяснимы:
- **13 гейтнутых (вариант A)**: shot_duration_entropy, shot_length_gini, tempo_entropy, 5×shot_length_histogram_5bins,
  pace_curve_peaks_mean_prominence, pace_curve_dominant_period_sec, pace_curve_power_at_period,
  semantic_change_burst_count, color_change_bursts. При дефолте B первые 8 оживают → остаётся 5 by-design.
- **2 structural**: climax_speed, pacing_symmetry — NaN СТРОГО при shots_count<4 (подтверждено: shots=2,3 → NaN;
  shots=4,7 → finite). Известный edge (durations[3*quarter:] пусто). By-design, зафиксировано в CRITERIA.

### C4 — golden → **PENDING (GPU), прокси есть**
Чистый golden (фикс деп + пиннинг потоков, 2-й прогон только video_pacing) не выполнен (бюджет).
Прокси: 17 прогонов -Q6fnPIybEI с ИДЕНТИЧНЫМИ frame_indices (N=12) → feature_values+кривые расходятся
на **max|Δ|=9.6e-05**. Это дреф на уровне ДЕП (core_clip/core_optical_flow пересчитывались каждый раз
без пиннинга/с GPU-стохастикой), а не самого video_pacing. Сам модуль при фикс. входе — чистый numpy/opencv;
по аналогии с color_light/core_optical_flow при `OMP_NUM_THREADS=1` ожидается побайтовый diff=0.
